Presents ToMAS, a framework and tool for incrementally rewriting inter-schema mappings when source or target schemas change structurally or in their constraints, formalizing mapping consistency via maximal logical associations over nested relational schemas with key/foreign-key chase, decomposing every change into a small operator set (rename, copy, move, create, delete element; add/remove constraint), and ranking candidate rewritings by relative similarity and batch support so user-authored mapping choices survive evolution.
The algorithm preserves user-chosen mappings by distinguishing user associations from chase-derived ones, generates the full set of consistent rewritings per operator, and is evaluated on five relational/XML schemas (ProjectGrants, DBLP, TPC-H, Mondial, GeneX) where it sustains 70-95% user-effort savings vs. from-scratch regeneration through 17 successive schema changes.
Directly relevant to propstore's source-of-truth migration discipline (e.g., the concept→concepts cutover affecting documents.py, sidecar projections, world model, graph builders, aspic_bridge, and web views): the operator-decomposition + logical-association membership criterion + user-choice preservation + similarity ranking together describe exactly the cross-mapping consistency problem propstore migrations encounter, and the architecture (Mapping Analyzer, Evolution Engine, Ranker) maps onto a propstore concept-schema migration toolchain.
