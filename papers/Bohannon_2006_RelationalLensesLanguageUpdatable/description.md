Defines a combinator language of relational lenses (selection, projection, join), each a paired get/put operation typed by a functional-dependency set in tree form, where well-typedness statically guarantees the bidirectional lens laws GetPut and PutGet (with PutPut for very-well-behaved variants). Establishes that updates on a derived view propagate uniquely back to the source under explicit FD constraints, supersedes Bancilhon-Spyratos view-complement and Cosmadakis-Papadimitriou projection-key methods by giving update policies as syntactic combinators rather than system-chosen translations, and extends the FGMPS tree-lens framework into the relational setting. Foundational for propstore's migration framework and non-commitment principle: the round-trip laws give a formal information-preservation criterion that any storage-collapsing migration must satisfy before being accepted, and the compositional combinator pattern aligns with propstore's render-time policy stack over the underlying corpus.
