Introduces the original Truth Maintenance System (TMS), a problem-solver subsystem that records justifications for beliefs and incrementally maintains the current belief set using support-list and conditional-proof justifications, enabling non-monotonic reasoning and dependency-directed backtracking when contradictions arise. Key contributions include the formal definitions of justifications, well-founded support, the seven-step truth maintenance algorithm, the dependency-directed backtracking procedure with nogood creation and culprit selection, and control patterns for encoding defaults, alternative sequences, and equivalence classes in assumption structures. This is the foundational paper for the entire truth maintenance line of work --- the system that de Kleer's ATMS was designed to improve upon, and the conceptual substrate for the propstore's assumption-tracking and belief-revision architecture.
