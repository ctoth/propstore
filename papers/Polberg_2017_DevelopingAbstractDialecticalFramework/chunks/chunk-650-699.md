## Theorems / Propositions / Lemmas

### Theorem 4.9.3 *(p.650)*
EAS-to-SETAF (Translation 72). `S` conflict-free in `ES` may not yield a conflict-free `S' ⊆ A'` of `SF^ES` with `S = ⋃S'`. If `S` is **strongly self-supporting** in `ES`, such `S'` exists. For `σ ∈ {admissible, complete, preferred, grounded, stable}`, σ-extensions translate both ways. (proof p.650-653, four-page extension-level case analysis)

### Theorem 4.9.4 *(p.653)*
EAS-to-SETAF (Translation 73, attack-propagated). Strongly self-supporting → conflict-free in `SF^ES`. σ-extensions translate both ways for `σ ∈ {admissible, complete, preferred, grounded, stable}`. The conflict-free direction does not commute. Counterexample: `({η,a,b,c},{({a},b)},{({η},a),({η},b),({b},c)})`. (proof p.653-654)

### Theorem 4.9.6 *(p.654)*
EAS-to-defender-SETAF (Translation 75). Defines `S_np`. Conflict-freeness preserved one-way; `S ∪ S_np` need not be conflict-free in `SF^ES` if `S` is only conflict-free. If `S` is strongly self-supporting, it is. σ-extension preservation for `σ ∈ {admissible, preferred, complete, grounded, stable}`. Reverse direction: σ'-extension `S' ⊆ A'` ⇒ `S = S' ∩ A` is σ'-extension for `σ' ∈ {conflict-free, admissible, preferred, complete, grounded, stable}`; conflict-free reverse may fail to be strongly self-supporting. (proof p.656-658)

### Theorem 4.9.7 *(p.658)*
EAS-to-defender-SETAF (Translation 76, **strongly valid** EAS). Same statement as 4.9.6 but with `S_np = {a' | for every self-supporting set C for a, ∃S' ⊆ S, c ∈ C \ {a} s.t. (S',c) ∈ R}` (no "no self-supporting set" clause). (proof p.659-661)

### Theorem 4.9.8 *(p.661)*
Support-singular, attack-binary, strongly valid EAS → BAF (Translation 77). `BF^ES` is **support acyclic** and `η` is the unique unsupported argument. (proof: η orders A topologically.) (proof p.661)

### Theorem 4.9.10 *(p.661)*
Attack-binary EAS → AFN (Translation 78). `FN^ES` strongly consistent ⇒ `ES` strongly consistent. Reverse fails. If `ES` all-supported and strongly consistent, then `FN^ES` is strongly consistent. `FN^ES` need not be in minimal form even if `ES` is. Weak/strong validity preserves; weak+relation-validity preserves relation-validity. (proof p.661-662)

### Theorem 4.9.11 *(p.662)*
Attack-binary EAS → AFN (Translation 78). `(a₀,...,aₙ)` is an **evidential sequence** for `a` on `S` in `ES` iff it is a **powerful sequence** for `a` on `S` in `FN^ES`. `S` self-supporting in `ES` iff coherent in `FN^ES`. (proof p.662-663)

### Theorem 4.9.12 *(p.663)*
Same translation. For self-supporting/coherent `S`, `a` acceptable w.r.t. `S` in `ES` iff `a` defended by `S` in `FN^ES`. (proof p.663)

### Theorem 4.9.13 *(p.663)*
Same translation. (Strongly) self-supporting in `ES` iff (strongly) coherent in `FN^ES`. σ-extensions coincide for `σ ∈ {conflict-free, admissible, preferred, complete, grounded, stable}`. (proof p.664-665)

### Theorem 4.9.14 *(p.665)*
EAS → AFN (Translation 79). Strong consistency translates neither direction in general. If `ES` all-supported & strongly consistent, `FN^ES` strongly consistent. Minimal form not preserved. Validity preservation parallels 4.9.10. (proof p.665-666)

### Theorem 4.9.15 *(p.666)*
EAS → AFN (Translation 79, with `att(S)` augmentation). `S` conflict-free in `ES` ⇒ both `S` and `S' = S ∪ att(S)` conflict-free in `FN^ES`. Strongly self-supporting in `ES` ⇒ both (strongly) coherent in `FN^ES`. σ-extension preservation; reverse from `FN^ES` to `ES` via `S = S' ∩ A`, with caveats: conflict-freeness in `FN^ES` does not preserve to `ES`; coherent does ⇒ self-supporting; strongly coherent does not ⇒ strongly self-supporting. (proof p.666-670)

### Theorem 4.9.16 *(p.670)*
Strongly consistent EAS → ADF (Translation 80). `D^ES` is **BADF**. If `ES` all-supported, `D^ES` is **cleansed**. If `ES` all-supported & minimal, `D^ES` is **redundancy-free**. Weak validity preserves to `D^ES`. Minimal+all-supported+relation-valid → relation-valid. Strong validity → `D^ES` is **AADF⁺**. Minimal+strongly-valid → strongly valid. (proof p.671-674)

### Theorem 4.9.17 *(p.674)*
Strongly consistent EAS → ADF (Translation 80). For an evidential sequence on `S` for `s ∈ S` we can construct a corresponding **acyclic pd-evaluation** in `D^ES` and vice versa. `S` self-supporting in `ES` iff **pd-acyclic** in `D^ES`. (proof p.674-675)

### Theorem 4.9.18 *(p.675)*
Strongly consistent EAS → ADF (Translation 80). `S ⊆ A` strongly self-supporting in `ES` iff **pd-acyclic conflict-free** in `D^ES`. (proof p.675-676)

### Lemma 4.9.19 *(p.676)*
Strongly consistent EAS → ADF, `S` self-supporting conflict-free / pd-acyclic conflict-free. `S` attacks `a` or every set e-supporting `a` in `ES` iff the **acyclic range** `v^a_S` of `S` blocks every acyclic pd-evaluation of `a` in `D^ES`. (proof p.676)

### Lemma 4.9.20 *(p.677)*
Same setup. `a` acceptable w.r.t. `S` in `ES` iff **decisively in** w.r.t. `v^a_S` in `D^ES`. (proof p.677)

### Theorem 4.9.21 *(p.677)*
Strongly consistent EAS → ADF. σ-extensions correspond: `σ ∈ {admissible, preferred, complete}` in `ES` ↔ aa-σ-extension in `D^ES`. Stable in `ES` ↔ stable in `D^ES`. Grounded in `ES` ↔ **acyclic grounded** in `D^ES`. (proof p.677-678)

### Theorem 4.10.1 *(p.679)*
ADF-to-AF (Translation 82). pd-acyclic conflict-free / aa-admissible / aa-complete / aa-preferred / stable / acyclic grounded extensions of `D` correspond to conflict-free / admissible / complete / preferred / stable / grounded extensions of `F^D_AA` via `S = ⋃ Fᵢ` from `S' = {(F₁,B₁),...,(Fₙ,Bₙ)}`. (proof p.679-682, exhaustive case-by-case round trip)

### Theorem 4.10.2 *(p.682)*
ADF-to-AF (Translation 83). pd-acyclic conflict-free / **ac-admissible / ac-complete / ac-preferred** / grounded extensions of `D` correspond to conflict-free / admissible / complete / preferred / grounded of `F^D_AC`. (Stable not stated.) (proof p.682-685)

### Theorem 4.10.3 *(p.685)*
ADF-to-AF (Translation 85). cc-σ-extension semantics on `A^ev` ↔ standard σ-extension on `F^D_CC` for `σ ∈ {conflict-free, admissible, complete, preferred, grounded}`. Construction `T = ⋃ᵢ Fᵢ ∪ Gᵢ`. (proof p.685-687)

### Theorem 4.10.4 *(p.687)*
ADF-to-AF (Translation 86). ca₂-extension semantics ↔ AF semantics on `F^D_{CA₂}` for `σ ∈ {ca₂-admissible, model, ca₂-complete, ca₂-preferred, grounded}` ↔ `{admissible, stable, complete, preferred, grounded}` (with model ↔ stable). (proof p.687-689)

### Theorem 4.10.5 *(p.689, completes p.698)*
ADF-to-AF labeling correspondence (Translation 86). Three-valued admissible / complete / preferred / grounded labeling `v` of `D` corresponds bijectively to admissible / complete / preferred / grounded labeling `v'` of `F^D_lab`, with `v^t = ⋃^E in(v')` and `EV^p(v^f) ⊆ out(v')` (admissible) or `v^f = ALL(out(v'))` (complete/preferred/grounded). The reverse map is not bijective for **admissible** semantics — pointer to **Example 153 on page 429** for counterexample. The grounded-direction reverse uses Theorem 2.1.17 to derive a contradiction with strictness of `out(z') ⊂ out(v')`. (proof p.690-698)

### Theorem 4.10.8 *(p.698, in progress at p.699)*
ADF-to-SETAF (Translation 87). For a conflict-free / cc-admissible / cc-complete / cc-preferred / grounded extension `S ⊆ A` of `D`, there is a corresponding conflict-free / admissible / complete / preferred / grounded extension `S' = {(F₁,G₁,B₁),...,(Fₙ,Gₙ,Bₙ)} ⊆ A^{ev}` of `SF^D_CC` such that `S = ⋃ Fᵢ ∪ Gᵢ`. Reverse: σ-extension on SETAF side recovers cc-σ on ADF side. Translation 87 forces attacks in `A^{ev}` to be **size-1** (single evaluation argument attackers), which simplifies the proof of `X_S ⊆ T^+`. (proof in progress p.698-699; spills into next chunk)

## Algorithms / Procedures

No numbered algorithms. Each proof is itself a constructive procedure for translating an extension/labeling between source and target frameworks; the construction recipes are:

1. **EAS→SETAF complete extension construction** *(p.651-652)*: For `S ⊆ A` strongly self-supporting, build `S' = {X | X is a minimal self-supporting set for some a ∈ S, X ⊆ S}`.
2. **EAS→SETAF defender extension construction** *(p.656)*: Augment `S` with `S_np` (or `np(S)`); claim `S ∪ S_np` is the σ-extension of `SF^ES`.
3. **EAS→AFN extension construction** *(p.666-668)*: `S' = S ∪ att(S)`; reverse `S = S' ∩ A`.
4. **EAS→ADF acyclic-pd-evaluation construction** *(p.674-675)*: For evidential sequence `(η, a₀,...,aₙ)`, choose minimal supporters as decisively in interpretations and construct an acyclic pd-evaluation; pd-sequence equals evidential sequence.
5. **ADF→AF extension construction** *(p.679-682)*: From `S' = {(F₁,B₁),...,(Fₙ,Bₙ)}` set `T = ⋃ᵢ Fᵢ`. Show `X_S = T^{a+}`.
6. **ADF→AF labeling-bridge construction** *(p.690-697)*: From three-valued `v` of `D`, take maximal two-valued subinterpretation `v_2`; partition pd-evaluations into `I^ev`/`O^ev` and breaker arguments into `I^b`/`O^b`; assign `in` to `I^ev ∪ I^b` and `out` to `O^ev ∪ O^b`.

## Comparisons With Prior Work

This chunk is internal scaffolding; it does not compare with external prior work directly. It does, however, expose important structural commitments:

- ADF aa-/ac-/cc-/ca₂-σ semantics are designed so that **AF-side computation** of standard σ-extensions on `F^D_xx` is sufficient to recover the desired ADF semantic. The target AF in each case is a finite Dung framework, which makes the chain `ADF → AF → existing AF solver` viable. *(p.679, 682, 685, 687)*
- The **labeling-side** construction in 4.10.5 is **not** symmetric for admissible semantics — admissible labelings of `F^D_lab` may fail to correspond to admissible labelings of `D` (Example 153, p.429). For complete/preferred/grounded, the correspondence is bijective. *(p.692-697)*

## Design Rationale

- **Why `S_np` rather than direct projection** *(p.654-660)*: Polberg adds `S_np` to the SETAF-side set so that primed (auxiliary) arguments representing evidential dependence are properly defended. The chosen path prevents under-defending in the target while preserving conflict-free-style behaviour up to the strongly-self-supporting boundary.
- **Why strong validity for Translations 75/76** *(p.654, p.658)*: Without strong validity, the auxiliary `np(S)` set would not necessarily contain all needed primed arguments because some standard arguments might lack any e-supported attack chain on A. The proofs show the strong-validity hypothesis is necessary, not merely convenient.
- **Why `att(S)` for Translation 79** *(p.666)*: Translation 79 already augments the EAS with attack arguments to mirror the AFN attack-set semantics; without `att(S)` the conflict-free direction collapses and stable extensions cannot be transferred.
- **Why partially acyclic evaluations for Translations 85/86** *(p.685, p.687)*: Standard pd-evaluations would conflate self-blocking and self-attacking; partially acyclic evaluations distinguish the two and let the discarded set `T^+` be recovered cleanly.
- **Why the maximal two-valued subinterpretation `v_2` in Theorem 4.10.5** *(p.690)*: A three-valued ADF labeling does not directly induce a two-valued AF labeling; `v_2` extracts the two-valued "skeleton" so blocked/unblocked status is well-defined and bijective for complete/preferred/grounded, with admissible flagged as the asymmetric case.

## Limitations Acknowledged

- **Translation 73 (attack-propagated SETAF) does not preserve conflict-free** *(p.653-654)*. Stated explicitly with worked counterexample.
- **Translation 75 (defender SETAF)** does not preserve conflict-freeness without strong self-support *(p.656)*. Conflict-free reverse direction does not always recover strong self-support *(p.656)*.
- **Translation 78 (AFN)** strong-consistency is one-way only *(p.665)*. Worked counterexample. Minimal form not preserved *(p.662)*.
- **Translation 79 (AFN with att augmentation)** strong-consistency, minimal form parallel limitations *(p.665-666)*. Conflict-free / strongly coherent reverse does not always recover EAS-side property *(p.668-670)*.
- **Translation 86 admissible labeling correspondence is not bijective** *(p.692)*. Author defers to Example 153 on p.429 — pointer for the cross-chunk synthesis.

## Implementation Notes

- For propstore, the relevant takeaway is that **EAS / SETAF / BAF / AFN / ADF / AF** form a translation lattice with clearly-stated semantic fidelity for each pair. propstore's argumentation surface (`propstore.aspic_bridge`, the AF construction in `aspic.py`) currently targets **AF**; if EAS or AFN evidential semantics are needed, this chunk gives the constructive recipes (and counterexamples) for round-tripping into AF.
- The `S_np`, `att(S)`, `acyclic range v^a_S`, and `partially acyclic discarded set E^{p+}` constructions are the **specific functions** an implementer would need to write when adding ADF or EAS extension translation to propstore.
- Theorem 4.10.5's labeling bijection (complete/preferred/grounded only, admissible asymmetric) means a labeling-driven ADF solver can be implemented by reduction to AF for those three semantics, but **admissible** must be solved natively or via explicit enumeration through Example 153's caveat.
- Stable semantics needs a separate code path in every translation — the inductive ⊆-maximality argument that handles complete/preferred/grounded does not carry over.
