## Key Findings

- EAF→ADF translations split into: consistent EAFs (Translation 47), general EAFs via bypass arguments (Translation 48), consistent EAFCs (Translation 49), general EAFCs via bypass (Translation 50). All four target ADFs whose acceptance conditions are EAF-style propositional formulas (`¬b ∨ ⋁ D_{b,a}`) per attacker, conjoined across attackers. *(p.301-318)*
- The acceptance-condition shape "≤1 negated literal per clause, with each negation unique among clauses" is named **EAF-style** and characterizes ADFs producible from consistent EAF/EAFC translations. *(p.302, p.314)*
- Bounded hierarchical EAFs/EAFCs produce AADF⁺ ADFs in relation+strongly valid forms; ca₂ semantic family collapses to the regular ADF semantics in that subclass via Theorem 3.5.2. *(p.305, p.315)*
- The EAFC family (Translation 49/50) has the widest input range and is the only EAF-class translation that is both **full and exact**. *(p.321 = printed 308)*
- BAF-AF translations rely on adding **indirect attacks** so the support relation can be dropped; three approaches exist: attack propagation (CLS13/CLS15), coalition (CLS10 deprecated; CLS13 newer), defender. *(p.324, p.331-339)*
- The original CLS10 coalition translation (Definition 4.7.6) is explicitly marked **Deprecated** because it does not preserve semantics in general. *(p.331-332)*
- BAF→AFN translation (Translation 59) is the only BAF translation in the chapter that is **modular and structural**. *(p.339, p.349 = Table 4.8)*
- BAF→EAS (Translation 60) requires adding a single evidence node `η`, making source-target subclass + injective; it loses modularity. *(p.341-343)*
- BAF→ADF gets **no formal translation** because abstract support is conceptually orthogonal to ADF acceptance conditions; the section gives only a discussion of why supported attack and overpowering support are problematic. *(p.345-348)*

## Definitions Introduced

### Definition 4.6.34 — Inconsistency origin (EAF) *(p.308)*
Let `EF = (A, R, D)` be an EAF and `a ∈ A`. The **inconsistency origin** of `a` is `O^a = { b ∈ A | ∃ c ∈ A s.t. (b, (c, a)) ∈ D and (b, a) ∈ R }`. If `a` is strongly consistent, `O^a = ∅`. Notation `O^E = ⋃_{a∈E} O^a`.

### Definition 4.6.35 — Replacement function (EAF) *(p.308)*
Given `S ⊆ A`, replacing arguments `P^b = { e^b | e ∈ S }`,
`rep(S, P^b, D_{b,a}) = D_{b,a}` if `D_{b,a} ∩ S = ∅`; otherwise `D'_{b,a} = (D_{b,a} \ S) ∪ { e^b | e ∈ D_{b,a} ∩ S }`.

### Definition 4.6.44 — Inconsistency origin (EAFC) *(p.316)*
Let `EFC = (A, R, D)` be an EACF, `a ∈ A`. **Inconsistency origin**:
`O^a = { b ∈ A | ∃ c ∈ A, B ⊆ A s.t. b ∈ B, (B, (c, a)) ∈ D and (b, a) ∈ R }`.
If `a` strongly consistent, `O^a = ∅`.

### Definition 4.6.45 — Replacement function (EAFC) *(p.317)*
With `D_{b,a} = { E | (E, (b, a)) ∈ D }` and `S ⊆ A`, `P^b = { e^b | e ∈ S }`:
`rep(S, P^b, D_{b,a}) = D_{b,a}` if `∀ B ∈ D_{b,a}: B ∩ S = ∅`; otherwise
`D'_{b,a} = { (B \ S) ∪ { e^b | e ∈ B ∩ S } | B ∈ D_{b,a} }`.

### Definition 4.7.6 — Coalition (Deprecated) *(p.318)*
Let `BF = (A, R, S)` be a BAF, `BF^{att} = (A, R)`. `C ⊆ A` is a **coalition** of `BF` iff:
- the support subgraph `(C, S ∩ (C × C))` induced by `C` is connected,
- `C` is conflict-free in `BF^{att}`, and
- `C` is maximal under `⊆` among the sets satisfying the previous two conditions.

### Definition 4.7.7 — Deductive coalition *(p.320)*
Let `BF = (A, R, S)` be a BAF with `S` a deductive support. The **deductive coalition** associated with `a ∈ A`: `C(a) = {a} ∪ { b | a supports b }` (closure of `a` under support edges).

## Translations Introduced

### Translation 47 — Consistent EAF → ADF *(p.301)*
`EF = (A, R, D)` bounded hierarchical or strongly consistent without symmetric attacks.
For conflict `(b, a) ∈ R`, `D_{b,a} = { c | (c, (b, a)) ∈ D }`.
ADF `D^{EFC} = (A, L, C)` with `L = { (a, b) | (a, b) ∈ R or ∃ x s.t. (a, (x, b)) ∈ D }`.
- functional form: for `B ⊆ par(a)`, `C_a(B) = out` if `∃ x ∈ B s.t. (x, a) ∈ R and ∄ b ∈ B s.t. (b, (x, a)) ∈ D`; else `in`.
- propositional form: `att^b_a = ¬b ∨ ⋁ D_{b,a}` (or `¬b` if `D_{b,a} = ∅`); `C_a = ⋀_{b∈A,(b,a)∈R} att^b_a`; `C_a = ⊤` if `a` unattacked.

### Translation 48 — General EAF → ADF *(p.310)*
`EF = (A, R, D)` bounded hierarchical or no symmetric attacks. Let `E` be the non-strongly-consistent arguments, `A^b = { a^b | a ∈ O^E }` the bypass arguments. For attack `(b, a)`, `D'_{b,a} = rep(O^a, A^b, D_{b,a})`.
ADF `D^{EF} = (A', L, C)` with `A' = A ∪ A^b`, `L = { (a, b) | (a, b) ∈ R, or ∃ (c, b) ∈ R, a ∈ D'_{c,b}, or b = a^b }`.
- functional form: for `a ∈ A`, `C_a(F) = out` if `∃ x ∈ F s.t. (x, a) ∈ R and ∄ f ⊆ F s.t. f ∈ D'_{x,a}`; else `in`. For `a^b ∈ A^b`: `C_{a^b}(∅) = out`, `C_{a^b}({a}) = in`.
- propositional form: as 47 with `D'_{b,a}`; for `a^b ∈ A^b`, `C_{a^b} = a`.

### Translation 49 — Strongly consistent EAFC → ADF *(p.313)*
`EFC = (A, R, D)`. For conflict `(b, a)`, `D_{b,a} = { E | (E, (b, a)) ∈ D }` (sets, not single args).
`L = { (a, b) | (a, b) ∈ R or ∃ c ∈ A, E ⊆ A s.t. a ∈ E and (E, (c, b)) ∈ D }`.
- functional form: `C_a(B) = out` if `∃ x ∈ B s.t. (x, a) ∈ R and ∄ B' ⊆ B s.t. (B', (x, a)) ∈ D`; else `in`.
- propositional form: with `D_{b,a} = {B_1, …, B_m}`, `att^b_a = ¬b ∨ (⋀ B_1 ∨ … ∨ ⋀ B_m)` (or `¬b` if empty). `C_a = ⋀ att^b_a` (or `⊤` if `a` unattacked).

### Translation 50 — General EAFC → ADF *(p.317)*
Mirrors Translation 48 but using set-defense replacement (Definition 4.6.45). Bypass argument `a^b` for inconsistency-origin args; functional/propositional forms parallel Translation 48 with set-based `D'_{b,a}`. ADF target's `L` includes `(a, b) | b = a^b` and the indirect-defense links from set-defenders.

### Translation 51 — BAF (deductive) → AF (attack propagation, CLS13) *(p.323)*
`BF = (A, R, S)` deductive-support BAF. `R' = {R^{sup}, R^{med}_{R^{sup}}} ⊆ R^{ind}`.
**Associated attack-propagation AF** for deductive support: `apd-F^{BF} = (A, R ∪ ⋃ R')`.

### Translation 52 — BAF (necessary) → AF *(p.324)*
`BF = (A, R, S)` necessary-support BAF, `R' = {R^{sec}, R^{ext}} ⊆ R^{ind}`.
`apn-F^{BF} = (A, R ∪ ⋃ R')`.

### Translation 53 — General BAF → attack propagation AF *(p.324)*
`BF = (A, R, S)`, `R' ⊆ R^{ind}` chosen indirect attacks. **Attack propagation AF**: `F^{BF} = (A, R ∪ ⋃ R')`.

### Translation 54 — Inverse closure attack propagation–defender AF *(p.325-326)*
`BF = (A, R, S)`, `R' ⊆ R^{ind}`. `iclo-F^{BF} = (A', R'')` where `A' = A ∪ S` (support edges become arguments) and
`R'' = R ∪ ⋃ R' ∪ { (x, x) | x ∈ S } ∪ { (b, (b, a)), ((b, a), a) | a, b ∈ A, (b, a) ∈ S }`.
Used for **i-admissibility**: each support edge becomes a self-conflicting auxiliary argument that the source attacks and that attacks the target — defending the target requires accepting the support edge, which forces accepting the supporter.

### Translation 55 — Closure attack propagation AF *(p.327)*
`clo-F^{BF} = (A', R'')` with `A' = A ∪ S` and `R'' = R ∪ ⋃ R' ∪ { (x,x) | x ∈ S } ∪ { (a, (b,a)), ((b,a), b) | a, b ∈ A, (b,a) ∈ S }`. The orientation of the auxiliary attacks is reversed vs. Translation 54 — used for **c-admissibility**.

### Translation 56 — Faulty (deprecated) coalition translation *(p.331)*
`c-F^{BF} = (A', R')` where `A' = { C | C is a coalition of BF }` (Definition 4.7.6) and `R' = { (C_1, C_2) | ∃ a_1 ∈ C_1, a_2 ∈ C_2 s.t. a_1 R a_2 }`. Marked as not preserving semantics.

### Translation 57 — Deductive coalition AF *(p.334)*
`cd-F^{BF} = (A', R')` with `A' = { C(a) | a ∈ A }` (deductive coalitions) and `R' = { (C_1(a), C_2(b)) | ∃ a_1 ∈ C_1(a), a_2 ∈ C_2(b) s.t. a_1 R a_2 }`.

### Translation 58 — BAF (necessary) → defender AF *(p.338)*
`BF = (A, R, S)`. **Defender AF for necessary support**:
`dn-F^{BF} = (A', R')`, `A' = A ∪ S`, `R' = R ∪ { (b, (b, a)) | a, b ∈ A, (b, a) ∈ S } ∪ { ((b, a), a) | a, b ∈ A, (b, a) ∈ S }`.
Each support link becomes an auxiliary argument; the supporter attacks the link, the link attacks the supported argument. Footnote 14 notes "b does not support a" would be more natural reading.

### Translation 59 — Support-acyclic BAF → AFN *(p.340)*
`BF = (A, R, S)` support-acyclic. **Associated AFN**:
`FN^{BF} = (A, R, N)`, where `N = { ({a}, b) | (a, b) ∈ S }`. Each support edge becomes a singleton support set in the AFN. Resulting AFN is binary, acyclic, in minimal/weakly/relation/strongly valid forms (Theorem 4.7.13).

### Translation 60 — Support-acyclic BAF → EAS *(p.342)*
`BF = (A, R, S)` support-acyclic. `ES^{BF} = (A ∪ {η}, R', E)`,
`R' = { ({a}, b) | (a, b) ∈ R }`, and
`E = { ({η}, a) | a ∈ A, ∄ c ∈ A s.t. cSb } ∪ { (S^a, a) | S^a is the collection of all arguments b ∈ A s.t. bSa }`.
Adds evidence node `η`. Each argument either gets evidence directly (if no supporters) or via the *whole* set of its supporters (singular EAS, not binary).

## Theorems / Propositions / Lemmas

### Theorem 4.6.28 *(p.302)*
Let `EF` be a bounded hierarchical EAF or strongly consistent EAF without symmetric attacks. `D^{EF}` via Translation 47 is a BADF, in redundancy-free, cleansed, weakly valid form. Not necessarily AADF⁺. If `EF` is bounded hierarchical, then `D^{EF}` is AADF⁺ in relation and strongly valid forms.

### Theorem 4.6.29 *(p.302)*
Under same hypotheses, `E ⊆ A` is a conflict-free extension of `EF` iff it is conflict-free in `D^{EF}`.

### Lemma 4.6.30 *(p.302)*
Discarded set of `E` in `EF` coincides with the partially acyclic discarded set of `E` in `D^{EF}`.

### Lemma 4.6.31 *(p.302)*
A conflict-free `E` defends `a ∈ A` in `EF` iff `a` is decisively in w.r.t. partially acyclic range `v^p_E` of `E` in `D^{EF}`.

### Theorem 4.6.32 *(p.303)*
Bounded-hierarchical / strongly-consistent EAF without symmetric attacks: `E` is conflict-free / stable / grounded / σ-extension of `EF` iff it is conflict-free / model / acyclic-grounded / ca₂-σ-extension of `D^{EF}`, for `σ ∈ {admissible, complete, preferred}`.

### Theorem 4.6.33 *(p.305)*
Bounded-hierarchical EAF case: σ-extensions of `bh-EF` correspond to xy-σ-extensions of `D^{EF}` for `x, y ∈ {a, c}` and `σ ∈ {admissible, preferred, complete}`. (Generalises Theorem 4.6.32 from ca₂-only to all xy-families because `D^{EF}` is AADF⁺.)

### Theorem 4.6.36 *(p.310)*
General EAF (BH or no-symmetric-attacks): `D^{EF}` via Translation 48 is BADF, redundancy-free, cleansed, weakly valid. AADF⁺ in relation/strongly valid forms iff `EF` is BH.

### Theorem 4.6.37 *(p.310)*
With `E^b` the bypass arguments generated by `E ⊆ A`:
- `E ⊆ A` cf in `EF` ⇔ `E ∪ E^b` cf in `D^{EF}`; reversal: `E' ⊆ A'` cf ⇒ `E' ∩ A` cf in `EF`.
- stable in `EF` ⇔ `E ∪ E^b` model of `D^{EF}`.
- grounded in `EF` ⇔ `E ∪ E^b` acyclic-grounded in `D^{EF}`.
- σ ∈ {admissible, complete, preferred}: `E` σ-extension ⇔ `E ∪ E^b` ca₂-σ-extension.

### Theorem 4.6.38 *(p.314)*
EAFC analog of 4.6.28: strongly consistent EAFC `EFC` → `D^{EFC}` (via Translation 49) is BADF, cleansed, weakly valid. AADF⁺ if `EFC` bounded hierarchical; redundancy-free if `EFC` minimal.

### Theorem 4.6.39 *(p.314)*
Strongly consistent EAFC: `E` cf in `EFC` iff cf in `D^{EFC}`.

### Lemma 4.6.40 *(p.314)*
Discarded set of `E` in `EFC` = partially acyclic discarded set in `D^{EFC}`.

### Lemma 4.6.41 *(p.314)*
`E` cf set defends `a` in `EFC` iff `a` decisively in w.r.t. partially acyclic range of `E` in `D^{EFC}`.

### Theorem 4.6.42 *(p.315)*
Strongly consistent EAFC: `E` cf/stable/grounded/σ-extension of `EFC` iff cf/model/acyclic-grounded/ca₂-σ-extension of `D^{EFC}` (`σ ∈ {admissible, complete, preferred}`).

### Theorem 4.6.43 *(p.315)*
Bounded hierarchical EAFC: σ-extension of `bh-EFC` iff xy-σ-extension of `D^{EFC}`, `x, y ∈ {a, c}`, `σ ∈ {admissible, preferred, complete}`. Stable/grounded handled separately.

### Theorem 4.6.46 *(p.318)*
General EAFC + Translation 50: `D^{EFC}` is BADF, cleansed, weakly valid. AADF⁺ in relation/strongly valid forms iff `EFC` is BH; redundancy-free iff minimal.

### Theorem 4.6.47 *(p.318)*
General EAFC + Translation 50, with bypass set `E^b`:
- `E` cf in `EFC` ⇔ `E ∪ E^b` cf in `D^{EFC}`.
- stable ⇔ model.
- grounded ⇔ acyclic-grounded.
- `σ ∈ {admissible, complete, preferred}`: `E` σ-extension ⇔ `E ∪ E^b` ca₂-σ-extension of `D^{EFC}`.

### Theorem 4.7.1 *(p.324)*
BAF + chosen indirect attacks `R'` + attack propagation AF `F^{BF}` (Translation 53): `E` is +cf in `BF` w.r.t. `R'` iff cf in `F^{BF}`; `E` is d-σ-extension of `BF` w.r.t. `(R', R')`, `σ ∈ {admissible, complete, preferred}`, iff σ-extension of `F^{BF}`; stable iff stable; d-grounded iff grounded.

### Theorem 4.7.2 *(p.325)*
Deductive BAF, `R' = {R^{sup}, R^{med}_{R^{sup}}}`, `apd-F^{BF}` via Translation 53. If `E` is a complete extension of `apd-F^{BF}`, then `E` is closed under `S` in `BF`.

### Theorem 4.7.3 *(p.325)*
BAF, `R' = {R^{sec}}`, `apd-F^{BF}` via Translation 53. If `E` is a complete extension of `apd-F^{BF}`, then `E` is inverse-closed under `S` in `BF`. (Inverse closure under both secondary AND extended attacks not certain — see Fig 4.60 caveat.)

### Theorem 4.7.4 *(p.326)*
BAF + `R' ⊆ R^{ind}` + inverse closure attack-propagation–defender AF `iclo-F^{BF}` via Translation 54: `E ⊆ A` is i-admissible (i-preferred) in `BF` w.r.t. `(R', R')` iff admissible (preferred) in `iclo-F^{BF}`.

### Theorem 4.7.5 *(p.328)*
Closure variant via Translation 55: `E ⊆ A` is c-admissible (c-preferred) in `BF` w.r.t. `(R', R')` iff admissible (preferred) in `clo-F^{BF}`.

### Theorem 4.7.8 — Deprecated *(p.320)*
Old CLS13 result: deductive BAF + `R' = {R^{sup}, R^{med}_{R^{sup}}}`. `apd-F^{BF}` via Translation 53, `cd-F^{BF}` via Translation 57. `E = {a_1, …, a_n}` σ-extension of `apd-F^{BF}` (`σ ∈ {cf, admissible, stable}`) iff `{C(a_1), …, C(a_n)}` σ-extension of `cd-F^{BF}`. Marked deprecated because the result is "not entirely correct" — the proof relies on a union casting function and the relation is not strong / not 1-1 in admissible/cf cases.

### Theorem 4.7.9 *(p.333)*
Corrected BAF→coalition result: deductive BAF, `R' = {R^{sup}, R^{med}_{R^{sup}}}`, `apd-F^{BF}` and `cd-F^{BF}` via Translations 53/57. If `E = {a_1, …, a_n}` σ-extension of `apd-F^{BF}` for `σ ∈ {complete, preferred, grounded, stable}`, then `E' = {C(a_1), …, C(a_n)}` σ-extension of `cd-F^{BF}`. If `E' ⊆ A'` σ-extension of `cd-F^{BF}`, then `E = ⋃ E'` σ-extension of `apd-F^{BF}`. (Union casting required.)

### Theorem 4.7.10 *(p.333-334)*
Multipart correspondence between BAF semantics and `cd-F^{BF}` extensions covering +cf-and-closed-under-S, c-admissible/c-preferred, d-complete/d-grounded/stable, and the inverse mappings (with `E = ⋃ E'`).

### Theorem 4.7.11 *(p.338)*
BAF specialised for necessary support, `dn-F^{BF}` via Translation 58. If `E ⊆ A'` admissible in `dn-F^{BF}`, then `E ∩ A` is inverse-closed under `S` in `BF`.

### Theorem 4.7.12 *(p.339)*
BAF specialised for necessary support, `R' = {R^{sec}, R^{ext}}`. If `E ⊆ A` is i-admissible in `BF` w.r.t. `(R', R')`, ∃ admissible `E' ⊆ A'` s.t. `E' ∩ A = E`. Reverse direction: if `E' ⊆ A'` admissible in `dn-F^{BF}`, then `E = E' ∩ A` *might not* be i-admissible in `BF` — explicit failure noted.

### Theorem 4.7.13 *(p.340)*
Support-acyclic BAF: `FN^{BF}` (Translation 59) is support-binary and acyclic; in minimal, weakly, relation and strongly valid forms.

### Theorem 4.7.14 *(p.340-341)*
BAF→AFN bidirectional semantic correspondence. `E ⊆ A`:
- +cf w.r.t. ∅ in `BF` ⇔ cf in `FN^{BF}`.
- inverse-closed under `S` in `BF` ⇔ coherent in `FN^{BF}`.
- +cf w.r.t. `R' = R^{sec}` and inverse-closed ⇔ strongly coherent.
- i-admissible w.r.t. `(R', R')` ⇔ admissible.
- i-preferred w.r.t. `(R', R')` ⇔ preferred.
- d-complete w.r.t. `(R', R')` ⇔ complete.
- d-grounded w.r.t. `R'` ⇔ grounded.
- stable w.r.t. `R'` ⇔ stable.

### Theorem 4.7.15 *(p.342)*
Support-acyclic BAF, `ES^{BF}` via Translation 60. `ES^{BF}` is attack-binary, support-singular, all-supported; in minimal, weakly, relation, strongly valid forms.

### Theorem 4.7.16 *(p.342-343)*
Support-acyclic BAF, `ES^{BF}` via Translation 60. For `X ⊆ A` and `R'' = {R^{sec}}`:
- +cf w.r.t. ∅ in `BF` if cf in `ES^{BF}`.
- inverse-closed under `S` in `BF` if `X ∪ {η}` self-supporting in `ES^{BF}`.
- +cf w.r.t. `R''` and inverse-closed if `X ∪ {η}` strongly self-supporting.
- i-admissible w.r.t. `(R'', R'')` if `X ∪ {η}` admissible.
- i-preferred w.r.t. `(R'', R'')` iff `X ∪ {η}` preferred.
- d-complete iff complete; d-grounded iff grounded; stable iff stable.
And for `X' ⊆ A ∪ {η}`: cf in `ES^{BF}` if `X' ∩ A` is +cf in `BF`; self-supporting if `X' ∩ A` is inverse-closed; strongly self-supporting analogously; admissible if `X' ∩ A` is i-admissible.

EAF `({a,b,c,d}, {(a,b),(b,a),(c,d),(d,c)}, {(a,(d,c)),(b,(c,d)),(c,(b,a)),(d,(a,b))})` from Example 5 page 19. Set `{a,b,c,d}` not cf in source but is in associated ADF; activates all support cycles. Demonstrates why ca₂ semantics over-generate.

### Example 115 *(p.311)*
Inconsistent EAF `({a,b,c}, {(a,b),(b,a),(c,b)}, {(c,(a,b))})` showing `c` both attacks `b` and defense-attacks the conflict to `b`. ### Example 116 *(p.315-316)*
EAFC over `{a,b,c,d,e,f}` with attacks `{(c,f),(d,e),(f,a)}` and group defense attacks `{({f},(d,e)), ({a,b,c},(d,e)), ({e},(c,f))}` from Example 11 page 29. ### Example 117 *(p.319-320)*
General EAFC `({a,b,c,d}, {(a,b),(b,a),(c,b),(b,d)}, {({c,d},(a,b))})` modifying Example 115. ADF `({a,b,c,d,c^b}, C_a=¬b, C_b=(¬a∨(c^b ∧ d))∧¬c, C_c=⊤, C_{c^b}=c, C_d=¬b)`. ca₂-admissible extensions include `∅, {c}, {c,c^b}, {a,c}, {a,c,c^b}, {c,d}, {c,d,c^b}, {a,c,d}, {a,c,d,c^b}`. ## Algorithms / Procedures

### Algorithm — EAF→ADF acceptance condition (Translation 47, propositional form) *(p.301)*

1. For each conflict `(b, a) ∈ R`:
   1. Collect defense attackers `D_{b,a} = { c | (c, (b, a)) ∈ D }`.
   2. Build attacker formula `att^b_a = ¬b ∨ ⋁ D_{b,a}`. If `D_{b,a} = ∅`, set `att^b_a = ¬b`.
2. For each `a ∈ A`:
   1. If `a` has no attacker, set `C_a = ⊤`.
   2. Else `C_a = ⋀_{b: (b,a) ∈ R} att^b_a`.
3. Set `L = { (a, b) | (a, b) ∈ R or ∃ x s.t. (a, (x, b)) ∈ D }`.
4. Return `(A, L, C)`.

### Algorithm — Bypass insertion for general EAF (Translation 48) *(p.309-310)*

1. Compute `E = { a ∈ A | a not strongly consistent }` (i.e., `O^a ≠ ∅`).
2. Generate bypasses `A^b = { a^b | a ∈ O^E }`.
3. For each conflict `(b, a)`, compute `D'_{b,a} = rep(O^a, A^b, D_{b,a})`.
4. Build acceptance conditions per Translation 47 substituting `D'_{b,a}` for `D_{b,a}`.
5. For each `a^b ∈ A^b`: set `C_{a^b}({}) = out`, `C_{a^b}({a}) = in` (functional) or `C_{a^b} = a` (propositional).
6. `L = { (a, b) | (a, b) ∈ R, or ∃ (c, b) ∈ R, a ∈ D'_{c,b}, or b = a^b }`.

### Algorithm — Inverse closure attack propagation–defender AF (Translation 54) *(p.325-326)*

1. Choose indirect attack family `R' ⊆ R^{ind}`.
2. New args `A' = A ∪ S` (each support edge becomes an arg).
3. For each `x ∈ S`, add self-attack `(x, x)`.
4. For each support edge `(b, a) ∈ S`: add `(b, (b, a))` and `((b, a), a)`.
5. `R'' = R ∪ ⋃ R' ∪ self-attacks ∪ defender-style attacks`.

## Semantics Defined / Compared

This chunk does not introduce new ADF/AF semantics. It uses:
- **For ADFs (target of EAF/EAFC translations):** ca₂-conflict-free, ca₂-admissible, ca₂-complete, ca₂-preferred, c₂-complete, model, acyclic-grounded, and the bounded-hierarchical-collapse families (xy-σ for `x, y ∈ {a, c}`, `σ ∈ {admissible, complete, preferred}`).

- **For AFs (target of BAF translations):** conflict-free, admissible, complete, preferred, grounded, stable.
- **For AFNs (target of Translation 59):** coherent, strongly coherent, admissible, preferred, complete, grounded, stable.

## Design Rationale

- **Why limit EAF→ADF to BH-or-no-symmetric-attacks subclass:** The ca₂ semantics over-generate on consistent EAFs with symmetric attacks (cycles get accepted). pd-acyclic conflict-freeness is too restrictive elsewhere — chosen path is to accept the subclass restriction rather than redesign ADF semantics. *(p.290 = pdf 304, p.302)*
- **Why bypass arguments instead of self-attacker patches** (general EAF): Avoids structural artifacts that could prevent admissible inclusion. The bypass `a^b` mediates between the inconsistency-origin source and the defended argument without introducing self-conflict in the source. *(p.309)*
- **Why NOT structure-preserving classification for Translation 47:** Defense attackers in EAFs go through indirect connections; in the ADF target they appear as direct edges. Even though no new arguments are added, the link topology changes — better classified as semi-structural. *(p.292 = pdf 305)*
- **Why not modular:** The defense-attack parts of acceptance conditions need to be wired differently than direct-attack parts; counterexamples at p.292-293 (PDF 305-306) show neither `⊕` nor `⊗` modularity is achievable. *(p.292-293)*
- **Why source-subclass-restricted for EAF translations even after bypass:** "Differing intuitions on conflict-free semantics" between EAFs and ADFs prevent a fully exact translation outside BH subclass. *(p.307, p.321 = pdf 320)*
- **Why self-attacker form rejected for EAF→ADF:** Discussion at p.307 (pdf 320) considers replacing bypass with the self-attacker normal form (Section 4.2.4.2) — full+exact in EAFC case but problematic for stable semantics; future work. *(p.320)*
- **Why no BAF→ADF translation:** Acceptance conditions in ADFs talk about "what should be present," conceptually opposite to abstract support's "what we are in favor of." ADF semantics parametrize w.r.t. support cycles rather than indirect-conflict types. Without redesigning ADFs (introducing strengths à la preference/structured argumentation), abstract support cannot be cleanly grasped. *(p.345-346)*
- **Why Translation 56 marked deprecated rather than fixed:** The conflict-freeness restriction within coalitions, combined with imprecise abstract-support interpretation, means defense in produced AF doesn't correspond to defense in source BAF — a structural rather than parametric problem. *(p.319)*
- **Why support cycles preclude the defender translation:** Arguments representing supports become attackers of `a` (and of `b`); if neither `a` nor `b` is independently attacked, the support args can be admitted, yielding extensions impossible in the source. *(p.338)*
- **Why BAF→AFN restricted to support-acyclic:** AFN's coherence semantics treat support cycles incompatibly with how BAFs treat them; common ground requires excluding cycles. *(p.339-340)*
- **Why singular EAS (not binary) in Translation 60:** In binary EAS, accepting any one supporter would suffice to derive a, but BAF necessary support requires the *whole* support set. Need singular EAS where the entire supporting set must be in the extension. *(p.341)*

## Implementation Notes

- **EAF/EAFC → ADF translations** map cleanly to acceptance-condition synthesis: each conflict `(b, a)` becomes a clause `¬b ∨ disjunction-of-defenders`. Implementation in `propstore.aspic_bridge` could use this translator if EAF input ever becomes a thing; the produced ADFs are propositionally simple (one negated literal per clause).
- **Bypass argument materialisation:** When ingesting general (non-strongly-consistent) EAFs, materialise `a^b` arguments as auxiliary nodes with `C_{a^b}=a`. These are pure adapters — safe to mark as derived/non-source nodes.
- **Inverse-closure auxiliary attacks** (Translation 54) provide a mechanical recipe for embedding BAF necessary-support constraints into a pure AF: each support edge becomes a self-conflicting auxiliary that the source attacks and that attacks the target. Useful if `propstore.aspic_bridge` ever needs to translate support-aware claim graphs into a pure Dung AF for the existing ASPIC+ flow.
- **AFN translation (Translation 59)** is the cleanest mapping: each support edge becomes a singleton support set in the AFN. **Modular and structural.** This is the recommended bridge from BAF-shaped support to AFN if propstore ever stores support-acyclic graphs.
- **`η` evidence node (Translation 60)** is a discipline pattern: any time we want to map a support-acyclic structure into a system requiring distinguished evidence, add a synthetic root and mark all initial supporters as supported by the root. The cost is target-subclass status.
- **Deprecated coalition translation (Definition 4.7.6, Theorem 4.7.8)** — if any propstore code or note references the CLS10 coalition translation, it should explicitly cite this deprecation and reference Theorems 4.7.9/4.7.10 (or Translation 57) as the correct version.
- **EAF-style condition shape** is identifiable mechanically: parse acceptance conditions; if every clause has ≤1 negated literal and negations are unique, the formula is EAF-style. Useful for round-tripping ADF-shaped data through EAF-aware code.
- **No BAF→ADF translator** is provided. If propstore needs to project ADFs from BAF-shaped data, fall back through BAF→AFN→AADF (using the AFN→ADF translator from earlier chapters) — this composition gives a route via the modular AFN bridge.
