## Key Findings

- This chunk is overwhelmingly **proof-only material**. Nearly every theorem
  and lemma here was *stated* earlier in Chapter 4 (the "Translations"
  chapter). The appendix supplies the omitted proofs. *(p.550-599)*
- The appendix is organized by translation family: validity-form
  translations (A3, continuing in this chunk), AF translations (A4), SETAF
  translations (A5), AFRA translations (A6), and EAF/EAFC translations (A7).
  *(p.570, 574, 584, 591)*
- The proofs establish **σ-extension preservation** (i.e. σ-extensions of the
  source framework correspond to σ-extensions of the translated framework)
  for σ ∈ {conflict-free, admissible, complete, preferred, grounded, stable,
  coherent, strongly coherent, self-supporting, strongly self-supporting,
  pd-acyclic, aa-admissible, aa-complete, acyclic grounded, etc.}. The
  recurring pattern is "iff" two-way preservation, sometimes degraded to
  one-way (e.g. some directions need σ' ∈ a smaller subset). *(p.550-599)*
- The chunk contains **subclass/normal-form lattice lemmas** (Lemma 4.2.67
  for EAF(C)s, Lemma 4.2.71 for AFNs, Lemma 4.2.73 for EAFs, Lemma 4.2.77
  for ADFs) — these define the strict-subset relations between the named
  normal forms (NDef, BH, SCons, NSup, SBin, SSig, Min, EvSup, AllSup, SV,
  WV, RV, SCons, ADF^AF ⊊ ADF^SETAF ⊊ (BADF ∩ AADF+), etc.). *(p.551-555)*
- Several theorems are sequence-existence characterizations:
  - Theorem 4.2.33: AFN strong validity ⇔ existence of a powerful sequence
    independent of support function. *(p.560)*
  - Theorem 4.2.34: AFN strong validity ⇔ induced support graph
    SG^FN is a directed acyclic graph (DAG). *(p.560)*
  - Theorem 4.2.38: EAS strong validity ⇔ existence of an evidential
    sequence over all arguments. *(p.561)*
  - Theorem 4.2.39: EAS strong validity ⇔ SG^ES is a rooted DAG with η as
    the root. *(p.562)*
  - Theorem 4.2.44: ADF strong validity ⇔ existence of a sequence
    inducing an acyclic pd-evaluation under any pd-function. *(p.564)*
  - Theorem 4.2.45: ADF strong validity ⇔ positive dependency graph
    PDG^D is a DAG. *(p.564)*

## Theorems / Propositions / Lemmas

### Theorem 4.2.58 (tail of proof) *(p.550)*
> Let ES = (A,R,E) be an EAS and ES^sc = (A',R,E') its corresponding
> strongly consistent framework obtained through Translation 16. Let
> S ⊆ A, S' ⊆ A' be sets of arguments and S^b the (possibly empty) set of
> bypass arguments generated by S in A'. S is a σ-extension of ES, where
> σ ∈ {conflict-free, self-supporting conflict-free, admissible, preferred,
> complete, grounded, stable}, iff it is a σ-extension of ES^sc.
> *(strong direction; reverse direction does not hold for stability when
> bypass arguments are present)*

Proof technique: stable extensions of ES^sc lift back to ES via bypass-
argument elimination; the auxiliary `A^b` arguments are self-attackers and
cannot appear in any extension. Coherent sets in `A \ E` carry over; the
counterexample for the reverse stability direction is that a stable set in
the bypass framework need not attack arguments in `A' \ A`.

### Theorem 4.2.18 *(p.551)*
> Let FN = (A,R,N) be an AFN and FN^wv = (A',R',N') be its weak validity
> form. A set E ⊆ A is a σ-extension of FN, where σ ∈ {coherent, strongly
> coherent, admissible, preferred, complete, grounded, stable}, iff it is a
> σ-extension of FN^wv. If E ⊆ A is a conflict-free extension of FN, then
> E ∩ A' is conflict-free in FN^wv. If E ⊆ A' is conflict-free in FN^wv,
> then it is conflict-free in FN.

Proof gloss: trimming preserves attacks (R' = R ∩ (A' × A')) and powerful
sequences (induction on the sequence). Defense correspondence is shown
through coherence preservation. *(p.551-552)*

### Theorem 4.2.33 *(p.560)*
> Let FN = (A,R,N) be an AFN. Let sup(a) = ⋃_{C⊆A, CNa} C denote all
> arguments supporting a, and suf(a) = {S | S ⊆ sup(a) and ∀C ⊆ s.t.
> CNa, C ∩ S ≠ ∅} stand for all subsets of sup(a) that have an element in
> common with every support set of a. FN is strongly valid iff there exists
> a sequence (a_0,...,a_n) of all arguments in A s.t. given any function
> f : A → {S | a ∈ A, S ∈ suf(a)}, (a_0,...,a_n) is a powerful sequence
> s.t. f(a_0) = ∅ and f(a_i) ⊆ {a_0,...,a_{i-1}} for i > 0.

### Lemma 4.2.77 *(p.567-569)*

- ADF^{SETAF} ⊊ (BADF ∩ AADF^+)
- ADF^{SETAF} ⊊ WV^{ADF}

arguments {a,b,c} with C_a, C_b each assigning `in` to ∅ only, and

The (a,c) link is redundant. *(p.568)*

## Algorithms / Procedures

No fully explicit numbered algorithms in this chunk, but the proofs of
Theorem 4.2.33, 4.2.34, 4.2.38, 4.2.39, 4.2.44, 4.2.45 are essentially
**constructive sequence-building procedures** that map directly to
implementation:

### Procedure — Powerful Sequence Construction (AFN strong validity) *(p.560)*
1. Initialize A_0 = {arguments requiring no support through N}.
2. If A_0 = A, terminate.
3. Find b ∈ A \ A_0 s.t. for every set S ∈ suf(b), S ⊆ A_0.
4. Add b to the sequence after A_0 (any order).
5. Collect all such arguments into A_1 ⊆ A; add A_1 to sequence.
6. Repeat with A_0 ∪ A_1 in place of A_0.
7. Terminate when all arguments have been placed in the sequence.

### Procedure — Acyclic pd-evaluation Sequence Construction (ADF strong validity) *(p.564)*
1. Initialize A_0 = {a ∈ A | min decisively-in interpretation v_a has v_a^t = ∅}.
2. Place A_0 at the start of the sequence (any order).
3. Find b ∈ A \ A_0 such that the minimal decisively-in interpretation
   v_b has v_b^t ⊆ A_0.
4. Collect all such arguments into A_1 ⊆ A; append A_1 to sequence.
5. Repeat until all arguments placed.

### Procedure — Topological Ordering Inversion (DAG ↔ powerful sequence) *(p.561, 564)*
- Forward: powerful sequence → topological ordering of (support graph or
  positive dependency graph).
- Backward: DAG topological ordering → powerful sequence/acyclic
  pd-evaluation under any support function/pd-function.

## Comparisons With Prior Work

This chunk repeatedly compares the named translations against three
broad backdrops:

- **Dung AFs** as the simplest target. The chunk shows that Dung AFs
  embed *cleanly* into ADFs, EAFs, and AFNs (Theorems 4.3.10, 4.3.18-
  4.3.20). The author repeatedly invokes Theorem 2.1.11 (AF
  characteristic operator) and 2.1.25 (AF preferred extensions) to map
  results between source and target frameworks. *(p.572, 573, 575, 581,
  584)*

- **AFRA** (Cayrol/Lagasquie-Schiex argument-on-attack generalization).
  The proofs show AFRA → AF translation preserves all standard semantics
  except that the conflict-free and admissible directions are not
  generally invertible. *(p.570-571, 584)*

- **AFN** (Argumentation Framework with Necessities, Nouioua) and
  **EAS** (Evidential Argumentation System). Polberg's proofs systematically
  preserve "coherence" in AFN and "self-supporting" in EAS through
  trimming/validity transformations. *(p.551-554, 559-562)*

- **BADF** (Brewka/Strass bipolar ADF). Lemma 4.2.77 shows
  ADF^{SETAF} ⊊ (BADF ∩ AADF+); BADF and AADF+ are *incomparable* —
  neither contains the other. The counterexample uses functional
  conditions making one (a,c) link redundant. *(p.567-568)*

- **SETAF** (Nielsen/Parsons set-attack AF). Theorems 4.4.1, 4.4.2, 4.4.6
  and 4.4.11-4.4.15 establish the full picture: SETAF translates cleanly
  into AF (admissible side), ADF (full bidirectional), and AFN
  (one-way conflict-free), with explicit caveats. *(p.574-583)*

Critique pattern: rather than rejecting prior frameworks, Polberg shows
they are *embedded* into ADF/EAF via translations, and uses the proof
appendix to *certify* this embedding. The criticism is implicit:
each prior framework collapses into a special case of the more general
ADF/EAF/EAFC machinery.

## Design Rationale

- **Two-direction proofs are not always available**: Polberg consistently
  states explicit "iff" preservation only when *both* directions are
  proved; otherwise, σ' is restricted to a smaller subset. This is a
  deliberate honesty design choice — admissible direction often fails for
  AFRA/AF translations because admissible AFRA extensions can include
  sets that don't lift back. *(p.570, 584, 591)*

- **Bypass arguments and self-attackers** are deliberately introduced as
  the implementation device for strongly consistent translations. Polberg
  argues the self-attacker construction is the cleanest way to keep
  bypassed inconsistency-origin arguments out of any extension. *(p.550)*

- **Strong validity is characterized via DAGs.** The author chooses
  graph-theoretic characterizations (Theorem 4.2.34, 4.2.39, 4.2.45)
  because they are *checkable in polynomial time*, in contrast to the
  set-of-functions characterization in Theorem 4.2.33, 4.2.38, 4.2.44
  which quantifies over all support/pd-functions. This is a deliberate
  computational accessibility choice — the framework supports two
  equivalent presentations and implementers can pick the cheaper one.
  *(p.560, 562, 564)*

- **Minimal forms preserve uniqueness of decisively-in interpretations.**
  Lemma 4.2.77 and surrounding work show that SETAF-style ADFs are
  always BADF and AADF+ but *not* always redundancy-free — the design
  choice is to keep the minimal form orthogonal to redundancy elimination.
  *(p.567-568)*

- **EAFC ↔ EAF translation does not preserve conflict-freeness in
  reverse**. Theorem 4.6.4 explicitly notes this design constraint — the
  authors chose not to force conflict-freeness across the translation
  because doing so would break admissibility correspondence. *(p.591)*

- **Proof-appendix orientation:** Polberg places proofs in the appendix
  rather than threading them through the chapters. Rationale: lets the
  body chapters remain readable as a translation-catalog while the
  appendix carries the heavy formal-correctness load.

## Limitations Acknowledged

- Reverse direction failures (explicit "*This does not necessarily hold for
  σ semantics*" or "*might not be*") are stated for nearly every theorem.
  These are not bugs — they are part of the design. *(p.570, 575, 588,
  591, 594, 598)*
- ADF translation 23 of Dung AFs is *not* arbitrary — it is bounded
  hierarchical (Theorem 4.3.8 referenced) and so its grounded extension
  can be characterized via the AF characteristic operator. The translation
  is not designed to handle arbitrary ADF→AF inversion. *(p.572)*
- The AFRA→AF (4.5.2) proof flags that admissibility fails to lift in one
  direction without restriction (E ≠ E^src case). *(p.587-588)*
- The SETAF→AFN translation has a one-way conflict-freeness issue: the
  AFN conflict-freeness is too coarse compared to SETAF's set-attack
  conflict-freeness. *(p.580-581)*
- The EAFC→EAF translation in Theorem 4.6.4 explicitly does not
  preserve conflict-freeness in reverse direction; analogously for
  bh-EAF→SETAF (Theorem 4.6.11) and bh-EAF→AFRA (Theorem 4.6.15).

## Implementation Notes

For someone implementing this in propstore:

- **Each translation is a typed mapping between framework objects.** In
  propstore terms, AFN→AF, EAS→AF, ADF→AF, SETAF→AF, AFRA→AF, EAFC→EAFC,
  bh-EAF→SETAF, bh-EAF→AFRA, bh-EAF→AFN are all candidate ports. The proofs
  in this chunk certify σ-extension preservation, so implementations can
  reuse extension solvers across translation boundaries — *but only for
  the σ-set proven*. The reverse direction is often partial; the
  translation must record this.

- **Bypass arguments and primed arguments are auxiliary objects** that
  exist only inside the translated framework. In storage terms, they
  must carry provenance pointing back to the source argument they
  represent. They should never appear in the source-of-truth concept
  layer.

- **Powerful sequences and pd-evaluations are essentially topological
  orderings** of induced support / positive-dependency graphs. The
  fast strong-validity check is "is the support/positive-dependency
  graph a DAG?" — implementable in O(V+E).

- **Reinstatement sets** (Theorem 4.6.4) are central to EAFC↔EAF
  translation and must be tracked as named entities, not inlined in the
  translation step. They are defeat-graph side conditions that can be
  cached.

- **Strongly consistent ↔ ADF AADF+ correspondence (Theorem 4.4.11)**
  means SETAF can be embedded into the ADF reasoning surface for free
  if SETAF is minimal. propstore's existing ADF surface should expose
  a `from_setaf(...)` constructor that uses Translation 31.

- **The deactivated set E^+ has different definitions in different
  frameworks** (defeats vs attacks vs union of parents). Implementations
  must not unify these semantically — Theorem 4.3.18 second bullet
  states E^+ in F coincides with the discarded set in D^F only if E
  is conflict-free.

- **AFRA→meta-AF translation introduces argument-on-attack indirection.**
  Implementing this requires representing attacks as first-class
  objects, which propstore's claim/stance model already supports via
  the support-revision module.
