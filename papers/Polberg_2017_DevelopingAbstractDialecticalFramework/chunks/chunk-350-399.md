## Key Findings

- Polberg systematizes translations from AFNs to AF, SETAF, BAF, EAF, EAS, ADF
  and then from EASs to AF, SETAF, ADF, classifying every translation along
  strength (cf/coh/str-coh/adm/comp/pref/grd/stb), functional, syntactical,
  semantical and computational dimensions, summarized in Table 4.9 *(p.385)*.
- AFN→AF coalition translation (Translation 61) is **strong** under
  grounded/complete/preferred/stable but only a **subclass-style** approach for
  conflict-free; the AFN/AF admissible relation is **not 1-1** but **becomes
  1-1 at complete semantics** *(pp.350–353)*.
- AFN→AF stable semantics: there exists **no** full exact translation under
  identity casting (Theorem 4.8.7, the AFN stable extensions need not be tight)
  *(p.355)*.
- AFN→AF admissible/preferred: a full exact translation **does** exist (Theorem
  4.8.6); precise construction left open *(p.354)*.
- AFN→SETAF has three families: chained (via AF–SETAF), attack propagation
  (Translation 63/64), and defender (Translation 65/66). Defender is the only
  one that handles arbitrary AFNs via the "you are unsupported" auxiliary `a'`.
- AFN→ADF requires **strong consistency** for the direct translation
  (Translation 69); general AFNs are first normal-formed via bypass
  (Translation 13) or self-attacker (Translation 15) consistency forms, then
  fed into Translation 69 (Translation 70) *(pp.378–384)*.
- The aa-family of ADF semantics (admissible/complete/preferred + their pd-
  acyclic variants) is what corresponds to AFN coherence-based semantics
  because AFN coherence requires support-wise acyclicity *(p.379)*.
- EAS→AF coalition translation (Translation 71, restating ORL10 with PO14b
  corrections — empty set is also self-supporting) is **not exact** — Theorem
  4.9.2 proves no full translation can be exact under conflict-free, admissible,
  complete, preferred, or stable semantics with identity casting, because EASs
  use group attack and AFs use binary attack *(p.390)*.
- Defender EAS→SETAF (Translation 75) follows AFN defender pattern but the
  primed `a'` "is not e-supported" is added for **every non-η argument**
  (since in EAS only η is valid by default), unlike AFNs where only supported
  arguments need primed counterparts *(p.395)*.

## Theorems / Propositions / Lemmas

### Theorem 4.8.1 *(p.351)*

Let `FN = (A,R,N)` be an AFN and `F^{FN} = (A',R')` its corresponding AF
obtained through Translation 61. A set `E ⊆ A` is a σ-extension, where σ ∈
{grounded, complete, preferred, stable}, of FN iff `F^{FN}` admits a set `Y =
{C_1, ..., C_n} ⊆ A'` s.t. `E = ⋃_{i=1..n} C_i` is a σ-extension.

### Theorem 4.8.2 *(p.351)*

Let `FN = (A,R,N)` be an AFN and `F^{FN}` its corresponding AF built from
Translation 61. If `E' ⊆ A'` is conflict-free in `F^{FN}`, then `⋃E'` is
conflict-free in FN, **but not vice versa**. A set `E ⊆ A` is a strongly
coherent extension of FN iff `F^{FN}` admits a set `E' = {C_1, ..., C_n} ⊆ A'`
s.t. `E = ⋃ C_i` as a conflict-free extension. A set `E ⊆ A` is admissible iff
`F^{FN}` admits a set `E'` s.t. `⋃ E' = E` as admissible. For every complete
extension of FN there exists exactly one corresponding complete extension of
`F^{FN}` (one-to-one at complete semantics).

### Theorem 4.8.3 *(p.351)*

Let `FN = (A,R,N)` and `F_{cf}^{FN} = (A,R)` from Translation 62. A set
`E ⊆ A` is a conflict-free extension of FN iff it is a conflict-free
extension of the AF `F_{cf}^{FN}`.

### Theorem 4.8.4 *(p.355)*

Let FN be an AFN and `E,E' ⊆ A` two admissible extensions of FN. If for every
`a ∈ E, b ∈ E'` it is not the case that `(a,b) ∈ R` and `(b,a) ∈ R`, then
`E ∪ E'` is also admissible.

### Theorem 4.8.5 *(p.355)*

For any two admissible extensions `E_1, E_2` of FN, if for every argument
`a ∈ E_1, b ∈ E_2` there exists an admissible extension `E_3` of FN s.t.
`a, b ∈ E_3`, then `E_1 ∪ E_2` is an admissible extension of FN.
(The adm-closed property of Definitions 2.3.2/2.3.3 redefined for AFNs.)

### Theorem 4.8.6 *(p.355)*

There exists a full translation from `Fr^{AFN}` to `Fr^{AF}` that is **exact**
under the admissible (preferred) semantics and identity casting functions.
(Existence-only; precise construction is left open.)

### Theorem 4.8.7 *(p.356)*

There does **not** exist a full translation from `Fr^{AFN}` to `Fr^{AF}` that
is exact under the **stable** semantics and identity casting functions.
(Argued from Example 126 — AFN stable extensions need not be tight.)

### Theorem 4.8.8 *(p.359)*

Let SF^{FN} be obtained from FN through Translation 63. Then SF^{FN} **might
not** be in minimal normal form, even if FN is.

### Theorem 4.8.9 *(p.359)*

Translation 63 (attack-propagation AFN→SETAF on weakly-valid AFNs):

- E strongly coherent in FN ⇒ E conflict-free in SF^{FN}; not vice versa for
  conflict-free.
- E σ-extension of FN (σ ∈ {admissible, preferred, complete, grounded, stable})
  ⇒ E σ-extension of SF^{FN}.
- E' σ'-extension of SF^{FN} (σ' ∈ {conflict-free, preferred, complete,
  grounded, stable}) ⇒ E' σ'-extension of FN; **not for admissible**.
- E' conflict-free in SF^{FN} ⇒ not necessarily strongly coherent in FN.

### Theorem 4.8.10 *(p.360)*

Let σ ∈ {complete, preferred, grounded, stable} and SC_σ^{Tr} the identity
casting. Translation `awv-Tr_{SETAF}^{AFN}` is **strong and semantics
bijective** under (σ, SC). It is ⊆-weak under strongly coherent — conflict-free
and admissible semantics with identity casting; ⊇-weak under conflict-free with
identity.

### Theorem 4.8.11 *(p.362)*

Let SF^{FN} be the corresponding **defender** SETAF obtained by Translation 65.
By `E_{np} = {a' | there is no coherent set containing a} ∪ {a' | for every
coherent set C for a, ∃ e ∈ E, c ∈ C\{a}, (e,c) ∈ R}` we denote primed
arguments corresponding to a subset of `E^{att}`. The full translation
relations: σ-extension correspondences for σ ∈ {admissible, preferred,
complete, grounded, stable} hold via `E ↔ E ∪ E_{np}`. Conflict-free direction
weak. Not all admissible directions hold both ways under strict admissible;
but pd/aa-σ-extension correspondences become tight under complete/preferred/
grounded/stable. *(p.363)*

### Theorem 4.8.12 *(p.364)*

Translation 66 — variant of Translation 65 for **strongly valid** AFNs only:
`E_{np}` simplification analogous to 4.8.11; correspondences match those of
Translation 65 but `Z_i` defenders need not be full powerful sequences, just
direct supporters; the construction yields a **semi-structural** (not
semantical) translation.

### Theorem 4.8.13 *(p.371)*

Let FN be a **strongly valid and support-binary AFN** and BF^{FN} its
associated BAF obtained through Translation 67. Then BF^{FN} is **support
acyclic**.

### Theorem 4.8.14 *(p.371)*

Translation 67 induces, with `R' = {R^{sec}}` the collection of secondary
attacks in BF^{FN}, the following correspondences for `E ⊆ A`:

- +conflict-free w.r.t. ∅ in BF^{FN} iff conflict-free in FN.
- inverse closed under S in BF^{FN} iff coherent in FN.
- +conflict-free w.r.t. R' and inverse closed under S iff strongly coherent.
- i-admissible w.r.t. (R',R') iff admissible.
- i-preferred w.r.t. (R',R') iff preferred.
- d-complete w.r.t. (R',R') iff complete.
- d-grounded w.r.t. R' iff grounded.
- stable w.r.t. R' iff stable.

### Theorem 4.8.15 *(p.374)*

Translation 68 (AFN→EAS): ES^{FN} is **attack binary**. If FN is weakly /
relation / strongly valid, so is ES^{FN}. If FN is strongly consistent, so is
ES^{FN}. ES^{FN} **need not** be in minimal form even if FN is.

### Theorem 4.8.16 *(p.374)*

Powerful sequence ↔ evidential sequence correspondence under Translation 68
with η as evidence root. Coherent in FN ⇒ S∪{η} self-supporting in ES^{FN};
self-supporting in ES^{FN} ⇒ S'∩A coherent in FN.

### Theorem 4.8.17 *(p.374)*

If `a ∈ A` is defended by a coherent set S in FN, then a is acceptable w.r.t.
S∪{η} in ES^{FN}; conversely, if a is acceptable w.r.t. a self-supporting set
S' in ES^{FN}, then a is defended by S'∩A in FN.

### Theorem 4.8.18 *(p.375)*

(Strongly) coherent in FN iff (strongly) self-supporting in ES^{FN} after
adding η. σ-extensions for σ ∈ {conflict-free, admissible, complete, preferred,
grounded, stable} correspond bijectively after η-adjustment.

### Lemma 4.8.20 *(p.380)*

For a given powerful sequence for an argument `e ∈ E` we can construct a
corresponding acyclic pd-evaluation and vice versa. E is coherent in FN iff
it is pd-acyclic in D^{FN}.

### Lemma 4.8.21 *(p.380)*

E ⊆ A is strongly coherent in FN iff it is a pd-acyclic conflict-free
extension of D^{FN}.

### Lemma 4.8.22 *(p.380)*

If E is strongly coherent in FN (and thus pd-acyclic conflict-free in D^{FN}),
then `E^{att}` coincides with the acyclic discarded set of E.

### Theorem 4.8.23 *(p.380)*

If E is strongly coherent in FN (and thus pd-acyclic conflict-free in D^{FN}),
then E defends an argument `a ∈ A` in FN iff a is decisively in w.r.t. `v_E^a`
in D^{FN}.

### Theorem 4.8.24 *(p.381)*

Big mapping theorem for Translation 69 (strongly consistent AFN → ADF):

- Coherent in FN iff pd-acyclic in D^{FN}.
- Strongly coherent iff pd-acyclic conflict-free.
- σ-extension for σ ∈ {admissible, complete, preferred} iff aa-σ-extension.
- Stable in FN iff stable in D^{FN}.
- Grounded in FN iff acyclic grounded in D^{FN}.

### Theorem 4.8.25 *(p.384)*

For Translation 70 chained with bypass-form Translation 13: with `E^b` the
bypass arguments generated by E in A', the full set of FN→ADF correspondences
maps coherent / strongly coherent / σ-extensions (admissible/complete/
preferred) / grounded / stable through `E ∪ E^b ↔ aa-σ`.

### Theorem 4.8.26 *(p.384)*

Translation 70 chained with self-attacker-form Translation 15: similar
correspondences, except every stable extension E of FN is stable in D^{FN}
**but not vice versa** (asymmetric).

### Theorem 4.9.1 *(p.388)*

Let ES be an EAS and `F^{ES}` its corresponding AF obtained by Translation 71
(EAS→AF coalition). If S ⊆ A is strongly self-supporting in ES, then there
exists a conflict-free extension S' ⊆ A' of `F^{ES}` s.t. S = ⋃S'. Same
conclusion (with σ-extensions) for σ ∈ {admissible, complete, preferred,
grounded, stable} (footnote 17 noting that complete is defined formally only
in PO14b but result still holds). Not every conflict-free extension of `F^{ES}`
is strongly self-supporting in ES (one-way only for conflict-free).

### Theorem 4.9.2 *(p.390)*

There exists **no** full translation from `Fr^{EAS}` to `Fr^{AF}` that is
exact under conflict-free, admissible, complete, preferred and stable
semantics with identity casting functions. (Proved by adapting the §4.4.1.3
construction via Translation 30 — EAS group attack cannot be reproduced by
AF binary attack.)

### Theorem 4.9.3 *(p.391)*

EAS→SETAF coalition (Translation 72): If S ⊆ A is conflict-free in ES, there
**might not** exist a conflict-free extension S' of SF^{ES} s.t. S = ⋃S'.
If S is strongly self-supporting in ES, such S' does exist. σ-extensions of
ES (σ ∈ {admissible, complete, preferred, grounded, stable}) lift to SF^{ES}
σ-extensions; reverse lift `S = ⋃S'` is σ'-extension of ES for the same σ'.
Conflict-free reverse direction: ⋃S' is strongly self-supporting (and thus
conflict-free) in ES.

### Theorem 4.9.4 *(p.393)*

For Translation 73 (attack propagation EAS→SETAF on weakly valid form): If
S ⊆ A is strongly self-supporting in ES, then it is conflict-free in SF^{ES}.
Does not necessarily hold for conflict-free direction. σ-extension of ES
implies σ-extension of SF^{ES} (σ ∈ {admissible, complete, preferred,
grounded, stable}). Reverse direction holds for σ' ∈ {conflict-free,
complete, preferred, grounded, stable}, but not necessarily admissible. Not
every conflict-free extension of SF^{ES} is strongly self-supporting in ES.

### Theorem 4.9.5 *(p.394)*

Translation 74 (`awv-Tr_{SETAF}^{EAS}`): strong and semantics bijective under
σ ∈ {complete, preferred, grounded, stable} with identity casting. ⊆-weak
under strongly self-supporting–conflict-free and admissible. ⊇-weak under
conflict-free.

### Theorem 4.9.6 *(p.396)*

Translation 75 (defender EAS→SETAF) lifts σ-extensions through
`E ↔ E ∪ S_{np}`. σ-extension correspondences for admissible/preferred/
complete/grounded/stable. Conflict-free direction one-way only; reverse
extension intersection might not be strongly self-supporting.

### Theorem 4.9.7 *(p.398)*

For strongly valid EAS, Translation 76 (`sdef-Tr_{SETAF}^{EAS}`) — defender
with simplified `S_{np}` (only direct supporters needed because every
argument has an evidential sequence) — preserves σ-extensions for σ ∈
{admissible, preferred, complete, grounded, stable}. Same one-way
conflict-free direction; reverse intersection might not be strongly self-
supporting.

## Examples Worked

### Example 125 *(p.341)*

The familiar AFN `FN = ({a,b,c,d,e,f}, {(a,e), (d,b), (e,c), (f,d)},
{({b,c},a), ({f},f)})`, depicted in Figure 4.65a. Coalition AF construction
gives `F^{FN}` with set-arguments {a,b}, {a,c}, {b}, {c}, {d}, {e}; the
argument f drops out (no coherent set). Walks through cf, admissible, complete
(now 1-1: {{d}}, {{d},{e}}, {{c},{d},{a,c}}), preferred and stable extensions,
illustrating Theorems 4.8.1/4.8.2.

### Example 126 *(p.356)*

AFN `FN = ({a,b,c,d,e}, {(a,c), (c,a), (b,d), (d,b), (c,b), (b,c)}, {({c,d},
e)})` of Figure 4.66 — chosen to demonstrate AFN stable extensions are not
tight: stable = {{a,b}, {b,c,e}, {a,d,e}} but conflict-free set {a,b,e} is
absent yet conflict-free; this proves Theorem 4.8.7 (no full exact AFN→AF for
stable semantics).

### Example 127 *(p.358)*

`FN_1 = ({a,b,c,d,e,f}, {(d,b), (e,c), (a,f)}, {({b,c},a)})`. Demonstrates
that direct attack propagation from coherent sets to source arguments
preserves semantics for binary attacks but breaks down when an invalid
argument (`b` becomes self-supporter in `FN_2`) introduces support cycles —
attack propagation must use **group attacks** because indirect attacks
target sets, not single arguments.

### Example 128 *(p.360)*

Trivial AFN ({a,b}, {(b,a)}, {({b},b)}). Shows that **without** removing
invalid argument `b`, allowing every argument to attack it (including
self-attack), the produced SETAF's admissible/preferred extensions are
preserved but **complete and grounded are not** — both ∅ and {a} become
complete in SETAF, only {a} is complete in FN. Justification for using
weakly-valid normal form before the propagation translation.

### Example 129 *(p.361)*

Minimal AFN ({a,b,c,d,e}, {(a,b),(c,d)}, {({b},e),({d},e)}) — three coherent
sets for e: {b,d,e} (minimal), {a,b,d,e}, {b,c,d,e}; all attacked by sets
{a},{b},{a,b}; demonstrates that the SETAF target may **lack minimality**
even when source AFN is minimal.

### Example 130 *(p.362)*

The standard 6-argument `FN` (a,e),(d,b),(e,c),(f,d) revisited; shows the
attack-propagation SETAF `SF^{FN} = ({a,b,c,d,e}, {({a},e), ({d},b), ({e},c),
({d,e},a)})` has cf-extensions ∅,{a},{b},{c},{d},{e},{a,b},{a,c},{a,d},
{b,c},{b,e},{c,d},{d,e},{a,b,c},{a,c,d}; only some are strongly coherent in
FN (e.g. {a},{a,d} are SETAF-cf but not FN-strongly-coherent). At complete
semantics correspondence becomes exact.

### Example 131 *(p.361)*

Simple AFN ({a,b,c}, ∅, {({a,b},c), ({a,c},b)}) demonstrating defender SETAF
construction with auxiliary `b'`,`c'` arguments. Key observation: after
translation, set {b,c} is admissible in SETAF even though it is **not even
coherent** in FN — motivates the additional `Z' ⊆ ⋃Z_i` construction in
Translation 65.

### Example 132 *(p.353)*

7-argument AFN with `({b,c},a), ({f},f), ({g},b)` showing minimal coherent
sets for each argument and that complete/preferred/stable extensions of the
defender SETAF `SF^{FN}` are in 1-1 correspondence with those of FN, while
admissible has multiple SETAF preimages per source set.

### Example 133 *(p.355)*

({a,b,c,d}, ∅, {({b,c},a), ({d},a)}) → defender EAF construction with `u_1,
u_2` "you are unsupported" arguments override conflicts; admissibility of a
requires {b,d} or {c,d} so `u_1, u_2` are defeated. Discusses pitfall of
treating supporters as conflicts vs. defenders.

### Example 134 *(p.356)*

({a,b,c}, ∅, {({b},a), ({c},a)}) → second-approach EAF where supports become
attacks and supporters defense-attack the supports. Adding `(d,b)` breaks the
construction unless we also add `(d,a)` to propagate the indirect rejection
of a; works only on **binary support, strongly valid** AFNs.

### Example 135 *(p.362)*

Same 7-argument AFN as Example 132, now translated to EAS. Trivial evidential
sequences for c, d, e, g; b has (η,g,b); a has both (η,g,b,a) and (η,c,a).
f has none. Self-supporting sets enumerated; admissible extensions match
AFN's after η is removed.

### Example 136 *(p.367)*

The 6-argument AFN translated to ADF via Translation 69. ADF acceptance
conditions: `C_a = b∨c, C_b = ¬d, C_c = ¬e, C_d = ¬f, C_e = ¬a, C_f = f`.
Walks through pd-acyclic conflict-free extensions E_1=∅ through E_13={a,c,d}
and identifies grounded {d}, preferred and stable {d,e} and {a,c,d}; argument
b excluded from aa-admissible because d not in discarded set; e excluded
because of c. Confirms aa-admissible extensions of D^{FN} = admissible of FN.

### Example 137 *(p.373)*

({a,b,c,η}, {({a,b},c)}, {({η},a), ({η},b), ({η},c)}) — simple EAS with
self-supporting sets ∅,{η},{η,a},{η,b},{η,c},{η,a,b},{η,a,c},{η,b,c},
{η,a,b,c}; shows that with only minimal-per-argument sets {η},{η,a},{η,b},
{η,c} the coalition AF becomes attack-free and {{η,c}} would emerge as
admissible — motivates using all (not minimal) self-supporting sets.

### Example 138 *(p.375)*

5-argument EAS ({η,a,b,c,d,e}, {({a,b},d), ({d},c), ({e},d)}, {({η},a),
({η},c), ({η},d), ({c},b)}) — coalition AF has 13 self-supporting set-args
s_1...s_13, 96 admissible extensions; only 3 are complete: {s_1,s_2,s_3},
{s_1,s_2,s_3,s_5,s_7}, {s_1,s_2,s_3,s_4,s_6,s_8,s_10}. Demonstrates
complexity of coalition AF.

### Example 139 *(p.378)*

Same EAS as Example 138 — coalition SETAF construction (Translation 72) is
**much simpler**, with arguments s_2,s_3,s_4,s_5,s_8 only and attacks
({s_3,s_8},s_5), ({a,b},d) becoming a single SETAF attack.

### Example 140 *(p.394)*

EAS ({η,a,b,c,d,e,f}, {({a,b},d),({a},f),({d},c),({e},d),({f},a)}, {({η},a),
({η},c),({η},d),({η},f),({a},b),({c},b)}) — admissible extensions: ∅, {η},
{η,a}, {η,f}, {η,a,b}, {η,d,f}, {η,a,b,c}; minimal attack-propagation SETAF
form is `({a,b,c,d,f}, {({f},a),({d,f},b),({d},c),({a,b},d),({a},f)})` with

### Example 141 *(p.397)*

Defender SETAF construction for the EAS of Example 140: introduces `a',b',
c',d',e',f'` for every non-η argument; adds attacks {({η},a'),({η},c'),
({η},d'),({η},f'),({η,a},b'),({η,c},b')}. SETAF admissible extensions enumerated

## Translations (Equations / Formulas)

### Translation 61 — AFN→AF coalition *(p.350)*

For every `a ∈ A`, add minimal coherent set `C ⊆ A` containing `a` to `A'`;
for any `C_1,C_2 ∈ A'`, `C_1 R' C_2` iff `∃x ∈ C_1, y ∈ C_2 s.t. xRy`.

### Translation 62 — AFN conflict-corresponding AF *(p.351)*

`F_{cf}^{FN} = (A,R)` — drop the support relation entirely. Specialized,
basic, structural, modular; classified as basic translation.

### Translation 63 — AFN attack-propagated SETAF (weakly-valid form) *(pp.358–359)*

For each `a ∈ A'` let `{X_1^a, ..., X_n^a}` be all coherent sets containing
a; let `Z_i^a = {b ∈ A' | ∃ c ∈ X_i^a, (b,c) ∈ R}`. Define
$$R'' = \{(Z', a) \mid a \in A', Z' \subseteq \bigcup_{i=1}^n Z_i^a, \forall_{i=1}^n Z' \cap Z_i^a \neq \emptyset\}.$$

### Translation 64 — strongly valid AFN to SETAF *(p.360)*

Same shape as 63 but applied directly without weakly-valid normal form (since
strongly valid AFNs are already validity-clean).

### Translation 65 — defender AFN→SETAF *(p.362)*

`A' = A ∪ {a' | a ∈ A ∧ ∃C ⊆ A, CNa}` where a' means "a is not powerful";
`R' = {({a},b) | (a,b) ∈ R} ∪ {({a'},a) | a ∈ A ∧ ∃C ⊆ A, CNa}`; for each
such a, for any minimal powerful sequence `(a_0,...,a_n)` for a, add
`({a_0,...,a_{n-1}}, a')` to R'.

### Translation 66 — strongly valid defender AFN→SETAF *(p.366)*

Same as 65 but for any `a ∈ A` with `∃C ⊆ A, CNa`, let `{Z_1,...,Z_n}` be
all sets s.t. `Z_iNa`; for all minimal `Z' ⊆ ⋃ Z_i` s.t. ∀ Z'∩Z_i ≠ ∅, add
(Z',a') to R'. (No need to traverse full powerful sequences.)

### Translation 67 — AFN→BAF (strongly valid + support binary) *(p.371)*

`BF^{FN} = (A,R,S)` where `S = {(a,b) | ({a},b) ∈ N}`.

### Translation 68 — AFN→EAS *(p.373)*

`A' = A ∪ {η}`; `R' = {({a},b) | (a,b) ∈ R}`; for each `a ∈ A` with
`Z = {Z_1,...,Z_n}` all sets `Z_i s.t. Z_iNa`: if Z is empty, add `({η},a)`
to E; otherwise for every `Z' ∈ Z_1 × ... × Z_n`, add `(Z'_S, a)` to E
where `Z'_S` is the set of all elements in Z' (continued formulation
*(p.374)* fixes redundancy).

### Translation 69 — strongly consistent AFN→ADF *(p.378)*

For every two arguments `a,b`, if `aRb` or `∃Z ⊆ A` containing a s.t. `ZNb`,
add `(a,b)` to L. Functional acceptance condition: every `P' ⊆ par(a)` is
mapped to `out` iff `∃p ∈ P' s.t. pRa` or `∃Z ⊆ par(a) s.t. ZNa and Z∩P' = ∅`;
otherwise `in`. Propositional acceptance:
$$C_a = att_a \wedge sup_a$$
where $att_a = \neg t_1 \wedge ... \wedge \neg t_n$ for all $t_i$ s.t. $t_iRa$,
and $sup_a = (\bigvee Z_1) \wedge ... \wedge (\bigvee Z_m)$ for all $Z_i$
s.t. $Z_iNa$. Initial arguments get `C_a = ⊤`.

### Translation 70 — general AFN→ADF *(p.382)*

1. Obtain strongly consistent `FN^{sc}` from FN by Translation 13 (bypass) or
   Translation 15 (self-attacker).
2. `D^{FN}` = result of feeding `FN^{sc}` to Translation 69.

### Translation 71 — EAS→AF coalition *(p.387)*

For every nonempty self-supporting set `S ⊆ A`, add S as an argument in A';
for every `(X,y) ∈ R`, add `(X',Y')` to R' where X',Y' ∈ A', X ⊆ X' and
y ∈ Y'.

### Translation 72 — EAS→SETAF coalition *(p.391)*

For every minimal self-supporting set `S ⊆ A` for some `a ∈ A`, add S as an
argument in A'; for every `(X,y) ∈ R`, add `(X',Y')` to R' where X',Y' ∈ A',
X ⊆ ⋃X', ∀V ∈ X', X∩V ≠ ∅, and y ∈ Y'.

### Translation 73 — EAS→SETAF attack propagation (weakly-valid form) *(p.393)*

For `a ∈ A'`, let `{X_1^a,...,X_n^a}` be all self-supporting sets s.t.
a ∈ X_i^a, `Z_i^a = {Z_{i,1}^a,...,Z_{i,n_i}^a}` all sets attacking X_i in R'.
$$R'' = \{(\bigcup Z', a) \mid a \in A', Z' \subseteq \bigcup_{i=1}^n Z_i^a, \forall_{i=1}^n Z' \cap Z_i^a \neq \emptyset\}.$$
Excludes η since `{η}` self-supports trivially.

### Translation 74 — EAS→SETAF, weakly-valid form intermediate *(p.394)*

Wraps Translation 73 around the weakly-valid form transform `wv-Tr^{EAS}`.

### Translation 75 — defender EAS→SETAF *(p.395)*

`A' = A ∪ {a' | a ∈ A\{η}}` — meaning `a' = "a is not e-supported"`;
`R' = R ∪ {({a'},a) | a ∈ A\{η}}`; for any minimal evidential sequence
`(a_0,...,a_n)` for `a ∈ A\{η}`, add `({a_0,...,a_{n-1}},a')` to R'.

### Translation 76 — strongly-valid EAS defender *(p.398)*

`A' = A ∪ {a' | a ∈ A\{η}}`; `R' = R ∪ {({a'},a) | a ∈ A\{η}}`; for every
set `C ⊆ A` s.t. `CEa`, add `(C,a')` to R'. Simplifies Translation 75
because strong validity guarantees every argument has an evidential
sequence — no need to enumerate the full sequence.

## Algorithms / Procedures

### Algorithm — Translation 61 (AFN→AF coalition) *(p.350)*

1. For each argument `a ∈ A`, compute every minimal coherent set C containing
   a; add C as a vertex in A'.
2. For every pair `C_1, C_2 ∈ A'` add edge `(C_1,C_2)` to R' iff `∃x ∈ C_1,
   y ∈ C_2 s.t. xRy`.

### Algorithm — Translation 65 (defender AFN→SETAF) *(p.362)*

1. Initialize `A' := A`.
2. For each `a ∈ A` with at least one supporting set `C s.t. CNa`, add `a'`
   to A' (meaning "a is not powerful").
3. R' starts as the singleton lift `{({x},y) | (x,y) ∈ R}`.
4. For each such a, add `({a'},a)` to R'.
5. For each minimal powerful sequence `(a_0,...,a_n)` for a, add the group
   attack `({a_0,...,a_{n-1}},a')` to R'.

### Algorithm — Translation 69 (strongly consistent AFN→ADF) *(p.378)*

1. For each pair `a,b ∈ A`: if `aRb` or there exists `Z ⊆ A` containing a
   s.t. `ZNb`, add `(a,b)` to L (the link relation).
2. For each argument `a ∈ A` not initial:
   - Build attack part `att_a = ⋀_{tRa} ¬t`.
   - Build support part `sup_a = ⋀_{ZNa}(⋁ Z)`.
   - Set propositional acceptance condition `C_a = att_a ∧ sup_a`.
3. For initial arguments set `C_a := ⊤`.

### Algorithm — Translation 70 (general AFN→ADF) *(p.382)*

1. Obtain `FN^{sc} = (A',R,N')` from FN via Translation 13 (bypass) or
   Translation 15 (self-attacker).
2. `D^{FN}` = Translation 69 applied to `FN^{sc}`.

### Algorithm — Translation 75 (defender EAS→SETAF) *(p.395)*

1. `A' := A ∪ {a' | a ∈ A\{η}}` — primed args mean "a is not e-supported".
2. `R' := R ∪ {({a'},a) | a ∈ A\{η}}`.
3. For each `a ∈ A\{η}` and every minimal evidential sequence
   `(a_0,...,a_n)` for a, add `({a_0,...,a_{n-1}},a')` to R'.

## Semantics Defined / Compared

This range does not introduce new ADF/AF semantics — instead it characterizes
how each existing semantics on each source framework maps under each
translation. Highlights from §4.8.6 and the summary table:

- **AFN coherent ↔ ADF pd-acyclic** (under strong consistency).
- **AFN strongly coherent ↔ ADF pd-acyclic conflict-free.**
- **AFN admissible/preferred/complete ↔ ADF aa-admissible/aa-preferred/
  aa-complete** — the aa-family is the ADF analog precisely because AFN
  semantics demand acyclic powerful sequences. *(p.379, p.381, Theorem 4.8.24)*
- **AFN grounded ↔ ADF acyclic grounded.**
- **AFN stable ↔ ADF stable** (under strongly consistent + Translation 69).
- For the bypass-form Translation 70 chained translation, every AFN stable is
  ADF stable; for the self-attacker form chain, only one direction holds —
  ADF stable can be wider than AFN stable (Theorem 4.8.26).
- **EAS conflict-free is irreducibly group** — therefore no EAS→AF translation
  is exact; EAS→SETAF can be exact at complete/preferred/grounded/stable for
  most translation families.

## Comparisons With Prior Work

- *(p.350)* The AFN→AF coalition translation was introduced in [Nou13]; this
  thesis observes that [Nou13] only proved results for the "more advanced
  semantics" and didn't address bijectivity. Polberg complements with cf,
  strongly-coherent, admissible analyses.
- *(p.354)* [Nou13] claims AFNs can express more than AFs and that no exact
  translation exists. Polberg pushes back: their counter-examples were
  trivial sets and trivially realizable in AFs under complete/preferred/
  stable, so the claim is **not actually supported**. Theorem 4.8.6 then
  shows a full exact AFN→AF translation for admissible/preferred **does**
  exist.
- *(p.355)* AFN-admissibility is shown to satisfy the same union-closure
  property as AF-admissibility (Theorem 4.8.4 corresponds to Definitions
  2.3.2/2.3.3); contrasts with BAFs whose semantics are "distinctively
  different".
- *(p.357)* AFN attack-propagation explicitly avoids reusing the BAF
  construction (Translation 53) because BAF necessary support assumes every
  argument is valid and has no support cycles — neither holds for AFNs.
- *(p.357)* The AFN defender translation must use Translation 58 for BAFs as
  inspiration but adapt it because AFN supports allow cycles BAFs don't.
- *(p.371)* AFN→BAF is restricted to strongly-valid + support-binary AFNs
  because group support cannot be encoded as BAF secondary attacks
  (which are binary).
- *(p.372)* Polberg notes that EAS sufficient support is naturally CNF (`⋀
  ⋁ ...`) while AFN sufficient support is naturally DNF (`⋁ ⋀ ...`) — these
  are duals and the translation between them is "basically a CNF↔DNF
  conversion".
- *(p.385)* Coalition AF translation from EASs (Translation 71) is a
  reformulation of [ORL10] with the [PO14b] correction that ∅ is also self-
  supporting. Polberg further proves an impossibility — no full exact
  EAS→AF translation can exist (Theorem 4.9.2), strengthening the [ORL10]
  result.
- *(p.379)* The AFN→ADF translation must use BADF as its codomain (not the
  full ADF), because not every EAF-style ADF can be produced by an AFN —
  open problem to characterize a tighter subclass.

## Design Rationale

- **Why three AFN→SETAF translations** *(p.342–367)*: Each captures a
  different "aspect" of supports: coalition (set-valued args), attack
  propagation (turn supports into induced indirect attacks), and defender
  (turn supports into supporter→auxiliary defense). Polberg keeps all three
  to expose the trade-offs explicitly.
- **Why prefer "validity removal" over "self-attacker on invalid"** *(p.345)*:
  The self-attacker approach for invalid arguments preserves admissible
  but not complete or grounded; weakly-valid normal form is cleaner and
  preserves more semantics.
- **Why only minimal powerful sequences in defender Translation 65**
  *(p.350)*: Smaller defender attack sets retain stronger admissible
  correspondence; using all powerful sequences inflates the SETAF without
  semantic gain.
- **Why two-step structure of Translation 64 / 70** *(p.348, p.382)*: First
  apply weakly-valid (or strongly-consistent) normal form, then the simple
  translation. Modularity through composition with established normal-form
  translations.
- **Why the "primed-only-for-supported" rule for AFN defender** *(p.395)* but
  "primed-for-every-non-η" rule for EAS defender: in AFN, an argument with no
  support requirement is valid by default; in EAS, only η is valid by
  default — every other argument needs an evidential sequence and so a
  primed mirror.
- **Why ADF→AFN does not justify an AFN→AFRA translation** *(p.384)*:
  Recursive attack does not naturally cast into positive support; abstract
  support is the more natural target. Polberg therefore proposes chained
  AFN→ADF→AFRA via existing translations rather than a direct construction.
- **Why aa-family of ADF semantics is the right correspondent for AFN
  coherence** *(p.379)*: Coherence is acyclic powerful sequence membership;
  this matches pd-acyclic decisive interpretations exactly.
- **Why allow ⊕-non-modularity in AFN→ADF** *(p.381)*: The cost of forcing
  ⊕-modularity is a more invasive translation; ⊗-modularity is preserved
  which is what matters for argument-by-argument condition synthesis.

## Limitations

- *(p.355)* Stable AFN→AF: no exact full translation exists (Theorem 4.8.7).
- *(p.356)* "we are not entirely sure how" the appropriate AFN→AF
  translation under admissible/preferred looks like — Theorem 4.8.6 gives
  existence only, no construction. **Open problem.**
- *(p.359)* Translation 63 produces SETAFs that **may not be in minimal
  normal form** even when source is. Theorem 4.8.8.
- *(p.361)* The defender SETAF approach **cannot be made modular** in
  general — illustrated by trivial AFN union counter-example.
- *(p.366)* The "support relation removing" classification of Translation 64
  is not tight; the translation is structurally close to support-preserving
  but technically alters set structure. Polberg leaves the precise
  classification open.
- *(p.367)* No AFN→EAF formal translation provided — only sketched in two
  ways, both rejected as inferior to the AF-bypass route.
- *(p.371)* AFN→BAF only handles **support-binary, strongly-valid** AFNs;
  general AFNs require chaining via SETAF.
- *(p.374)* Translation 68 (AFN→EAS): produces **non-minimal** EAS even when
  source AFN is minimal (Theorem 4.8.15).
- *(p.376)* Even after the "Z'_S" fix, Translation 68 is **not modular** —
  union of two AFNs maps differently from union of their EAS images.
- *(p.379)* AFN→ADF: BADF subclass codomain is not tight. "We leave
  describing a more fitting subclass for future work."
- *(p.381)* Translation 70 with self-attacker form is asymmetric for stable
  semantics (only AFN→ADF, not the reverse).
- *(p.384)* AFN→ADF stable correspondence is missing; Polberg "believes" a
  direct exact stable translation is possible by chaining AFN→SETAF→ADF
  — open future work.
- *(p.385)* "Although our results are quite interesting, the analysis in
  Section 4.8.1.1 shows that there are still possibilities for improvement.
  In particular, we know that it is possible to create an exact translation
  from AFNs to AFs under the admissible and preferred semantics. We suspect
  it to be a combination of attack propagation and defender approaches;
  nevertheless, we are yet to create a satisfactory method."
- *(p.385)* No exact full EAS→AF translation exists (Theorem 4.9.2);
  fundamental cardinality mismatch between binary and group attacks.
- *(p.391)* The EAS→SETAF coalition (Translation 72) **fails on conflict-
  free** in one direction (Theorem 4.9.3) — the only translation in the
  family that doesn't preserve cf in both directions.
- *(p.397)* No modular EAS→SETAF translation is known. "Although we are not
  yet sure how a modular translation can be constructed, we can still
  propose certain improvements that bring us closer to a structural
  approach."

## Implementation Notes

- **Group attack as boolean formula.** Polberg's Translation 69 turns AFN
  group support into a propositional acceptance condition `C_a = att_a ∧
  sup_a` where `sup_a = ⋀_i ⋁ Z_i` (CNF). This is exactly the shape that
  pyboolector / Z3 solvers expect, so propstore's CEL/Z3 layer can carry
  AFN→ADF directly *(p.378)*.
- **CNF↔DNF duality between EAS and AFN sufficient support** *(p.372)* tells
  us that propstore's existing AFN-style support `aspic.py` rules can be
  re-cast into EAS form by a syntactic conversion — relevant if we ever
  want to ingest EAS-shaped knowledge.
- **Defender translation pattern as implementation receipt.** The "I am not
  e-supported" auxiliary argument trick from Translations 65/75 is a
  generic mechanism for turning support discipline into negation in attack
  graphs; might inform how we encode "missing evidence" claims in the ATMS
  layer (`propstore.world`).
- **Strong consistency precondition for ADF translation (Translation 69).**
  We need to detect strongly-consistent vs general AFNs before we can apply
  the direct ADF mapping — otherwise we fall back to the bypass / self-
  attacker normal-form chain (Translation 70).
- **"E ∪ E^b" mapping rule** *(p.383)*: when running the bypass-form
  Translation 70, the ADF extensions include "bypass arguments generated by
  E in A'" — the implementation must track which arguments were introduced
  by the bypass step and remove them when casting back to AFN.
- **Casting function discipline.** Almost every theorem in this range
  conditions on a particular `SC_σ^{Tr}` (union, identity, removal). When
  propstore's argumentation engine reports "extension preserved", it must
  state which casting function it assumed — there is no semantics-free
  "preserved".
- **Failure modes worth catching.** AFN→AF stable: no exact full translation
  (Theorem 4.8.7). EAS→AF: no exact full translation under any of the listed
  semantics (Theorem 4.9.2). Code that claims to ground EAS into AF must
  either reject these or document the loss explicitly.
- **Non-modularity warning.** Translations 64, 65, 67 (full BAF mapping
  excepted), 68, 69, 73, 75, 76 are all marked **not modular**. propstore
  must not assume composability of AFN/EAS imports — joining two imported
  sub-frameworks and re-translating is *not* the same as translating each
  and joining. (Modular translations: 62, 67, 76 final defender variant.)
- **Table 4.9 *(p.385)* is the canonical correctness scorecard** for AFN
  outputs — propstore should mirror it as a runtime classification when
  reporting cross-framework conversions.
