## Key Findings

### Thesis identity & framing
- Polberg PhD thesis, TU Wien, 1 April 2017, advisor Stefan Woltran, external reviewers Leon van der Torre & Sanjay Modgil. Funded FWF I1102, FWF I2843, EPSRC Computational Persuasion. *(p.0,5)*
- Stated four contributions *(p.18)*: (1) new extension-based ADF semantics + preliminary complexity, (2) creation/correction/classification of ~90 inter-framework translations, (3) framework normal forms, (4) ADFs as a unifying super-framework.
- Frameworks studied: AF, SETAF, AFRA, EAF, EAFC, BAF, AFN, EAS, ADF *(p.8)*.
- Polberg uses framework-type variable `T ∈ {AF, SETAF, AFRA, EAF, EAFC, BAF, AFN, EAS, ADF}` *(p.20)*.

### Chapter 1 critique themes
- ADF original BW10 extension semantics: limited to bipolar subclass, "did not always produce desired results." *(p.16)*
- Labeling-based ADF semantics (BES+13/Str13a): drawbacks — opacity of Dung notions, self-influence under cycles, info-vs-subset maximality mismatch under support. *(p.16)*
- Graph-based generalizations of Dung get unwieldy (NP07, CLS13, Nou13, Mod09a, BCGG11); ADFs replace this with arbitrary Boolean acceptance conditions. *(p.15)*
- Translations valued for: (a) reusing AF/ADF solvers (EGW08, ES14), (b) measuring expressive power, (c) automatic best-tool selection. *(p.17)*

### Background framing (Ch. 2 opener)
- Universal argument domain `U`. `Fr_U^T` = collection of all frameworks of type `T` whose argument set is a subset of `U`. Focus on finite-argument frameworks (results often extend to infinite). *(p.21)*
- Semantics domain `U_σ` (general) and `U_σ^Fr` (framework-restricted, e.g., `2^A` for extension semantics). Set of subsets actually realisable as `σ(Fr)` forms the **semantics signature** (§2.3 / DDLW15, Dyr14). *(p.21)*
- Polberg restricts to extension- and 3-valued labeling-based semantics; rankings/4-valued labelings (BCG11, JV99b, ABN13) are not pursued. *(p.20)*

## Definitions Introduced

### §2.1.1 Dung AF *(printed pp.8-12)*

**Definition 2.1.1** *(p.22)* — A **Dung's abstract argumentation framework** (AF) is `F = (A, R)` with `A` arguments and `R ⊆ A × A` an **attack** relation.

**Definition 2.1.2** *(p.22)* — Defense and conflict-free / admissible / preferred / complete / stable. Abbreviations `cf, adm, pref, comp, stb`.

**Definition 2.1.3** *(p.22)* — Range. `E^+ = {a | ∃e ∈ E, (e,a) ∈ R}`; `E^- = {a | ∃e ∈ E, (a,e) ∈ R}`; `E^Ran = E^+ ∪ E`.

**Definition 2.1.4** *(p.22)* — **semi-stable**: range-maximal complete extension.

**Definition 2.1.5** *(p.23)* — Characteristic function `F_F : 2^A → 2^A`, `F_F(E) = {a | a defended by E in F}`. Grounded = least fixed point.

**Definition 2.1.12** *(p.24)* — `F` is **well-founded** iff no infinite sequence `a_0, a_1, ...` with each `a_{i+1}` attacking `a_i`.

**Definition 2.1.14** *(p.25)* — Three-valued labeling `Lab : A → {in, out, undec}`; legally in / legally out / legally undec defined w.r.t. attackers.

**Definition 2.1.15** *(p.25)* — Conflict-free / admissible / complete / preferred / grounded / stable labelings (preferred ↔ max in; grounded ↔ min in; stable ↔ undec=∅). Prefix `lab-`.

### §2.1.2 SETAF *(printed pp.13-15)*

**Definition 2.1.19** *(p.27)* — A **SETAF (CAF)** is `SF = (A, R)` with `R ⊆ (2^A \ ∅) × A`.

**Definition 2.1.20** *(p.27)* — Defense in SETAF and standard semantics (cf, adm, pref, comp, stb).

**Definition 2.1.21** *(p.27)* — SETAF characteristic function and grounded extension.

**Definition 2.1.26** *(p.28)* — Discarded set / range in SETAF.

### §2.1.3 AFRA *(printed pp.16-17)*

**Definition 2.1.28** *(p.29)* — **AFRA**: `(A, R)` with attacks `(a, X)` where `X ∈ A ∪ R` (recursive attack target).

**Definition 2.1.29-2.1.31** *(p.29)* — Direct/indirect/full defeat: `V` directly defeats `W` iff `W = trg(V)`; indirectly iff directly defeats `src(W)`; defeats iff direct or indirect.

**Definition 2.1.32** *(p.29)* — Acceptability w.r.t. `E`: every defeater is itself defeated.

**Definition 2.1.34, 2.1.35** *(pp.29-30)* — AFRA conflict-free / admissible / preferred / complete / stable; `E ⊆ A ∪ R`.

**Definition 2.1.36** *(p.30)* — AFRA characteristic function `F_FR : 2^{A∪R} → 2^{A∪R}`.

**Definition 2.1.37** *(p.30)* — AFRA discarded set.

### §2.1.4 EAF *(printed pp.18-27)*

**Definition 2.1.40** *(p.31)* — **EAF**: `(A, R, D)` with `R ⊆ A×A`, `D ⊆ A × R`. Symmetry restriction: defense attackers of mutually attacking pairs must mutually attack.

**Definition 2.1.41** *(p.32)* — `defeats_E`: `(a,b) ∈ R` and no `c ∈ E` with `(c,(a,b)) ∈ D`. Strictly defeats iff one-sided.

**Definition 2.1.43** *(p.32)* — EAF conflict-free.

**Definition 2.1.45** *(p.33)* — Reinstatement set `R_E` for a defeat — includes a recursive condition over defense attacks `(y', (x,y)) ∈ D`.

**Definition 2.1.46** *(p.33)* — EAF acceptability via reinstatement set.

**Definition 2.1.48** *(p.34)* — EAF admissible / preferred / complete / stable.

**Definition 2.1.49** *(p.34)* — EAF characteristic operator `F_EF : 2^CF → 2^A` (restricted to conflict-free domain — operator non-monotone in general).

**Definition 2.1.50** *(p.34)* — Finitary EAF.

**Definition 2.1.51** *(p.34)* — Iterative grounded extension over finitary EAF.

**Definition 2.1.53** *(p.35)* — EAF discarded set with reinstatement.

**Definition 2.1.56** *(p.35)* — **Hierarchical EAF (HEAF)** with partition `HF_H = (((A_1, R_1), D_1), ..., ((A_j, R_j), D_j), ...)`; bounded hierarchical iff finite with `D_n = ∅`.

**Definition 2.1.58** *(p.35)* — Bounded HEAF grounded extension via least fixed point of `F_HF`.

### §2.1.4.3 EAFC *(printed pp.28-29)*

**Definition 2.1.62** *(p.41)* — **EAFC**: `(A, R, D)` with `R ⊆ A×A`, `D ⊆ (2^A \ ∅) × R` (collective defense attacks). Bounded hierarchical EAFC defined by partition `δ_H = (((A_1, R_1), D_1),...,((A_n, R_n), D_n))` with `D_n = ∅`, `(c, (a,b)) ∈ D_i ⇒ (a,b) ∈ R_i`, `c ⊆ A_{i+1}`.

**Definition 2.1.63** *(p.41)* — EAFC: `defeats_E` (no `c ⊆ A` with `c ⊆ E` and `(c,(a,b)) ∈ D`); reinstatement set with collective defense attacks; **conflict-free** redefined: `E` conflict-free iff for every `a,b ∈ E` with `(a,b) ∈ R`, ∃ `c ⊆ A` s.t. `(c,(a,b)) ∈ D` — i.e., every internal attack is collectively defended (no `defeats_E` inside `E`). MP10's resolution to the EAF symmetry forcing.

### §2.2 Support-Based Frameworks (Intro)

§2.2 introduces support as a positive relation orthogonal to defense; spans abstract (CLS09), deductive (BGvdTV10), necessary (NR11/Nou13), and evidential (ON08/ORL10/PO14b) supports — last two get dedicated frameworks (AFN, EAS) covered later. *(p.43)*

### §2.2.1 BAF *(printed pp.30-36)*

**Definition 2.2.1** *(p.43)* — **BAF**: `(A, R, S)` with `R ⊆ A×A` attack, `S ⊆ A×A` support. (Original CLS09 also assumed `R ∩ S = ∅`; later works drop that.) *(footnote)*

**Definition 2.2.2** *(p.45)* — Indirect attacks parametrised by support sequences:
- **supported attack** `(a,b)`: ∃ `c` with sequence `aS...Sc` and `cRb`.
- **secondary attack** `(a,b)`: ∃ `c` with `cS...Sb` and `aRc`.
- **mediated attack** `(a,b)`: ∃ `c` with sequence `bS...Sc` and `aRc`.
- **extended attack** `(a,b)`: ∃ `c` with `cS...Sa` and `cRb`.
Notations: `R^sup, R^sec, R^med, R^ext`.

**Definition 2.2.3** *(p.45)* — **Super-mediated attack** `(a,b)`: ∃ `c` with `bS...Sc` and `a` direct- or supported-attacks `c`.

**Definition 2.2.4** *(p.46)* — **Tiered indirect attacks** `R^ind`. `R^ind_0 = ∅`; `R^ind_1 = {R^sup_∅, R^sec_∅, R^med_∅, R^ext_∅}`; `R^ind_i = {R^sup_E, ..., R^ext_E | E ⊆ R^ind_{i-1}}` for `i > 1`. Defines layered indirect attacks parametric on which earlier indirect attacks are added to `R`. `R^ind = ⋃ R^ind_i`.

**Definition 2.2.5** *(p.47)* — Defense in BAF parametrised on `R' ⊆ R^ind`: `E` defends `a` iff for every `(b,a) ∈ R ∪ ⋃R'`, ∃ `c ∈ E` with `(c,b) ∈ R ∪ ⋃R'`.

**Definition 2.2.6** *(p.47-48)* — In BAF: `+conflict-free` w.r.t. `R'`: no `a,b ∈ E` with `(a,b) ∈ R ∪ ⋃R'`. `safe`: no `b ∈ A` is both attacked by `E` (in `R ∪ ⋃R'`) and either `b ∈ E` or supported by chain from `E` to `b`. `closed under S`: `bSa, b ∈ E ⇒ a ∈ E`. `inverse closed`: `aSb, b ∈ E ⇒ a ∈ E`.

**Definition 2.2.7** *(p.48)* — BAF semantics families parametrised by `(R', R'')`:
- **d-admissible**: `+conflict-free` w.r.t. `R'`, defends w.r.t. `R''`.
- **s-admissible**: safe w.r.t. `R'`, defends w.r.t. `R''`.
- **c-admissible**: `+conflict-free` w.r.t. `R'`, closed for `S`, defends w.r.t. `R''`.
- **i-admissible**: `+conflict-free` w.r.t. `R'`, inverse closed for `S`, defends w.r.t. `R''`.
- d-/s-/c-/i-**preferred**: maximal w.r.t. each family.
- **stable**: `+conflict-free` w.r.t. `R'` and ∀ `b ∉ E`, `E` attacks `b` in `R ∪ ⋃R'`.

**Definition 2.2.9** *(p.49)* — **d-complete**: d-admissible and contains all arguments defended by `E` w.r.t. `R'`.

**Definition 2.2.10** *(p.49)* — BAF **d-characteristic operator** `d-F_BF : 2^A → 2^A`, `d-F_BF(E) = {a | a defended by E w.r.t. R' in BF}`. Monotone w.r.t. set inclusion. Yields conflict-free-preserving operator on `+conflict-free` sets.

## Algorithms / Procedures

**AF iterative grounded extension** *(Proposition 2.1.7, p.23)*:
1. Put each `a ∈ A` not attacked in `F` into `E`. If none, return `E`.
2. Remove from `F` all (new) arguments in `E` and all arguments attacked by `E` (with adjacent attacks). Continue with Step 1.

**EAF iterative grounded** *(Definition 2.1.51, p.34)*: `F^0 = ∅`, `F^{i+1} = F_EF(F^i)`, grounded = `⋃F^i`. Finitary EAFs only.

**BAF tiered indirect attack construction** *(Definition 2.2.4, p.46)*: Start `R^ind_0 = ∅`; build `R^ind_1` from base support+attack; recurse using subsets of `R^ind_{i-1}` as additional attacks.

## Semantics Defined / Compared

For each of AF, SETAF, AFRA, EAF, EAFC, BAF Polberg recasts conflict-free / admissible / preferred / complete / stable / grounded in the framework's idiom. Parametrisation in BAF makes "admissible" a 4-family (d/s/c/i).

Inclusion patterns:
- AF, SETAF, AFRA: stable ⊆ preferred ⊆ complete; grounded = least complete; complete = semilattice. *(Th 2.1.9-2.1.11; Th 2.1.24-2.1.25; Th 2.1.39)*
- EAF: stable ⊆ complete; preferred ⊆ complete; complete extensions don't form a semilattice; grounded only minimal complete; stable need not be preferred. *(Th 2.1.55, p.34; §2.1.4.2)*
- Bounded HEAF: traditional Dung-style relations recovered. *(Th 2.1.59, 2.1.60)*
- EAFC: parametric collective defense; bounded hierarchical EAFC's `F_EF` is monotone. *(Prop 2.1.64)*
- BAF: only d-admissible/d-complete/d-grounded supports a clean Fundamental Lemma; other admissibility families break; complete/grounded only defined under shared parameter `R'=R''`. *(Lemma 2.2.8; Defs 2.2.9-2.2.10)*

## Comparisons With Prior Work
- *(p.27)* SETAF (NP07) extensions can include sets not realisable in any AF without auxiliary arguments.
- *(p.29)* AFRA (BCGG09, BCGG11) raises attacks to argument level; contrasted with VBVDT11 (attack-based without recursion).
- *(p.31)* EAF lineage: Mod07, BCM09, ML09, MP10, DMBC10, MBC11, Mod14, DGLW15.
- *(p.32)* Defense attacks (EAF) coexist with standard attacks and only target attacks; AFRA recursive attacks replace standard ones with arbitrary depth.
- *(pp.36-40)* EAF design controversies enumerated: non-semilattice complete extensions (Ex 8, BCGG11); grounded only minimal not least; stable that isn't preferred (Ex 9, DMBC10 vs Mod09b); conflict-free forces self-attack on perception-error case (Ex 10); induced symmetric attacks generated only for length-2 cycles (`EF_2` vs `EF_3`).
- *(p.41)* MP10 introduced **collective defense attacks** (EAFC) to fix EAF: instantiation creates appropriate frameworks, redefines conflict-freeness as "no defeats inside", retains rationality postulates (CA07).
- *(p.43)* Support frameworks: abstract (CLS09 — same-strength assumption rejected), deductive (BGvdTV10), necessary (NR11/Nou13), evidential (ON08/ORL10/PO14b). Two main BAF semantic styles:
  - *(p.47)* Original CLS09: defense unchanged from Dung; indirect attacks added to conflict-freeness only.
  - *(p.47)* Newer CLS13/CLS15: define BAF semantics via translation to AF ("attack propagation translation", §4.1.3, §4.7.1).
- *(p.50)* Newer works (CLS13, CLS15) abandon the d/s/c/i admissibility families in favour of translation families. Polberg leaves the BAF defense fix to future work but defines `d-`only complete/grounded.

## Design Rationale
- *(p.21)* Distinct semantics domains for extensions vs labelings vs AFRA's `A ∪ R` are explicit because translations don't preserve the semantic-domain shape.
- *(p.27)* SETAF set-attack chosen for evidence whose force only emerges in combination.
- *(p.29)* AFRA `(a, X)` with `X ∈ A ∪ R` treats attacks as first-class; direct/indirect defeat distinctions follow naturally.
- *(p.32)* EAF symmetry restriction motivated by "is more preferred" reading of defense attacks.
- *(p.34)* EAF characteristic operator restricted to `2^CF` because non-monotone over arbitrary subsets.
- *(p.35)* HEAF subclass introduced to recover monotonicity and Dung-style semantic relations; transformations to other frameworks done only on HEAFs.
- *(p.41)* EAFC's redefinition of conflict-freeness as "no defeats inside" + collective defense attacks lets MP10 instantiation create the right framework rather than enforcing abstract restrictions.
- *(p.43, footnote 8)* CLS09's `R ∩ S = ∅` was relaxed in later works because supporting and attacking the same argument is not nonsensical (BAF design progressively generalises).
- *(pp.46-47)* Polberg's tiered indirect-attack apparatus and parametrisation (`R'`, `R''`) chosen to *organise* rather than replace existing definitions: "Our intent is merely to gather and organize the available approaches."
- *(p.49)* BAF d-complete/d-grounded only defined with shared parameter (`R' = R''`) because the BAF Fundamental Lemma fails otherwise; Polberg explicitly leaves cross-parameter complete semantics to future work.

## Limitations Acknowledged
- *(p.34)* EAF grounded extension not necessarily least complete extension.
- *(p.36)* Translations from EAFs to other frameworks "problematic" due to design controversies; existing transformations (MBC11) only for HEAFs.
- *(p.38)* MP10 observation that not all EAF instantiations succeed in constructing induced attacks.
- *(p.38, p.40)* EAF induced attacks fire only for length-2 cycles; longer defense-attack cycles inconsistent.

## Implementation Notes
- Polberg's `(A, R)` AF, `(A, R ⊆ 2^A × A)` SETAF, `(A, R ⊆ A × (A ∪ R))` AFRA, `(A, R, D)` EAF, `(A, R, D ⊆ 2^A × R)` EAFC, `(A, R, S)` BAF give a clean type hierarchy for propstore's argumentation surfaces.
- The `defeats_E` operator (Definition 2.1.41) makes EAF semantics depend on the candidate set, mirroring propstore's render-time policy.
- EAFC's collective defense attacks (Definition 2.1.62) align with how propstore should treat group-supported overrides (sets of supporting claims jointly cancelling an attack).
- BAF's tiered indirect attacks (Definition 2.2.4) provide a clean recipe for compiling support chains into attack edges — useful for translating support-bearing concepts (concept-merge / co-reference) into the AF substrate.
- The `+conflict-free` / `safe` / `closed` / `inverse closed` quartet (Definition 2.2.6) is a useful design template for propstore claim-set membership policies under support.
- BAF Fundamental Lemma's failure outside `R' = R''` (Lemma 2.2.8 + p.50) is a warning: any propstore feature mixing different "kinds" of indirect attacks needs a defense parameter that exactly matches the conflict-freeness parameter.
- Range `E^Ran = E^+ ∪ E` and `discarded set` notations (Definitions 2.1.3, 2.1.26, 2.1.37, 2.1.53) directly map to "covered" claim status in propstore's render filter.
