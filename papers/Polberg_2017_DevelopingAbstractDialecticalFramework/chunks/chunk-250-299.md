## Key Findings
- Standard AFRA→AF translation (Tr_AF^AFRA, Translation 32 from earlier chunk) is exact under cf/adm/comp/pref/grd/stb but **not modular** because indirect defeats are added; produces a *basic–attack propagation hybrid* *(p.250)*.
- Defender variant (Translation 33) restores modularity but loses exactness for admissible semantics; faithful, classified *basic–defender* *(p.251–254)*.
- Direct AFRA→EAF translation rejected: bypasses through AF or AFN+ADF chain are preferred because the source EAF semantics are non‑monotonic (lemma 2.1.33) and cannot be made to match AFRA admissible extensions cleanly *(p.256–259)*.
- AFRA→BAF (Translation 34) is the **most "interesting"**: full, target‑subclass, injective, generic, semantics‑domain preserving, **exact** under (+/d-)cf, (d-)adm, (d-)comp, (d-)pref, (d-)grd, stb under identity casting and **modular**; the only AFRA→target translation that is both modular and exact under any semantics *(p.260–261)*.
- AFRA→AFN (Translation 35) produces well-structured, elementary, support-binary, support-depth-1 AFNs — i.e. all normal forms hold — but conflict-free and admissible semantics are not preserved one-to-one *(p.262–264)*. Requires `Definition 4.5.5` (source–complete set) to bridge to AFN admissibility.
- AFRA→ADF: no direct translation is offered; the recommended path is AFRA → AF → ADF or AFRA → AFN → ADF; produced ADFs satisfy `redundancy-free`, `cleansed`, `weak`, `relation valid`, `strongly valid`, `BADF`, `AADF^+`. AFRA→AFN→ADF is injective even though AFN→ADF in general isn't *(p.264–265)*.
- AFRA→SETAF and AFRA→EAS only via chained AF/AFN bypass *(p.265)*.
- **Table 4.5** *(p.266)* summarizes AFRA→{AF, BAF, AFN} property matrix; only standard AFRA→AF (Tr 32) is non-modular; all four are full, target-subclass, injective, generic, semi-structural, argument domain altering, and exact except for the defender AF variant (which is faithful instead).
- EAF↔EAFC translation collapses on `BH^EAF ∪ NSym^EAF` (bounded hierarchical or no symmetric attacks) where the two semantics agree; outside that subclass conflict-freeness diverges *(p.267–271)*.
- EAF→AF (Translation 38, [MBC11]-style) restricted to bounded hierarchical EAFs; faithful for complete/preferred/grounded/stable; admissible is not one-to-one (auxiliary primed arguments give freedom); modular, basic-defender hybrid *(p.272–274)*.
- **Impossibility result (4.6.8, 4.6.9):** No full translation Fr^EAF → Fr^AF is exact under conflict-free, complete and stable semantics with identity casting. Reason: EAF cf-extensions are not downward closed; EAF complete extensions need not have a least element; EAF stable extensions can be comparable, breaching AF stable signature *(p.277)*.
- EAF→SETAF (Translation 40) bounded hierarchical only, target = `Min^SETAF`; minimality follows from each attack set having a unique unprimed direct attacker *(p.279–280)*.
- EAFC→SETAF (Translation 41) source-subclass, target-subclass, **overlapping**, weakly argument domain altering, structural; not modular *(p.282–283)*.
- **Impossibility (4.6.13, 4.6.14):** No full EAF→SETAF or EAFC→SETAF that is exact under cf, complete, stable with identity casting *(p.283–284)*.
- EAF→AFRA: **Translation 42** is just `(A, R ∪ D)` (defense attacks become attacks); produces frameworks in `Rec_0^AFRA ∪ Rec_1^AFRA`; preserves σ-extensions one way for cf, adm, complete, grd, pref, stb on bounded hierarchical EAFs but reverse only for complete, grounded, preferred, stable *(p.285–286)*.
- **Impossibility (4.6.16, 4.6.17):** No full EAF→AFRA (or EAFC→AFRA) translation is exact under cf, adm, complete, preferred, stable with identity casting; even from `BH^EAF` *(p.286)*.
- EAF→AFN comes in two flavors:
  - **Translation 43** "defense-as-support": defense-attacked attacks `R_D` are removed; primed copies introduced; original attacker `c` joins `x'` as a support set for the previously-defense-attacked target *(p.289)*. Strongly consistent, minimal, weakly/relation/strongly valid; faithful for comp/pref/grd/stb; ⊆-weak under cf; ⊇-weak under admissible *(p.290–291)*.
  - **Translation 44** chained (EAF→AFRA→AFN): both direct and defense conflicts become arguments; conflict is preserved; produces well-structured-elementary-support-depth-1 AFNs *(p.292–293)*. Distinguished by `c-Tr^EAF_AFN` prefix.
- EAFC→AFN (Translation 45 / Translation 46) generalize 43/44 by enlarging support sets to reflect group defense attacks; basic and basic respectively *(p.295–297)*.
- **Impossibility (4.6.26, 4.6.27):** No full exact EAF→AFN (or EAFC→AFN) translation under cf, complete, stable. Same argument: AFN grounded is least complete; AFN stables are ⊆-incomparable; EAF semantics violate both *(p.298)*.
- §4.6.6 introduces "overpowering support" — defense-attack-as-positive-relation that overrides incoming conflicts via ADF acceptance conditions, exploiting the ADF `ca-semantics` family. ADF translation is the **only EAF/EAFC translation not restricted to bounded hierarchical frameworks** *(p.299)*.

## Definitions Introduced

### Definition 4.5.5 — Source-complete set *(p.263)*
Let `FR = (A, R)` be an AFRA and `E ⊆ A ∪ R`. The **source–complete set** of E is
$$E^{src} = E \cup \{src(V) \mid V \in E \cap R\}.$$
Note: if E^src is conflict-free and admissible, so is E (Lemma 2.1.33). Also if E is complete then `E = E^src`.

## Translations Introduced (verbatim near-form)

### Translation 32 (AFRA→AF, standard) — *recap from earlier chunk; analyzed on p.250*
Mentioned for analysis: full, target–subclass, injective; argument-domain altering, argument-introducing, induced-attack-introducing; generic, semantics-domain preserving, exact; semi-structural; **not modular**. Classified *basic-attack-propagation hybrid* under cf/adm/comp/pref/grd/stb and identity casting *(p.250)*.

### Translation 33 — AFRA→AF, defender (meta-level) *(p.252)*
Let `FR = (A, R)` be an AFRA. The corresponding AF is `F^FR_m = (A', R')` for
- `A' = A ∪ R ∪ X'` where `X' = {x' | x ∈ A}`
- `R' = {(x, x') | x ∈ A} ∪ {(x', a) | a ∈ R, src(a) = x} ∪ {(a, b) | a ∈ R, b ∈ A ∪ R, trg(a) = b}`

Indirect defeats do **not** become direct conflicts; the attack source becomes the defender against `Rej` arguments; conflict-free preservation only one-way.

### Translation 34 — AFRA→BAF *(p.259)*
For AFRA `FR = (A, R)`, BAF `BF^FR = (A', R', S')`:
- `A' = A ∪ R`
- `R' = {(X, Y) | X ∈ R, Y ∈ A ∪ R, Y = trg(X)}`
- `S' = {(X, Y) | Y ∈ R, X ∈ A, X = src(Y)}`

Theorem 4.5.3: Let `R^sec` be the first-tier secondary attacks; `E ⊆ A ∪ R` is cf/stb/d-grd in FR iff +cf/stb/d-grd in BF^FR w.r.t. `R^sec`; for σ ∈ {adm, comp, pref}, iff d-σ-extension w.r.t. `(R^sec, R^sec)` *(p.259)*.

### Translation 35 — AFRA→AFN *(p.262)*
For AFRA `FR = (A, R)`, AFN `FN^FR = (A', R', N')`:
- `A' = A ∪ R`
- `R' = {(X, Y) | X ∈ R, Y ∈ A ∪ R, Y = trg(X)}`
- `N' = {({X}, Y) | Y ∈ R, X ∈ A, X = src(Y)}`

### Translation 36 — EAF→EAFC *(p.269)*
Let `EF = (A, R, D)` be a bounded hierarchical EAF, or an EAF without symmetric direct attacks. The EAFC is `EFC^EF = (A, R, D')` where
$$D' = \{(\{c\}, (a,b)) \mid (c, (a,b)) \in D\}.$$
Properties: source/target-subclass, injective; argument domain and structure preserving; generic, semantics-domain preserving, **exact**; structural; **modular**. Theorem 4.6.1, 4.6.2: σ-extensions coincide for cf, adm, comp, pref, grd, stb *(p.269)*.

### Translation 37 — EAFC→EAF *(p.270)*
For EAFC `EFC = (A, R, D)` bounded hierarchical (or w/o symmetric attacks):
- `A' = A ∪ Gr_D ∪ X'`, `Gr_D = {(a, (b, c)) | (a, (b, c)) ∈ D, |a| > 1}`, `X' = {x' | ∃(a, (b, e)) ∈ Gr_D, x ∈ a}`
- `R' = R ∪ {(x, x') | x ∈ X'} ∪ {(x', (a, (b, c))) | (a, (b, c)) ∈ Gr_D, x ∈ a}`
- `D' = D \ Gr_D ∪ {((a, b), b) | (a, b) ∈ Gr_D, b ∈ R}`

Faithful basic-defender hybrid; ⊆-weak under cf *(p.270–272)*.

### Translation 38 — EAF→AF (defender, [MBC11] style) *(p.273)*
For bounded hierarchical EAF `bh-EF = (A, R, D)`, the AF is `F^EF = (A', R')`:
- `A' = A ∪ R ∪ D ∪ X'`, `X' = {x' | x ∈ A}`
- `R' = {(x, x') | x ∈ A} ∪ {(x', a) | a ∈ R ∪ D, src(a) = x} ∪ {(a, b) | trg(a) = b, and either (a ∈ R ∧ b ∈ A) or (a ∈ D ∧ b ∈ R)}`

### Translation 39 — EAFC→AF *(p.276)*
For bounded hierarchical EAFC, the AF `F^EFC = (A', R')`:
- `A' = A ∪ R ∪ D ∪ X'`, `X' = {x' | x ∈ A}`
- `R' = {(x, x') | x ∈ A} ∪ {(x', a) | a ∈ R, src(a) = x} ∪ {(x', a) | a ∈ D, x ∈ src(a)} ∪ {(a, b) | trg(a) = b, and either (a ∈ R ∧ b ∈ A) or (a ∈ D ∧ b ∈ R)}`

### Translation 40 — EAF→SETAF *(p.279)*
For bounded hierarchical EAF `bh-EF = (A, R, D)`, the SETAF is `SF^EF = (A', R')`:
- `A' = A ∪ X'`, `X' = {x' | x ∈ A, ∃y ∈ R s.t. (x, y) ∈ D}`
- `R' = {({x}, x') | x' ∈ X'} ∪ {(datt'(x, y) ∪ {x}, y) | (x, y) ∈ R}` with `datt'(x, y) = {c' | (c, (x, y)) ∈ D}`

### Translation 41 — EAFC→SETAF *(p.282)*
For bounded hierarchical EAFC, `SF^EFC = (A', R')`:
- `A' = A ∪ X'`, `X' = {x' | ∃y ∈ R, B ⊆ A s.t. x ∈ B and (B, y) ∈ D}`
- `R' = {({x}, x') | x' ∈ X'} ∪ {(G ∪ {x}, y) | (x, y) ∈ R, G ⊆ ⋃ gdatt'(x, y) s.t. ∀C ∈ gdatt'(x, y), G ∩ C ≠ ∅}` with `gdatt'(x, y) = {{c'_1, ..., c'_n} | ({c_1,...,c_n}, (x, y)) ∈ D}`

### Translation 42 — EAF→AFRA *(p.285)*
For bounded hierarchical EAF `bh-EF = (A, R, D)`, the AFRA is `FR^EF = (A, R ∪ D)`. Defense attacks become regular attacks. Resulting AFRA in `Rec_0^AFRA ∪ Rec_1^AFRA`.

### Translation 43 — EAF→AFN (defense-as-support) *(p.289)*
For bounded hierarchical EAF `bh-EF = (A, R, D)`, with `R_D = {(x, y) | (x, y) ∈ R, ∃c ∈ A s.t. (c, (x, y)) ∈ D}`:
- `A' = A ∪ X'`, `X' = {x' | x ∈ A, ∃y ∈ A s.t. (x, y) ∈ R_D}`
- `R' = (R \ R_D) ∪ {(x, x') | x' ∈ X'}`
- `N' = {(datt(x, y) ∪ {x'}, y) | (x, y) ∈ R_D}` with `datt(x, y) = {c | (c, (x, y)) ∈ D}`

### Translation 44 — EAF→AFN (chained, c-Tr) *(p.293)*
For bounded hierarchical EAF, the AFN is `FN^EF = (A', R', N')`:
- `A' = A ∪ R ∪ D`
- `R' = {((a, b), b) | (a, b) ∈ R, b ∈ A} ∪ {((c, (a, b)), (a, b)) | (c, (a, b)) ∈ D, (a, b) ∈ R}`
- `N' = {({a}, (a, b)) | (a, b) ∈ R, a ∈ A} ∪ {({c}, (c, (a, b))) | (c, (a, b)) ∈ D, c ∈ A}`

### Translation 45 — EAFC→AFN (defense-as-support) *(p.295)*
For bounded hierarchical EAFC `bh-EFC`, `FN^EFC = (A', R', N')`:
- `A' = A ∪ X'`, `X' = {x' | x ∈ A, ∃y ∈ A s.t. (x, y) ∈ R_D}`
- `R' = (R \ R_D) ∪ {(x, x') | x' ∈ X'}`
- `N' = {(G ∪ {x'}, y) | (x, y) ∈ R_D, G ⊆ ⋃ gdatt(x, y) s.t. ∀C ∈ gdatt(x, y), G ∩ C ≠ ∅}` with `gdatt(x, y) = {C | (C, (x, y)) ∈ D}`

### Translation 46 — EAFC→AFN (chained) *(p.296)*
For bounded hierarchical EAFC, `FN^EFC = (A', R', N')`:
- `A' = A ∪ R ∪ D`
- `R' = {((a, b), b) | (a, b) ∈ R, b ∈ A} ∪ {((C, (a, b)), (a, b)) | (C, (a, b)) ∈ D, (a, b) ∈ R}`
- `N' = {({a}, (a, b)) | (a, b) ∈ R, a ∈ A} ∪ {({c}, (C, (a, b))) | (C, (a, b)) ∈ D, c ∈ C}`

## Theorems / Propositions / Lemmas

### Theorem 4.5.2 *(p.252)*
Let `FR = (A, R)` AFRA, `F^FR_m = (A', R')` corresponding AF (Translation 33). For σ ∈ {cf, comp, pref, grd, stb}: if `E ⊆ A ∪ R` is σ-extension of FR then `E' = E ∪ {x' | x ∈ A ∩ E^+}` is σ-extension of F^FR_m, where `E^+ = {x | ∃y ∈ E s.t. y defeats x}` is the discarded set. Does not necessarily hold for adm. Conversely for σ' ∈ {adm, comp, pref, grd, stb}: if E' σ'-ext of F^FR_m then `E = E' ∩ (A ∪ R)` is σ'-ext of FR (not necessarily for cf).

### Theorem 4.5.3 *(p.260)*
AFRA→BAF preserves cf/stb/d-grd as **+cf/stb/d-grd** w.r.t. `R^sec`; for σ ∈ {adm, comp, pref}, as d-σ w.r.t. `(R^sec, R^sec)`.

### Theorem 4.5.4 *(p.262)*
AFRA→AFN (Translation 35) produces minimal, (strongly) consistent, weakly, strongly and relation valid AFNs.

### Theorem 4.5.6 *(p.263)*
For σ ∈ {cf, comp, pref, grd, stb}: σ-extension of FR is σ-extension of FN^FR. If `E = E^src` is admissible in FR, it is admissible in FN^FR; conflict-free in FR ⇒ strongly coherent in FN^FR (might fail otherwise). Reverse direction: not every cf in FN^FR is cf in FR; if E' strongly coherent in FN^FR then cf in FR; if E' σ'-ext of FN^FR for σ' ∈ {adm, comp, pref, grd, stb} then `E = E' ∩ A` is σ'-ext of FR.

## Semantics Defined / Compared

The semantic landscape used in this chunk:

- **AFRA semantics**: `cf, adm, comp, pref, grd, stb` over `E ⊆ A ∪ R`. Lemma 2.1.33 (referenced from earlier chunks): defending an attack implies defending its source.
- **AF semantics**: standard Dung. Translations to AFs require treating conflicts and source-args together.
- **BAF semantics**: `+cf` (positive conflict-free), `d-adm`, `d-comp`, `d-pref`, `d-grd`, `stb`, with second-order/secondary attacks `R^sec`. Translation 34 lifts AFRA into this richer semantic family with `R^sec` parameterization. *(p.259–260)*
- **AFN semantics**: strongly coherent, d-admissible, d-complete, d-preferred, d-grounded, stable; characteristic operator over support cycles. Translations 35, 43, 44, 45, 46 produce normal-form AFNs (well-structured, elementary, support-depth-1).
- **EAF/EAFC semantics**: cf can disagree on symmetric attacks (NSym subclass forces agreement); characteristic operator generally non-monotonic; complete need not have least; stable can be comparable. Bounded hierarchical EAFs restore monotonicity. *(p.267, p.286)*
- **SETAF semantics**: cf, adm, comp, pref, grd, stb on group-attacks. Theorems 2.1.24/2.1.25 from earlier chunks force complete-semilattice and stable-incomparability.
- **ADF ca-semantics**: only the EAF→ADF translation can avoid bounded-hierarchical restriction. ADF acceptance conditions (e.g. `C_b = ¬c ∨ b`) explicitly express "if attacker is accepted, then accept defense attacker" reasoning that is not tractable in other frameworks. *(p.299)*

Inclusions/equivalences worth noting:
- EAF cf ≡ EAFC cf on `BH^EAF ∪ NSym^EAF` (Theorem 4.6.1, 4.6.2). Outside this class they diverge (Example 5).
- EAF complete ≢ AFN complete in general; coincide via Translation 43 modulo `E ∪ {x' | x ∈ E^+}` projection.
- AFRA stable can be `⊆`-comparable; EAF stable can be `⊆`-comparable; these break AF/SETAF/AFN stable signatures (Theorems 4.6.8–4.6.9, 4.6.13–4.6.14, 4.6.26–4.6.27).

## Design Rationale

- **Choice: keep standard AFRA→AF (Tr 32) despite non-modularity** rather than restoring modularity at cost of exactness; "indirect defeats are a semantical notion … without them, semantic behavior is lost" *(p.252, p.256)*. Alternatives rejected: attack semantics [VBVDT11] (doesn't solve recursion); turning primed args into self-attackers (loses defense-via-indirect-defeat).
- **Defender Translation 33 redefined** to add `m-` prefix (`m-Tr_AF^AFRA`) so it can be put into the *system* of translations even though not modular and only faithful for adm/comp/pref/grd/stb (not exact for cf/adm in both directions). Trade-off: explicit modularity for one-way preservation *(p.253)*.
- **AFRA→BAF chosen as preferred translation** because it is the only one both modular and exact under any considered semantics; secondary attacks are exactly the abstract-support analog of indirect defeats *(p.260)*.
- **AFRA→AFN preferred over direct AFRA→ADF** because (i) ADF semantics don't list links in extensions, (ii) AFNs are minimal/binary/depth-1, (iii) AFN→ADF chain stays injective if AFN comes from AFRA *(p.262, p.265)*.
- **EAF translations restricted to bounded hierarchical** subclass because BH-EAFs restore monotonic characteristic operator, eliminate self-reinstatement, and allow defeat-based cf to coincide with EAFC cf *(p.254, p.267)*.
- **Defense-as-support (Translation 43)** is dual to defense-as-conflict (Translation 41 SETAF): "the negative direct attack relation that is simulated with a mix of conflict and support" — exposes a deep duality between AFN support and SETAF group attack. Both required to characterize EAF defense attacks under different semantics *(p.291)*.
- **Two parallel EAF→AFN translations (43 and 44)**: 43 is non-modular but tight; 44 is chained (EAF→AFRA→AFN) and produces support-binary depth-1 AFNs, matching the AFRA→AFN structure. Reader chooses based on whether semantics fidelity or normal-form simplicity matters more *(p.292–293)*.
- **EAF→ADF "overpowering support"**: ADF is the *only* target where defense attack can be expressed as a positive relation that overrides incoming conflicts but is *not necessary* for acceptance — not expressible in BAF (necessary) or AFN (necessary group support) frames *(p.299)*. ADF ca-semantics replicate EAF non-monotonicity; this is presented as a *positive* discovery rather than a defect, since previous "exactness impossibility" stems from this same non-monotonicity.
- **Impossibility theorems are kept as separate results** (4.6.8–9, 4.6.13–14, 4.6.16–17, 4.6.26–27) rather than collapsed into a single meta-result, because the failure mode differs by target: AF/SETAF fail on cf-downward-closure + complete-least + stable-incomparability; AFRA fails additionally on adm/pref; AFN fails on cf/comp/stb but admissible/preferred is open *(p.286, p.298)*.

## Limitations Acknowledged

- **AFRA–AF "exact" classification is not entirely uncontroversial**: the translation must "read back" some arguments as conflicts, which contradicts the principle of exact translations being free of such reading. Polberg flags this as "open for discussion" *(p.250)*.
- **AFRA→AFN admissible semantics weak** because necessary support enforces sources to appear, dropping AFRA admissible sets like `{α}` *(p.264)*. No improvement available without losing modularity (Translation 32 case).
- **EAF/EAFC translations restricted to `BH ∪ NSym`** subclass; "fullness" (every EAF/EAFC translatable) is left for future work. Authors are not aware of any solution at the time of writing *(p.270, p.275)*.
- **EAF→AF/SETAF/AFRA/AFN exactness for admissible and preferred semantics: open question** *(p.279, p.284, p.286, p.298)*. Only necessary conditions known for complete; sufficiency is conjectured "no" but not proven for adm/pref.
- **EAF→AFRA produces frameworks with problematic admissible behavior** (Example 107: `{a, b, c, d}` cf in AFRA but not in source EAF) — fundamental EAF cf-asymmetry that AFRA cannot accommodate *(p.286)*.
- **AFN translation 43 not modular** (Example with `EF_1 = ({a, b}, {(a, b)}, ∅)` and `EF_2 = ({a, b, c}, {(a, b)}, {(c, (a, b))})` — `FN_1 ∪ FN_2 ≠ FN^{EF_1 ∪ EF_2}`) *(p.291)*.
- **"Source-subclass and lack of modularity"**: trade-off in Translation 41 (EAFC→SETAF) — overlapping but not modular *(p.283)*.
- **Faithful but not exact**: Translation 33, Translation 38, Translation 39, Translation 44, Translation 45, Translation 46 are all faithful (semantics bijective for σ ∈ {comp, pref, grd, stb}) but not exact under cf/adm.

## Implementation Notes

- **For propstore's argumentation layer**, the BAF translation (Translation 34) is the most attractive AFRA target: modular + exact ⇒ stored AFRAs can be incrementally lifted to BAFs without rebuilding, and union-of-AFRAs distributes through the translation. If propstore stores AFRAs as `(args, attacks_with_recursion)`, the BAF view is `(args ∪ attacks, attack_targets, support_to_source)`.
- **Translation 36 (EAF→EAFC)** is the *cheapest* normalization: just lift each defense attacker `c` to `{c}`. Useful as a canonicalization step before semantics queries on the bounded-hierarchical subclass; preserves all six standard semantics.
- **Translation 38 / 39 (EAF/EAFC → AF, defender style)** require introducing `|A|` primed arguments and roughly `|A| + |R| + |D|` extra attacks; modular; faithful for complete/preferred/grounded/stable. Reasonable to implement as an in-memory rewrite when delegating to a Dung-AF solver, *provided* the source is bounded hierarchical or symmetric-attack-free.
- **Bounded-hierarchical check** (Definition 2.1.62 referenced p.262) is a precondition for many translations. propstore should annotate EAF stores with a "bh" sidecar flag and reject non-bh inputs from the affected translation paths, or fall back to ADF.
- **Translation 42 (EAF→AFRA = (A, R ∪ D))** is trivially structural: just merge the defense attacks into the attack set. `Rec_0 ∪ Rec_1` recursion class is automatic (defense attacks attack only attacks). Cheapest possible EAF→AFRA path, useful when a downstream component only consumes AFRA shape.
- **Translation 43 (EAF→AFN)** is the most subtle: dynamic computation of `R_D` (defense-attacked attacks), then *removing* attacks while introducing primed copies and a support set per removed `(x, y)`. Implementation needs to walk `D` once to build `R_D`, walk `R_D` to introduce primed args, walk `D` again to fill support sets. O(|R| + |D|) memory.
- **For honest-non-commitment**: the impossibility theorems (4.6.8–9, 4.6.13–14, 4.6.16–17, 4.6.26–27) are *load-bearing* for propstore's design — they say that EAF/EAFC semantics genuinely cannot be losslessly squeezed into AF/SETAF/AFRA/AFN. Storage of EAF claims must therefore retain `D` separately; downgrading to AF semantics is a *semantic-altering* operation that must be marked at render policy time, never silently in storage. Propstore's policy `RenderPolicy(target=AF)` should attach a provenance note "translation lossy: cf/adm/pref under bounded-hierarchical subclass only" referencing the relevant theorem.
- **ADF as the only universal target**: if propstore wants a *single* internal normalized form that all EAF/EAFC/AFRA/SETAF claims project into without bounded-hierarchical restriction, it should be ADF (per §4.6.6 introduction, p.299). The Polberg ADF translation handles inconsistency and overpowering support; cost is that ADF semantics are themselves richer (ca-semantics) and require more computational machinery. **Recommendation**: store source frameworks natively, use ADF as the lazy render-time rendezvous when cross-framework reasoning is required.
- **Defense-attacker computation**: `datt(x, y) = {c | (c, (x, y)) ∈ D}` and `gdatt(x, y) = {C | (C, (x, y)) ∈ D}` are reusable primitives — propstore should expose them as helpers since they appear in nearly every EAF/EAFC translation.
- **Casting functions** (`SC_σ^Tr` removal/identity) are first-class objects; propstore's translation API should accept a casting function parameter rather than hardcoding identity, so users can request semantics-bijective vs strong vs weak transformations explicitly.
- **Property classification vocabulary** (full / target-subclass / source-subclass / injective / argument-domain altering / argument introducing / induced-attack-introducing / induced-support-introducing / generic / semantics-domain preserving / semantics-domain altering / structural / semi-structural / modular / exact / faithful / strong / ⊆-weak / ⊇-weak / overlapping / basic / basic-defender / basic-attack-propagation / hybrid) — propstore should mirror this taxonomy as a typed classification of every stored translation; the analysis blocks at end of each translation can serve as canonical type-checks.
