## Key Findings
- Range covers Chapter 4 "Intertranslatability of Abstract Argumentation Frameworks": completion of normal-form / subclass taxonomy for AFN/EAS/ADF, and exhaustive translations from AFs and SETAFs into every other framework in the thesis (SETAF, AFRA, EAF/EAFC, BAF, AFN, EAS, ADF). *(p.200-249)*
- Each translation is presented twice: first informally with a Translation N statement, then as a "Redefinition of Translation N" using the typed translation operator `Tr_target^source : Fr^source ‚Üí Fr^target` from Section 4.1, followed by an "Analysis of Translation N" listing properties (functional, syntactical, semantical, computational, modularity) and a final classification (basic / basic-coalition hybrid / basic-defender hybrid). *(passim)*
- AF is exact under (almost) every other framework (SETAF, EAF, BAF, AFN, ADF); only AFRA and EAS lose exactness because their semantics extend the AF semantics domain (attacks-in-extensions, evidence argument). *(p.222 Summary; Table 4.3 p.223)*
- SETAF is the first framework where exact translation to AF is provably impossible (Theorem 4.4.4) ‚Äî group attacks cannot be matched by AF signatures (downward closure / completion-set / tightness fail). *(p.235)*
- AF‚ÜíADF translation collapses the ADF sub-semantics classification: AF-style ADFs are simultaneously AADF‚Å∫, BADF, redundancy-free, weakly/relation/strongly valid and cleansed (Theorem 4.3.17). *(p.219)*
- The translation classification system of Section 4.1 is now operational across many translations and is the central organizing tool for the chapter.

## Definitions Introduced

### Definition 4.2.70 ‚Äî AFN subclasses *(p.200)*
Let `FN = (A,R,N)` be an AFN and `SG^FN = (A,N')` where `N' = {(a,b) | ‚àÉE‚äÜA, a‚ààE s.t. ENb}` (the support graph induced by FN). Then FN is:
- **without support** iff `N = ‚àÖ`.
- **support binary** iff for every `S‚äÜA` s.t. `‚àÉa, (S,a)‚ààN`, `|S|=1`.
- **support singular** iff for every argument `a‚ààA` there exists at most one set `S‚äÜA` s.t. `(S,a)‚ààN`.
- **of support depth n** if all (simple) paths in `SG^FN` are of length at most n.
- **elementary with depth n** if it is support binary, strongly valid and of support depth n.
- **well‚Äìstructured** if it is minimal, strongly consistent and strongly valid.

Notation: `NSup^AFN ‚ää Fr^AFN`, `SBin^AFN ‚ää Fr^AFN`, `SSig^AFN ‚ää Fr^AFN` for no-support, support-binary, support-singular subclasses; `Sup_n^AFN ‚äÇ Fr^AFN` for depth-n; `SEle_n^AFN = SBin^AFN ‚à© SV^AFN ‚à© Sup_n^AFN`; `WSt^AFN = Min^AFN ‚à© SCons^AFN ‚à© SV^AFN`.

### Definition 4.2.72 ‚Äî EAS subclasses *(p.201)*
Let `ES = (A,R,E)` be an EAS. It is:
- **support singular** iff for every `a‚ààA` there exists at most one set `S‚äÜA` s.t. `(S,a)‚ààE`.
- **attack singular** iff for every `a‚ààA` there exists at most one `S‚äÜA` s.t. `(S,a)‚ààR`.
- **support binary** iff for every `S‚äÜA` s.t. `‚àÉa,(S,a)‚ààE`, `|S|=1`.
- **attack binary** iff for every `S‚äÜA` s.t. `‚àÉa,(S,a)‚ààR`, `|S|=1`.
- **all‚Äìsupported** iff for every `a‚ààA\{Œ∑}` there is `S‚äÜA` s.t. `(S,a)‚ààE`.
- **pure evidence supported** iff every `S‚äÜA` s.t. `‚àÉa,(S,a)‚ààE` is of the form `S={Œ∑}`.

Notation: `ABin^EAS`, `ASig^EAS`, `SBin^EAS` (support binary, but on the EAS side this name is also used; the text labels them `ABin^EAS / SBin^EAS` for support side), `AllSup^EAS ‚ää Fr^EAS`, `EvSup^EAS ‚ää Fr^EAS`.

### Definition 4.2.74 ‚Äî Dung-style ADF *(p.202)*
Let `D = (A,L,C)` be an ADF. D is **Dung‚Äìstyle (AF-style)** if for every argument `a‚ààA`, `C_a(‚àÖ) = in` and for every nonempty `E‚äÜpar(a)`, `C_a(E) = out`. Equivalently every condition is a conjunction of negated arguments or `‚ä§`.

### Definition 4.2.75 ‚Äî SETAF-style ADF *(p.202)*
Let `D = (A,L,C)` be an ADF. D is **SETAF‚Äìstyle** if for every `a‚ààA`:
- (a) `C_a(‚àÖ) = in`,
- (b) if `par(a)‚âÝ‚àÖ`, then there exists `E‚äÜpar(a)` s.t. `C_a(E) = out`,
- (c) for every nonempty `E‚äÜpar(a)` s.t. `C_a(E) = out`, `C_a(E') = out` for every `E ‚äÜ E' ‚äÜ par(a)`.

Equivalently: every condition is a conjunction of clauses containing only negated arguments or `‚ä§`.

### Definition 4.2.76 ‚Äî EAFC- and EAF-style ADF *(p.202)*
Let `D = (A,L,C)` be an ADF. D is **EAFC‚Äìstyle** if for every `a‚ààA`, `C_a` is a conjunction of clauses containing exactly one negated argument and an arbitrary amount of positive ones, or `C_a = ‚ä§`. D is **EAF‚Äìstyle** if it is EAFC-style and no negation of an argument appears in two different clauses.

Subclasses: `ADF^AF ‚ää Fr^ADF`, `ADF^SETAF ‚ää Fr^ADF`, `ADF^EAFC ‚ää Fr^ADF`, `ADF^EAF ‚ää ADF^EAFC`.

### Definition 4.3.4 ‚Äî AFRA `‚ÜíAFRA` operator *(p.206)*
Let `FR = (A,R)` be an AFRA. Given `E‚äÜA`, `E^{‚ÜíAFRA} ‚âú E ‚à™ {V‚ààR | src(v)‚ààE}`. Used as casting function so AFRA extensions (which contain attacks) line up with AF extensions.

### Translation 17 ‚Äî AF as SETAF *(p.204)*
Let `F = (A,R)`. Corresponding SETAF `SF^F = (A,R')` where `R' = {({a},b) | (a,b)‚ààR}`. Targets `Bin^SETAF`. Also in `Min^SETAF` because `Bin ‚äÇ Min` (Lemma 4.2.63).

### Translation 18 ‚Äî AF as AFRA *(p.206)*
Let `F = (A,R)`. Corresponding AFRA `FR^F = (A,R)` (same tuple). The produced AFRA is non-recursive (`Rec_0^AFRA`).

### Translation 19 ‚Äî AF as EAF *(p.211)*
Let `F = (A,R)`. Corresponding EAF `EF^F = (A,R,‚àÖ)` (empty defense attack set). Belongs to `NDef^EAF` (subclass with empty defense), and is also bounded hierarchical and (strongly) consistent (Theorem 4.3.8).

### Translation 20 ‚Äî AF as BAF *(p.213)*
Let `F = (A,R)`. Corresponding BAF `BF^F = (A,R,S)` with `S = ‚àÖ`. No indirect attacks; `R^ind = ‚àÖ`; safety/closure trivially preserved (Lemma 4.2.69).

### Translation 21 ‚Äî AF as AFN *(p.214)*
Let `F = (A,R)`. Corresponding AFN `FN^F = (A,R,N)` with `N = ‚àÖ`. Lies in `NSup^AFN`; minimal, (strongly) consistent, weakly/relation/strongly valid (Theorem 4.3.12).

### Translation 22 ‚Äî AF as EAS *(p.216)*
Let `F = (A,R)`. Corresponding EAS `ES^F = (A‚à™{Œ∑}, R', E)` where `R' = {({a},b) | (a,b)‚ààR}` and `E = {({Œ∑},a) | a‚ààA}`. Belongs to subclass `AF^EAS = ABin^EAS ‚à© EvSup^EAS ‚à© AllSup^EAS`. Argument domain weakly altered (`Œ∑` introduced).

### Translation 23 ‚Äî AF as ADF *(p.220)*
Let `F = (A,R)` and `X^a = {x_1,...,x_{n_a}}` the attackers of `a`. The ADF `D^F = (A,R,C)` with `C = {C_a}_{a‚ààA}` and each `C_a` defined either:
- **Functional form:** `C_a(‚àÖ) = in`, `C_a(B) = out` for every nonempty `B ‚äÜ X^a`.
- **Propositional form:** `C_a = att_a = ‚ãÄ ¬¨X^a` (and `‚ä§` if `X^a` empty).

Produced ADF is `AADF‚Å∫ ‚à© BADF`, redundancy-free, weakly/relation/strongly valid and cleansed (Theorem 4.3.17). Sub-semantics classification collapses on Dung-style ADFs *(p.220)*.

### Translation 24 ‚Äî SETAF as AF (Deprecated coalition) *(p.225)*
Let `SF = (A,R)`. The AF `F^SF = (A',R')`:
- `arg(S) = {{a} | a‚ààS}` for `S‚äÜA`,
- `att(S) = {S' | S'‚äÜS ‚àß |S'|>1 ‚àß ‚àÉy‚ààA. (S',y)‚ààR}`,
- `A' = arg(A) ‚à™ att(A)`,
- `R' = {(X,Y) | ‚àÉy‚ààY, X'‚äÜX s.t. (X',y)‚ààR}`.

Deprecated because of conflict-freeness propagation issues; subset-attack step is overlapping.

### Translation 25 ‚Äî SETAF as AF (Coalition, current) *(p.226)*
Same `arg(S)`, `att(S)` definitions. `A' = arg(A) ‚à™ att(A)` and `R' = {(X,Y) | ‚àÉy‚ààY s.t. (X,y)‚ààR}`. Removes the subset-propagation step. Strong and semantics-bijective under {admissible, preferred, complete, grounded, stable} with union casting. Classified **basic-coalition hybrid**. *(p.227)*

### Translation 26 ‚Äî SETAF as AF (Defender / `def-Tr_AF^SETAF`) *(p.230)*
Let `SF = (A,R)`. `F_def^SF = (A',R')`:
- `A' = A ‚à™ R ‚à™ X'`, where `X' = {x' | x‚ààA}` are primed copies,
- `R' = {(x,x') | x'‚ààX'} ‚à™ {(x',(X,y)) | x‚ààX, (X,y)‚ààR} ‚à™ {((X,y),y) | (X,y)‚ààR}`.

Each attack `(X,y)` is "expanded": each `x‚ààX` defeats `x'` and each `x'` attacks the conflict argument `(X,y)` which then attacks `y`. Faithful (not bijective) for {admissible, preferred, complete, grounded, stable}. Classified **basic-defender hybrid**. *(p.232)*

### Translation 27 ‚Äî SETAF as AF (Improvement of 25) *(p.234)*
Drops `arg(S)` part: `att(S) = {S' | S'‚äÜS ‚àß |S'|>1 ‚àß ‚àÉy‚ààA. (S',y)‚ààR}`, `A' = A ‚à™ att(A)`, `R' = {(X,Y) | ‚àÉy‚ààY s.t. (X,y)‚ààR or ({X},y)‚ààR}`. Strong and semantics-bijective under complete, preferred, grounded, stable; ‚äÜ-weak under conflict-free and admissible. Faithful under complete-and-up. Conflict-free fails completely (not just one direction).

### Translation 28 ‚Äî SETAF as AFN (Deprecated) *(p.238)*
Same `arg(S)`, `att(S)`. `A' = arg(A) ‚à™ att(A)`, `R' = {(X,{y}) | (X,y)‚ààR}`, `N = {({X},Y) | X‚ààarg(A), Y‚ààatt(A), X‚äÜY}`.

### Translation 29 ‚Äî SETAF as AFN (current) *(p.238-239)*
- `att(S) = {S' | S'‚äÜS ‚àß |S'|>1 ‚àß ‚àÉy‚ààA.(S',y)‚ààR}`
- `A' = A ‚à™ att(A)`
- `R' = {(X,y) | (X,y)‚ààR or ({X},y)‚ààR}`
- `N = {({X},Y) | X‚ààA, Y‚ààatt(A), X‚ààY}`

Targets `WSt^AFN ‚à© SEle_1^AFN`: support binary, depth 1, well-structured + elementary. Semantics-bijective under complete/preferred/grounded/stable; ‚äÜ-weak under conflict-free. Modular (unlike Translation 25). Classified **basic-coalition**.

### Translation 30 ‚Äî SETAF as EAS *(p.241)*
Let `SF = (A,R)`. `ES^SF = (A',R,E)` with `A' = A‚à™{Œ∑}` and `E = {({Œ∑},a) | a‚ààA}`. Targets `SETAF^EAS = EvSup^EAS ‚à© AllSup^EAS` (does not satisfy minimality). Faithful (not bijective) for complete/preferred/grounded/stable; weakly faithful under admissible. Classified **basic**.

### Translation 31 ‚Äî SETAF as ADF *(p.243)*
Let `SF = (A,R)`. `D^SF = (A,L,C)`:
1. For every `x,y‚ààA` s.t. `‚àÉB‚äÜA, x‚ààB` and `BRy`, add `(x,y)` to `L`.
2. For every `a‚ààA`, with `X = {X_1,...,X_n}` the collection of all sets s.t. `X_iRa`:
   - **Functional:** `C_a(B) = out` for `B ‚äÜ ‚ãÉX` s.t. `‚àÉX_i‚ààX, X_i‚äÜB`. All remaining subsets are `in`.
   - **Propositional:** `C_a = att_a = ‚ãÅ¬¨X_1 ‚àß ... ‚àß ‚ãÅ¬¨X_n`. If `X` is empty, `C_a = ‚ä§`.

Produced ADF is `AADF‚Å∫ ‚à© BADF`, cleansed and weakly valid; if SF minimal, also redundancy-free, relation- and strongly-valid (Theorem 4.4.11). **Exact** under conflict-free (pd-acyclic), xy-admissible/complete/preferred (`x,y‚àà{a,c}`), acyclic grounded, stable model. **Overlapping** (not injective) ‚Äî minimal SETAFs do collapse to distinct ADFs. ‚äó-modular but not ‚äï-modular. *(p.245-246)*

### Translation 32 ‚Äî AFRA as AF *(p.249)*
Let `FR = (A,R)`. Corresponding AF `F^FR = (A',R')` with `A' = A‚à™R` and `R' = {(V,W) | V,W‚ààA‚à™R and V defeats W}`. Domain growth `‚ãÉ_{i=1}^‚àû U^i` to accommodate attacks-as-arguments.

**There exists no full translation from SETAFs to AFs that is exact under conflict-free, admissible, complete, preferred and stable semantics with identity casting functions.** Proof technique: signature-mismatch via Example 91 ‚Äî SETAF extensions of `({x,y,z},{({x,y},z),({y,z},x),({x,z},y)})` are `S_1 = {‚àÖ,{x},{y},{z},{x,y},{x,z},{y,z}}`. `S_1` is not adm-closed (no `{x,y,z}`), `S_2 = {‚àÖ,{x,y},{x,z},{y,z}}` is not com-closed, etc. ‚Äî so `S_1, S_2, S_3` fail every AF semantics signature. Note: exact translation for grounded **does** exist (single-extension trick ‚Äî drop everything not in the extension). *(p.235)*

### Theorem 4.4.5 *(p.239)*
`FN^SF` (Translation 29) is minimal, (strongly) consistent, weakly, relation and strongly valid.

### Theorem 4.4.6 *(p.239)*
For Translation 29: conflict-freeness preservation in one direction (E conflict-free in SF ‚áí conflict-free in `FN^SF`). œÉ-extension correspondence with `E' = E ‚à™ att(E)`.

### Theorem 4.4.7 ‚Äî Impossibility (SETAF‚ÜíAFN) *(p.241)*

### Theorem 4.4.9 *(p.241)*

### Theorem 4.4.10 ‚Äî Impossibility (SETAF‚ÜíEAS) *(p.243)*

## Algorithms / Procedures

### Algorithm ‚Äî Translate AF to ADF (Translation 23) *(p.220)*
1. Keep argument set A and link relation R.
2. For each `a ‚àà A`:
   - If `X^a = ‚àÖ`, set `C_a = ‚ä§`.
   - Else set `C_a` functionally: `C_a(‚àÖ) = in`, `C_a(B) = out` for nonempty `B ‚äÜ X^a`.
   - Equivalently propositional: `C_a = ‚ãÄ ¬¨X^a`.

### Algorithm ‚Äî Translate SETAF to AF via defender expansion (Translation 26) *(p.230)*
1. For each `a ‚àà A`, introduce primed copy `a'`. `A' = A ‚à™ R ‚à™ X'` where `X' = {x' | x‚ààA}`.
2. For every `a‚ààA`, add attack `(a, a')`.
3. For every set attack `(X,y) ‚àà R` and every `x ‚àà X`, add `(x', (X,y))`.
4. For every set attack `(X,y) ‚àà R`, add `((X,y), y)`.

### Algorithm ‚Äî Translate SETAF to ADF (Translation 31) *(p.243)*
1. For every pair `x,y ‚àà A` such that there exists `B‚äÜA` with `x‚ààB` and `BRy`, add `(x,y)` to L.
2. For each `a‚ààA`, let `X = {X_1,...,X_n}` with `X_iRa`.
   - Functional: `C_a(B) = out` for `B‚äÜ‚ãÉX` such that some `X_i ‚äÜ B`; remaining subsets `in`.
   - Propositional: `C_a = ‚ãÅ¬¨X_1 ‚àß ... ‚àß ‚ãÅ¬¨X_n` (or `‚ä§`).

## Semantics Defined / Compared

For each target framework Polberg restates which AF semantics map onto which target semantics:

- **AF ‚Üí SETAF (Tr 17):** identity casting; conflict-free, admissible, preferred, complete, grounded, stable all **exact**. Target subclass `Bin^SETAF`.
- **AF ‚Üí AFRA (Tr 18):** **‚äÜ-weak** under conflict-free and admissible (asymmetry in Theorem 4.3.6); strong + bijective under complete, preferred, grounded, stable with the `‚ÜíAFRA` casting; improvable to bijective for admissible via Theorem 4.3.7.
- **AF ‚Üí EAF (Tr 19):** all six exact (target `NDef^EAF`).
- **AF ‚Üí BAF (Tr 20):** all six exact under d-family with `R',R'' ‚äÜ R^ind = ‚àÖ` parametrization.
- **AF ‚Üí AFN (Tr 21):** all six exact; target `NSup^AFN ‚ää WSt^AFN ‚à© SEle`.
- **AF ‚Üí EAS (Tr 22):** complete, preferred, grounded, stable **faithful** (not bijective) due to Œ∑; conflict-free + admissible **strong**. Target `AF^EAS`.
- **AF ‚Üí ADF (Tr 23):** all six exact under `xy ‚àà {a,c}` flavors. Target `ADF^AF`.
- **SETAF ‚Üí AF (Tr 25):** strong + bijective for adm/comp/pref/grd/stb (with union casting); ‚äÜ-weak for cf. Hybrid (basic-coalition).
- **SETAF ‚Üí AF (Tr 26):** faithful for comp/pref/grd/stb; ‚äÜ-weak for cf and adm. Hybrid (basic-defender).
- **SETAF ‚Üí AFN (Tr 29):** strong + bijective for comp/pref/grd/stb; ‚äÜ-weak for cf. Modular (key advantage over Tr 25).
- **SETAF ‚Üí EAS (Tr 30):** faithful for comp/pref/grd/stb; weakly faithful adm.
- **SETAF ‚Üí ADF (Tr 31):** **exact** under all six (with `xy ‚àà {a,c}`). Overlapping (not injective) ‚Äî first non-injective translation in the chapter. ‚äó-modular only.

Thus the trajectory: AF nests trivially everywhere; SETAF is the first true generalization (no exact SETAF‚ÜíAF possible) but ADF can recover SETAF exactly.

## Design Rationale

- **Why two SETAF‚ÜíAF translations (coalition + defender)?** Because each illustrates a distinct family in the classification: the coalition approach gains injectivity but loses modularity (one big arg per attacker set; union of frameworks introduces extraneous conflicts); the defender approach uses primed copies and keeps modularity but at the cost of overlapping behavior under admissibility *(p.227-232)*. Both classified as hybrids ‚Äî a feature of the property lattice, not a bug.
- **Why drop the subset-attack propagation (Tr 24 ‚Üí Tr 25)?** Conflict-freeness was not preserved by Tr 24 either, but Tr 24 also "spuriously" linked frameworks that did not contain those subsets in the first place (overlapping). Removing it gives a strictly less lossy translation while retaining all critical semantics correspondences thanks to minimal-form properties of SETAFs *(p.225-226)*.
- **Why introduce Œ∑ outside U?** So that EAS-targeting translations cannot be confused with reusing existing arguments in evidence role; the problem `({a,b},{(a,b),(b,a)})` in *(p.218)* shows neither argument can play Œ∑'s role *(p.218)*. This makes EAS translations weakly domain-altering by design.
- **Why use propositional + functional ADF conditions in parallel?** Functional is direct (defines `C_a` as a function); propositional gives `att_a` as a logical formula `‚ãÄ¬¨X^a` so that classification under EAFC/EAF/SETAF subclasses can be inferred syntactically *(p.220, p.243)*.
- **Why classify the SETAF‚ÜíADF translation as `overlapping`?** Two distinct SETAFs can produce the same ADF when one is the minimal form of the other (Section 4.2.1.5 forward reference) *(p.246)*; restricting to minimal-form SETAFs would restore injectivity. Polberg accepts overlapping rather than restrict the source domain because it preserves expressivity.
- **Why is the AF‚ÜíADF translation NOT ‚äï-modular?** Because joining two AF-style ADFs via ‚äï can produce a condition that is no longer AF-style (target subclass not closed under ‚äï) *(p.221)*. ‚äó-modular (intersection-of-conditions) survives because conjoining `‚ãÄ¬¨X^a` with itself stays AF-style.

## Limitations Acknowledged

- **AF‚ÜíAFRA bijection under conflict-freeness/admissibility:** "we are not aware of any method that would allow us to tackle the first issue and are not convinced that a direct faithful translation for admissible semantics exists" *(p.210)*. AFRA-produced AFNs (Translation 35, forward reference) can address this for some subset of cases.
- **AF‚ÜíEAS exactness under any non-trivial semantics:** Theorem 4.3.16 ‚Äî no full translation exists. Only conflict-free can be exact (because it ignores support) *(p.219)*.
- **SETAF‚ÜíAF exact under cf/adm/comp/pref/stb:** Theorem 4.4.4 ‚Äî impossible *(p.235)*. Grounded is exact but trivially so (lossy single-extension hack).
- **SETAF‚ÜíAFN exact under admissible/preferred:** Theorem 4.4.7 ‚Äî impossible *(p.241)*. Open whether stable is exact (signatures unresearched).
- **SETAF‚ÜíEAS exact:** Theorem 4.4.10 ‚Äî impossible *(p.243)* (Œ∑ addition unavoidable for any non-empty case).
- **SETAF‚ÜíADF injectivity:** translation is overlapping; can be made injective by restricting to minimal SETAFs *(p.246)*.
- **SETAF‚ÜíBAF unfinished:** group-support / group-closure absent in BAFs prevents a clean translation; redirected to AFN as substitute *(p.237)*.
- **SETAF‚ÜíAFRA, EAF, EAFC:** not analyzed in the section; conjectured to behave like SETAF‚ÜíAF *(p.247)*.

Store ADF acceptance conditions either as functional Python callables `C_a: 2^A ‚Üí {in,out}` or as propositional formulas `‚ãÄ¬¨X^a` in a small AST. The dual storage matches Polberg's both-forms style and lets sub-semantics classification (`Dung^ADF`, `SETAF^ADF`, `EAFC^ADF`, `EAF^ADF`) be checked syntactically.
- **For lazy/non-commitment storage:** every translation here is invertible only in one direction (AF‚Üítarget) but generally not target‚ÜíAF (e.g. SETAF‚ÜíAF can't be exact). When storing rival framework views, never collapse a SETAF claim down to AF representation ‚Äî Theorem 4.4.4 says you'd lose extensions. Keep both forms with provenance.
- **Casting functions as first-class objects:** Polberg's casting `SC_œÉ^X` takes the source framework AND the target extension. Implement as `cast(framework, extension, sigma) -> source_extension`. The fact that some translations need *two-step* casting (removal then extraction, *(p.208-209)*) means the casting interface must allow composition.
- **The `xy` flavor for ADF semantics:** when implementing ADF‚ÜíAF correspondence checks, the four flavors (`{a,c}^2 = {aa, ac, ca, cc}` admissible, complete, preferred, grounded) must all be tracked; Translation 23 makes them collapse for AF-style ADFs (Theorem 3.5.2 forward ref) so a single ADF‚ÜíAF check suffices for AF-produced ADFs but not in general.
- **Translation classification system:** the per-translation properties ‚Äî strength (cf/adm/comp/pref/grd/stb each ‚àà {exact, strong, faithful, ‚äÜ-weak}), functional (full/target-subclass/injective vs overlapping), syntactical (argument-domain preserving/altering, structure preserving/altering, argument-introducing, induced-attack-introducing, induced-support-introducing), semantical (generic/non-generic, semantics-domain preserving/altering), computational (structural/semi-structural, modular/‚äó-modular/non-modular) ‚Äî should be encoded as flags on a Translation object so analyses can be queried generically.
- **Œ∑ argument as ATMS-style assumption:** EAS translations introduce Œ∑ whose support is the source of every other support relation. In ATMS terms Œ∑ is the assumption that "evidence is available"; translating SETAF/AF into EAS effectively wraps each argument under that single assumption. This maps cleanly to a sidecar bundle environment.
- **Defender expansion as graph rewrite:** Translation 26 (and Translation 25 for SETAF‚ÜíAF) is straight graph rewriting. Concretely: replace each set attack `(X,y) ‚àà R` with a 4-node gadget `(x ‚àà X, x', (X,y), y)`. Easy to implement as a pass over an AF graph.
- **No exact SETAF‚ÜíAF means:** propstore must keep set-attack semantics first-class; translating set attacks to single-element AF attacks during render is a one-way lossy step that should be flagged.
