## Key Findings
- BAF d-grounded extension is the least fixed point of the d-characteristic operator and is +conflict-free (Def. 2.2.14, Prop. 2.2.15) *(p.50)*.
- Original (binary) AFNs from [NR11] suffer from the unintended-admissibility problem demonstrated in Example 14 — the formulation gives extensions like {a,b} that violate intended intuitions; thesis abandons binary AFNs in favor of set-AFNs from [Nou13] *(p.52–53)*.
- Set-AFN semantics introduces "powerful sequence" instead of N-cycle-freeness, plus a `discarded set` E^att that is a subset of the deactivated set E+ (Lemma 2.2.28) *(p.53–54)*.
- Evidential argumentation systems (EAS) require all valid arguments and attackers to trace back to a special evidence node η; e-support is built via evidential sequences ending at η (Theorem 2.2.35) *(p.56)*.
- EAS Fundamental Lemma (Lemma 2.2.46) and full ADF-style lattice of stable ⊂ preferred ⊂ complete; grounded = least fixpoint (Theorem 2.2.48) *(p.58)*.
- Realizability theory: signatures Σ_σ^AF for cf, stable, adm, pref characterized by combinations of {downward-closed, tight, incomparable, adm-closed} (Theorem 2.3.4); complete still has only necessary conditions (Prop. 2.3.5) — `com-closed` plus contains-empty plus adm-closed *(p.61)*.
- Dyrek/Dyr14 result lifted: any framework can be translated into a Dung AF that realizes the same set of preferred or semi-stable labelings with auxiliary arguments (Theorem 2.3.6) *(p.61)*.
- ADFs replace separate attack/support relations with **acceptance conditions** Cₐ : 2^par(a) → {in, out} per argument; links L are derivable from the conditions (notation D = (A, C) when L is omitted) *(p.66)*.
- A link is supporting iff no R⊆par(s) flips Cs(R)=in to out by adding r; attacking iff no R flips out→in by adding r; an ADF is **bipolar** (BADF) if every link is supporting or attacking — links can be both (`redundant`) or neither (Def. 3.1.2) *(p.67)*.
- BADFs have lower complexity than general ADFs [SW14], but Polberg points out that "support cycle" in ADFs is not the same as cycles on supporting links — uses `positive dependency cycles` instead *(p.67)*.
- Three-valued interpretations: precision ordering u ≤ᵢ t and u ≤ᵢ f (information order); ({t,f,u}, ≤ᵢ) is a complete meet-semilattice with meet ⊓ giving u in all mismatch cases *(p.67)*.
- **Decisiveness** (Def. 3.1.4): a two-valued interpretation v on E ⊆ A is decisive for s iff every two completions of v to E ∪ par(s) evaluate Cs identically *(p.69)*.
- `acc(G,B)`/`reb(G,B)` set-form of decisiveness (Def. 3.1.5) coincides with checking decisively in/out w.r.t. an interpretation v with v^t = G, v^f = B *(p.70)*.
- Positive dependency function pd_E^D maps each argument in E to a minimal decisively-in interpretation (or N for null); sound iff no argument maps to N; maximally sound iff cannot be extended without breaking soundness (Def. 3.1.6) *(p.70)*.
- Three flavors of evaluations build on pd-functions: standard (Def. 3.1.7), partially acyclic (Def. 3.1.8 + body p.72 stating the ordering requirements), and acyclic (Def. 3.1.9, F=∅) — these are the central technical device for the thesis's classification system *(p.71–72)*.
- Range comes in three flavors mirroring the evaluation flavors: standard range E^Ran = E ∪ E^+ (Def. 3.1.13, p.75), acyclic range E^a+ via "block all acyclic pd-evaluations" (Def. 3.1.15, p.75; also marked `Deprecated` since Lemma 3.1.16 shows the iteration terminates after one round), and partially acyclic range E^p+ (Def. 3.1.20, p.77). Lemma 3.1.21: E+ ⊆ E^p+ ⊆ E^a+ for conflict-free E *(p.77)*.
- Labeling-based ADF semantics (Strass 2013, Brewka et al. 2013) use a three-valued characteristic operator Γ_D(v)(a) = ⊓_{w ∈ [v]_2} w(φ_a) (Def. 3.2.1) — admissible iff v ≤ᵢ Γ_D(v); complete iff v = Γ_D(v); preferred iff ≤ᵢ-maximal admissible; grounded = least fixpoint *(p.79)*.
- Stable labeling defined via reduct D^E (Def. 3.2.3) where C_e^E = φ_e[b/f : b ∉ E]; M is stable iff M = gv^t of D^M (Def. 3.2.4) *(p.80)*.
- Theorem 3.2.5: every preferred labeling is complete; grounded is ≤ᵢ-least complete; complete labelings form a complete meet-semilattice w.r.t. ≤ᵢ — same shape as Dung *(p.80)*.
- Polberg's classification (xy-) prefix: x = "inside" (in/on the extension), y = "outside" (validity of attackers), each ∈ {a, c} (acyclic vs cyclic permitted). Yields four families: cc, aa, ac, ca; ca splits further into ca₁ (Pol14a — discard cyclic args independently of origin) and ca₂ (Pol15 — exception when cycle follows from arguments already accepted) *(p.81)*.
- Conflict-free + pd-acyclic conflict-free are equally treated: cf requires all Cs(E ∩ par(s)) = in for s ∈ E; pd-acyclic cf additionally requires every a ∈ E to have an unblocked acyclic pd-evaluation on E (Def. 3.1.12) *(p.82)*.
- Theorem 3.3.2: E is pd-acyclic conflict-free iff E is the grounded extension of the reduct D^E. Theorem 3.3.3: a model E is stable iff it is pd-acyclic cf in D — so stability does not require a separate reduct-based definition once pd-acyclicity is in hand *(p.84)*.
- Two grounded semantics: standard `grd` (Def./Prop. 3.3.7 — repeat decisive in/out until fixpoint) and `acy-grd` (Def. 3.3.8 — falsify b iff all of its acyclic pd-evaluations are blocked) *(p.85)*. Grounded falls in ac-family (acyclic inside, cyclic outside in rejection); acyclic grounded falls in aa-family.
- The full extension-based semantic family is: cf, acy-cf, mod, stb, grd, acy-grd, cc-{adm,pref,comp}, aa-{adm,pref,comp}, ac-{adm,pref,comp}, ca₁-{adm,pref,comp}, ca₂-{adm,pref,comp} *(p.81–93)*.
- Subset relations among admissible types (Lemma 3.3.15): ac ⊂ cc, ac ⊂ aa, aa ⊂ ca₂, cc ⊂ ca₂, ca₂ ⊂ ca₁ (every) — but ca₁ ⊄ ca₂ in general *(p.93)*.
- Fundamental Lemma generalizes: cc/ac/aa version (Lemma 3.3.18); weaker CA₁ version (Lemma 3.3.19) where decisiveness w.r.t. v_E' is preserved but A \ E'^{a+} need not be; CA₂ version (Lemma 3.3.20) *(p.94)*.
- Theorem 3.3.21: cc, ac, aa-complete extensions form complete meet-semilattices w.r.t. ⊆; ca₁, ca₂-complete may not. Grounded extension is least ac/cc-complete; acyclic grounded is least aa-complete and a minimal (not least) ca₁/ca₂-complete *(p.95)*.
- Three-valued model corresponds to conflict-free set via v^t (Theorem 3.4.2); admissible labelings correspond to ca₂-admissible extensions but not in general to cc/ac/aa/ca₁-admissible extensions (Theorem 3.4.4) — this is Polberg's central justification for adding extension-based semantics to ADFs *(p.98)*.
- The thesis position: extension-based and labeling-based semantics for ADFs do not match in general; only the grounded extension and ca₂-style approach line up cleanly with labelings *(p.96, p.99)*.

## Definitions Introduced

### Definition 2.2.14 — d-grounded extension *(p.50)*
Let BF = (A, R, S) be a BAF and R' ⊆ R^ind a collection of indirect attacks in BF. The d-grounded extension of BF w.r.t. R' is the least fixpoint of the d-characteristic operator of BF w.r.t. R'.

### Definition 2.2.17 — (binary) AFN *(p.51)*
A (binary) abstract argumentation framework with necessities (AFN) is a tuple (A, R, N) where A is a set of arguments, R ⊆ A × A is the attack relation, and N ⊆ A × A is the necessity relation.

### Definition 2.2.18 — Extended necessity / extended attack *(p.51)*
Let FN = (A,R,N) be an AFN and a,b ∈ A. There is an extended necessity from a to b, denoted a N+ b, iff there is a sequence a₁ N … N aₙ (n ≥ 2) where a₁ = a and aₙ = b. There is an extended attack of b by a, denoted a R+ b, iff: (a) aRb, (b) ∃c∈A s.t. aRc N+ b, or (c) ∃c∈A s.t. cRb and c N+ a.

### Definition 2.2.19 — N-Cycle-Free / coherent / strongly coherent (binary AFN) *(p.51)*
a is N-Cycle-Free in FN iff not (a N+ a) and not (b N+ a ∧ b N+ b). S is coherent iff every s ∈ S is N-Cycle-Free and S is closed under N⁻¹. S is strongly coherent iff coherent and conflict-free w.r.t. R.

### Definition 2.2.20 — Binary AFN extension types *(p.51)*
Stable iff strongly coherent and for each a ∈ A \ E either ∃e ∈ E with eRa or bNa for some b ∈ A \ E. Admissible iff strongly coherent and if there is a ∈ E with bRa then for each coherent E' ⊆ A \ E with b ∈ E' there exist e ∈ E, c ∈ E' s.t. eRc. Preferred iff maximal admissible.

### Definition 2.2.21 — (set) AFN *(p.52)*
A (set) AFN is (A, R, N) with R ⊆ A × A the attack relation and N ⊆ (2^A \ ∅) × A the necessity relation.

### Definition 2.2.22 — Powerful in E *(p.53)*
a is powerful in E iff a ∈ E and there is a sequence a₀,…,aₖ ∈ E with aₖ = a, no B ⊆ A s.t. BNa₀, and for 1 ≤ i ≤ k: for each B ⊆ A with BNaᵢ, B ∩ {a₀,…,aᵢ₋₁} ≠ ∅.

### Definition 2.2.23 — coherent / strongly coherent (set AFN) *(p.53)*
E is coherent iff each a ∈ E is powerful in E. Strongly coherent iff coherent and conflict-free w.r.t. R.

### Definition 2.2.24 — defends / characteristic function (set AFN) *(p.53)*
E defends a iff E ∪ {a} coherent and for each b ∈ A with bRa, for each coherent C ⊆ A containing b, ∃e ∈ E, c ∈ C with eRc. F_FN(E) = {a | E defends a in FN}.

### Definition 2.2.25 — deactivated set *(p.53)*
E^+ = {a | ∃e ∈ E with eRa or there is B ⊆ A with BNa and E ∩ B = ∅}.

### Definition 2.2.26 — set AFN extension types *(p.53)*
Admissible (strongly coherent + defends all of E), preferred (maximal admissible), complete (admissible + contains all defended args), grounded (least fixpoint of F_FN), stable (complete + E^+ = A \ E).

### Definition 2.2.27 — discarded set *(p.54)*
E^att = {a | every coherent set containing a is attacked by E}.

### Definition 2.2.32 — Evidential Argumentation System (EAS) *(p.55)*
EAS = (A, R, E) with R ⊆ (2^A \ ∅) × A attack, E ⊆ (2^A \ ∅) × A support, and a special argument η ∈ A s.t. ∄(x,y) ∈ R with η ∈ x and ∄x with (x,η) ∈ R or (x,η) ∈ E.

### Definition 2.2.33 — evidential support / minimal e-support *(p.56)*
a has e-support from S ⊆ A iff a = η or there is a non-empty S' ⊆ S s.t. S'Ea and ∀x ∈ S', x has e-support from S \ {a}. Minimal iff no proper subset gives e-support.

### Definition 2.2.34 — evidential sequence *(p.56)*
For X ⊆ A, a sequence (a₀,…,aₙ) of distinct elements of X is an evidential sequence for a ∈ X iff aₙ = a, a₀ = η, and for n > 0 and i = 1..n there exists nonempty T ⊆ {a₀,…,aᵢ₋₁} with TEaᵢ.

### Definition 2.2.36 — e-supported attack / minimal *(p.56)*
S carries out an e-supported attack on a iff (S',a) ∈ R with S' ⊆ S and every s ∈ S' has e-support from S in ES. Minimal iff no proper subset does.

### Definition 2.2.37 — acceptable in EAS *(p.56)*
a is acceptable w.r.t. S iff (i) a is e-supported by S, and (ii) every minimal e-supported attack T on a is countered by S e-supporting an attack against a member of T.

### Definition 2.2.38 — EAS extension types *(p.56)*
self-supporting (every member is e-supported by S), conflict-free (no a ∈ S, S' ⊆ S with S'Ra), strongly self-supporting (both), admissible, preferred, complete, stable.

### Definition 2.2.39 — discarded set in EAS *(p.57)*
S^+ = {a | for every self-supporting C with a ∈ C, ∃ S' ⊆ S, c ∈ C with S'Rc}.

### Definition 2.2.41 — characteristic function of EAS *(p.57)*
F_ES(S) = {a | a acceptable w.r.t. S in ES}; grounded = least fixpoint for finitary EAS.

### Definition 2.3.1 — Signature of an AF semantics σ *(p.59)*
Σ_σ^AF = {σ(F) | F ∈ AF_U}.

### Definition 2.3.2 — Args_S, Pair_S *(p.59)*
Args_S = ⋃_{E∈S} E; Pair_S = {(a,b) | ∃E ∈ S s.t. {a,b} ⊆ E}.

### Definition 2.3.3 — Realizability properties *(p.59)*
For S ⊆ 2^U: downward closed (S = dcl(S)), incomparable (E ⊆ E' ⇒ E = E'), tight (∀E ∈ S, a ∈ Args_S: E ∪ {a} ∉ S ⇒ ∃s ∈ E with (a,s) ∉ Pair_S), adm-closed, com-closed.

### Definition 3.1.1 — Abstract Dialectical Framework *(p.66)*
ADF = (A, L, C) where A = arguments/nodes/statements, L ⊆ A × A links/edges, C = {C_a}_{a∈A} acceptance conditions; each C_a: 2^par(a) → {in, out}. par(a) = {p ∈ A | (p,a) ∈ L}.

### Definition 3.1.2 — Supporting / attacking link, BADF *(p.67)*
Link (r,s) is supporting iff no R⊆par(s) has C_s(R)=in and C_s(R∪{r})=out; attacking iff no R has C_s(R)=out and C_s(R∪{r})=in. ADF is bipolar (BADF) iff all links are supporting or attacking.

### Definition 3.1.3 — Completion *(p.68)*
v two-valued on E ⊆ A; v' on Z ⊇ E is a completion of v to Z iff ∀a ∈ E v(a)=v'(a). v' is an x-completion iff all of Z\E mapped to x ∈ {t,f}.

### Definition 3.1.4 — Decisive / decisively out / decisively in *(p.69)*
v decisive for s iff for any two completions v_par(s), v'_par(s) of v to E ∪ par(s), v_par(s)(C_s) = v'_par(s)(C_s). s is decisively in (out) iff v decisive and all completions evaluate C_s to in (out).

### Definition 3.1.5 — acc / reb *(p.70)*
acc(G,B) = {r ∈ A | G ⊆ A' ⊆ (A\B) ⇒ C_r(A' ∩ par(s)) = in}; reb(G,B) similar but … = out.

### Definition 3.1.6 — Positive dependency function *(p.70)*
On E, pd_E^D assigns each a ∈ E an interpretation v ∈ min_dec(in, a) with v^t ⊆ E, or N if none. Sound iff no argument maps to N.

### Definition 3.1.7 — Standard positive dependency evaluation *(p.71)*
For pd_E^D maximally sound on E ⊆ X, a standard pd-evaluation for e ∈ E is (F, B) with F ⊆ E s.t. e ∈ F, ∀a ∈ F pd_E^D(a)^t ⊆ F, and B = ⋃_{a ∈ F} pd_E^D(a)^f. F = pd-set, B = blocking set.

### Definition 3.1.8 — Partially acyclic pd-evaluation *(p.72, body of statement on p.73)*
A triple (F, (a₀,…,aₙ), B) for x ∈ E with F ∩ {a₀,…,aₙ} = ∅, sequence elements distinct from E, and (i) if sequence non-empty aₙ = x else x ∈ F; (ii) ∀i: pd_E^D(aᵢ)^t ⊆ F ∪ {a₀,…,aᵢ₋₁}, pd_E^D(a₀)^t ⊆ F; (iii) ∀a ∈ F pd_E^D(a)^t ⊆ F; (iv) ∀a ∈ F ∃b ∈ F with a ∈ pd_E^D(b). B = ⋃ pd-set blocking ∪ ⋃ pd-sequence blocking.

### Definition 3.1.9 — Acyclic pd-evaluation *(p.73)*
A partially acyclic eval (F, (a₀,…,aₙ), B) is acyclic iff F = ∅. Set E ⊆ A is pd-acyclic iff every a ∈ E has an acyclic pd-evaluation.

### Definition 3.1.10 — Blocked evaluation *(p.74)*
Two-valued v on subset of A blocks (F, (a₀,…,aₙ), B) iff ∃b ∈ B with v(b) = t or ∃x ∈ {a₀,…,aₙ} ∪ F with v(x) = f.

### Definition 3.1.11 — Minimal evaluations *(p.75)*
Minimal standard, minimal acyclic, minimal partially acyclic: minimality w.r.t. F⊆, B⊆ (and G' subsequence of G).

### Definition 3.1.12 — Conflict-free / pd-acyclic conflict-free extension *(p.75, restated p.82)*
E is conflict-free iff ∀s ∈ E, C_s(E ∩ par(s)) = in. E is pd-acyclic conflict-free iff additionally every a ∈ E has an unblocked acyclic pd-evaluation on E w.r.t. v^E.

### Definition 3.1.13 — Range (interpretation) *(p.75)*
Build v_E by (1) v_E(a)=t for a ∈ E; (2) for b ∈ A\M decisively out w.r.t. v_E set v_E(b)=f; (3) repeat. E^+ = v_E^f, E^Ran = E ∪ E^+.

### Definition 3.1.15 (Deprecated) — Original acyclic range *(p.76)*
Iterative algorithm parametrised on acyclic pd-evaluations being blocked; superseded by Lemma 3.1.16 showing single-iteration sufficiency.

### Definition 3.1.18 — Partially acyclic discarded set *(p.77)*
E^{p+} = {a ∈ A | no partially acyclic eval (F', G', B') for a with F' ⊆ E and B' ∩ E = ∅}.

### Definition 3.1.20 — Partially acyclic range interpretation *(p.77)*
v_E^p maps t to all and only E, f to all and only E^{p+}.

### Definition 3.2.1 — Three-valued characteristic operator *(p.79)*
Γ_D : V_A → V_A, Γ_D(v)(a) = ⊓_{w ∈ [v]_2} w(φ_a).

### Definition 3.2.2 — ADF labelings *(p.79)*
v is: three-valued model iff ∀a, v(a) ≠ u ⇒ v(a) = v(φ_a); admissible iff v ≤ᵢ Γ_D(v); complete iff v = Γ_D(v); preferred iff ≤ᵢ-maximal admissible; grounded iff ≤ᵢ-least fixpoint of Γ_D.

### Definition 3.2.3 — Reduct *(p.80)*
D^E = (E, L^E, C^E) with L^E = L ∩ (E×E) and for e ∈ E, C_e^E = φ_e[b/f : b ∉ E].

### Definition 3.2.4 — Stable model *(p.80)*
M ⊆ A is stable iff M = gv^t where gv = grounded model of D^M.

### Definition 3.3.1 — Model *(p.83)*
A conflict-free E is a model iff ∀s ∈ A, C_s(E ∩ par(s)) = in ⇒ s ∈ E.

### Definition 3.3.6 — Grounded model via Γ'_D *(p.85)*
Γ'_D(A,R) = (acc(A,R), reb(A,R)). E is grounded model iff for some E' ⊆ S, (E,E') is least fixpoint of Γ'_D.

### Definition 3.3.8 — Acyclic grounded extension *(p.85)*
v starts empty; t for decisively-in args; f for b s.t. all acyclic pd-evals blocked by v; iterate.

### Definition 3.3.9 — CC family *(p.87)*
cc-admissible: conflict-free + every e ∈ E decisively in w.r.t. standard range v_E. cc-complete: cc-admissible + every a decisively in w.r.t. v_E is in E. cc-preferred: maximal cc-admissible.

### Definition 3.3.10 — AA family *(p.88)*
aa-admissible: pd-acyclic conflict-free + every e ∈ E has an acyclic pd-evaluation on E whose blocking set is mapped to f by v_E^a (acyclic range). aa-complete, aa-preferred analogous.

### Definition 3.3.11 — AC family *(p.89)*
ac-admissible: pd-acyclic conflict-free + every e ∈ E has an acyclic pd-evaluation whose blocking set is mapped to f by the **standard** range v_E (i.e. acyclicity inside, full-range outside).

### Definition 3.3.12 — CA₁ / CA₂ family *(p.90–91)*
ca₁-admissible: conflict-free + every e ∈ E decisively in w.r.t. acyclic range v_E^a. ca₂-admissible: conflict-free + every e ∈ E decisively in w.r.t. partially acyclic range v_E^p. ca₁-complete: ca₁-admissible + every a ∈ A \ E^{a+} that is decisively in w.r.t. v_E^a is in E. ca₂-complete: ca₂-admissible + every a decisively in w.r.t. v_E^p is in E. ca₁/ca₂-preferred: maximal of respective admissible.

### Definition 3.4.1 — Correspondence *(p.96)*
Three-valued v over A and E ⊆ A correspond iff v^t = E.

## Theorems / Propositions / Lemmas

### Lemma 2.2.11 *(p.50)*
d-characteristic operator of a BAF is monotonic w.r.t. set inclusion.

### Lemma 2.2.12 *(p.50)*
If E ⊆ A is +conflict-free w.r.t. R', so is d-F_BF^R'(E).

### Lemma 2.2.13 *(p.50)*
For E +conflict-free w.r.t. R': E is d-admissible iff E ⊆ d-F_BF^R'(E); E is d-complete iff E = d-F_BF^R'(E).

### Proposition 2.2.15 *(p.50)*
The d-grounded extension is +conflict-free.

### Theorem 2.2.16 *(p.50)*
Every d-preferred is d-complete (not vice versa); d-grounded is least d-complete; every stable extension is d-preferred (not vice versa).

### Lemma 2.2.28 *(p.54)*
For strongly coherent E in AFN, E^att ⊆ E^+.

### Lemma 2.2.29 *(p.54)*
a is defended by E iff E ∪ {a} coherent and ∀b ∈ A with bRa, b ∈ E^att.

### Lemma 2.2.30 *(p.54)*
E ⊆ A is stable in AFN iff strongly coherent and E^att = A \ E.

### Theorem 2.2.31 *(p.54)*
AFN: grounded is least complete; preferred is maximal complete; every stable is preferred.

### Theorem 2.2.35 *(p.56)*
a is e-supported by X in EAS iff there exists evidential sequence for a on X ∪ {a} in ES.

### Lemma 2.2.40 *(p.57)*
S is stable in ES iff strongly self-supporting and A \ S = S^+.

### Lemma 2.2.42 *(p.57)*
Conflict-free S is admissible iff S ⊆ F_ES(S); complete iff S = F_ES(S).

### Lemma 2.2.43 *(p.57)*
Minimal e-support set is self-supporting.

### Lemma 2.2.44 *(p.58)*
Set carrying out a minimal e-supported attack is self-supporting.

### Lemma 2.2.45 *(p.58)*
Admissible ⇒ self-supporting.

### Lemma 2.2.46 — EAS Fundamental Lemma *(p.58)*
S admissible, x,y acceptable w.r.t. S. Then S ∪ {x} admissible and y acceptable w.r.t. S ∪ {x}.

### Lemma 2.2.47 *(p.58)*
S e-stable iff S = {a | a not e-support attacked by S and a e-supported by S}.

### Theorem 2.2.48 *(p.58)*
EAS: every stable is preferred; every preferred is complete; grounded is least complete.

### Theorem 2.3.4 *(p.61)*
Σ_cf^AF = {S ≠ ∅ | downward-closed and tight}; Σ_stable^AF = {S | incomparable and tight}; Σ_adm^AF = {S ≠ ∅ | adm-closed and contains ∅}; Σ_pref^AF = {S ≠ ∅ | incomparable and adm-closed}.

### Proposition 2.3.5 *(p.61)*
For each F ∈ AF_U, com(F) is non-empty, com-closed, and ⋂_{E∈com(F)} E ∈ com(F).

## Algorithms / Procedures

### Algorithm — Standard range computation *(p.76, Def. 3.1.13)*
1. Let M = E and set v_E(a) = t for a ∈ E.
2. For every b ∈ A \ M decisively out w.r.t. v_E, set v_E(b) = f and add b to M.
3. Repeat (2) until no new elements added.
4. E^+ = v_E^f, range E^Ran = E ∪ E^+.

### Algorithm — Acyclic range (deprecated form) *(p.76, Def. 3.1.15)*
1. M = E, v_E^a(a) = t for a ∈ M.
2. For b ∈ A \ M with every acyclic pd-eval blocked by v_E^a, set v_E^a(b) = f, add to M.
3. Repeat. (Lemma 3.1.16 shows the loop terminates after the first iteration in conflict-free cases.)

### Algorithm — Iterative grounded extension *(p.85, Prop. 3.3.7)*
1. Start with empty interpretation v.
2. For every a decisively in w.r.t. v, set v(a) = t.
3. For every b decisively out w.r.t. v, set v(b) = f.
4. Repeat until fixpoint. Grounded = v^t.

### Algorithm — Acyclic grounded extension *(p.85, Def. 3.3.8)*
1. v empty.
2. For every a decisively in w.r.t. v, set v(a) = t.
3. For every b s.t. all acyclic pd-evaluations of b blocked by v, set v(b) = f.
4. Repeat to fixpoint. Acyclic grounded = v^t.

### Algorithm — Stable model (via reduct) *(p.80)*
1. Take candidate M.
2. Build reduct D^M with C_e^M = φ_e[b/f : b ∉ M].
3. Compute grounded labeling gv of D^M.
4. M is stable iff M = gv^t.

### Algorithm — Stable extension (via pd-acyclic cf, Theorem 3.3.3) *(p.84)*
1. Find all models E.
2. For each, check if E is pd-acyclic conflict-free in D.
3. Stable iff yes.

## Semantics Defined / Compared

ADF extension-based semantics introduced/used in this chunk:
- **Conflict-free** (cf): C_s(E ∩ par(s)) = in for s ∈ E. *(p.75, p.82)*
- **Pd-acyclic conflict-free** (acy-cf): cf + every a ∈ E has unblocked acyclic pd-evaluation on E. *(p.75, p.82)*
- **Naive** (mentioned in Example 29): maximal cf. *(p.83)*
- **Pd-acyclic naive**: maximal pd-acyclic cf. *(p.83)*
- **Model** (mod): cf + ∀s ∈ A, C_s(E ∩ par(s)) = in ⇒ s ∈ E. *(Def. 3.3.1, p.83)*
- **Stable** (stb): model + pd-acyclic cf, equivalently strongly coherent + E^{a+} = A \ E. *(p.84)*
- **Grounded** (grd): least fixpoint of decisive-in/out iteration. ac-family. *(p.85)*
- **Acyclic grounded** (acy-grd): variant using acyclic evaluations. aa-family. *(p.85)*
- **cc-admissible / cc-complete / cc-preferred**: standard range, no acyclicity assumptions. *(Def. 3.3.9, p.87)*
- **aa-admissible / aa-complete / aa-preferred**: pd-acyclic cf + acyclic range, with evaluation-protection. *(Def. 3.3.10, p.88)*
- **ac-admissible / ac-complete / ac-preferred**: pd-acyclic cf + standard range protection of acyclic evaluations. *(Def. 3.3.11, p.89)*
- **ca₁-admissible / ca₁-complete / ca₁-preferred**: cf + decisively in w.r.t. acyclic range. *(Def. 3.3.12, p.90–91)*
- **ca₂-admissible / ca₂-complete / ca₂-preferred**: cf + decisively in w.r.t. partially acyclic range. *(Def. 3.3.12, p.91)*

ADF labeling-based semantics:
- Three-valued model (Def. 3.2.2)
- Admissible labeling (v ≤ᵢ Γ_D(v))
- Complete labeling (v = Γ_D(v))
- Preferred labeling (≤ᵢ-maximal admissible)
- Grounded labeling (≤ᵢ-least fixpoint of Γ_D)
- Stable model (via reduct)

Coincidence with AF semantics:
- For ADFs, the labeling-based admissible/complete/preferred do NOT in general coincide with extension-based admissible/complete/preferred — they coincide most closely with the ca₂ family (Theorems 3.4.4–3.4.6).
- Grounded: Theorem 3.4.7 shows v^t = E for the standard grounded extension; that's a clean correspondence.
- Acyclic grounded only corresponds via u-completion to a complete labeling, not the ≤ᵢ-least.

## Comparisons With Prior Work

- Binary AFNs (NR11) have a design flaw (Example 14) — extensions like {a,b} satisfy the formal definition but violate authors' intentions. Set AFNs (Nou13) supersede them; binary form is dropped going forward. *(p.52–53)*
- BAFs (CLS09): Their main drawback is that an argument can appear in an extension regardless of whether it is supported. EAS [PO14b, PO14a] addresses this by requiring trace-back to evidence η. *(p.55)*
- AFN N-cycle-freeness vs EAS evidential sequences: Both are formal versions of "support traces back" but with different group-vs-binary structure. *(p.55–56)*
- Realizability work on AFs (DDLW15, Dyr14, GLW16, DLSW16): only signatures for cf, stable, adm, preferred fully characterized; complete only has necessary conditions (Prop. 2.3.5). Preferred/semi-stable can be realized in any AF given enough auxiliary args (Theorem 2.3.6). For ADFs, the only realizability work is [P15]. *(p.59–61)*
- Original ADF semantics (BW10): of the original semantics only cf, model, grounded ("well-founded") are still used; the rest were superseded by labeling approaches (BES+13, Str13a). *(p.64)*
- Labeling semantics (BES+13, Str13a): elegant via three-valued operator, but uses information maximality (subset of three-valued ordering) rather than subset maximality, which differs in the presence of support; preferred labelings are incomparable but their accepted sets may be in subset relation. *(p.64)*
- Set-AFN versus EAS: AFN/EAS prevent cyclic arguments; Polberg's ADF semantics permit a *user-controlled* choice of whether cycles are allowed inside or outside (the xy classification). *(p.65)*
- Original BW10 stable definition: not adequate, addressed in BES+13 via grounded reduct. Polberg's Theorem 3.3.3 simplifies further: stability ≡ model + pd-acyclic cf. *(p.84)*
- Pol14a vs Pol15: ca₁ (Pol14a) discards cyclic args independently of origin; ca₂ (Pol15) makes exception for cycles arising from already-accepted args. *(p.81, p.90)*
- Originals BW10 grounded: defined via acc/reb; same concept reformulated via decisiveness in Prop. 3.3.7 (cleaner). *(p.85)*
- DT09, JV99a "argumentation as dialogue": cited as prior dialogue-based view of acceptability semantics, providing motivation for the inside/outside classification. *(p.81)*

## Design Rationale

- **Why multiple semantics, not one?** Polberg argues ADFs should be "toolboxes" — the framework should provide many semantics and let the user choose, not bake in cyclicity assumptions. Cites MP10 for prior similar argument. *(p.65)*
- **Why both acyclic and cyclic?** Use case dependent (legal reasoning prosecutor vs jury, gaming forums vs medical), and cycles can sometimes be eliminated by simulating with self-attacking proxies (a' attacks a and is attacked by b, etc.) — so semantics with vs without cycles can be inter-translated. *(p.65)*
- **Positive dependency vs support links:** Polberg uses "positive dependency" rather than "support" for ADF cycles to avoid confusion with BADF supporting links and to remain neutral on which interpretation of support is intended. *(p.70)*
- **Decisiveness via interpretations vs sets:** Both representations are kept (set: acc/reb in BW10 style; interpretation: min_dec) — interpretation form is more convenient, set form is more natural for grounded. *(p.69–70)*
- **Three flavors of evaluation (standard, partially acyclic, acyclic):** Allow finer-grained control. Standard easiest; acyclic strictest; partially acyclic is the bridge needed for ADF→AF/SETAF translations (Translations 85, 86, 87). *(p.71)*
- **Extension-based semantics added because:** (1) labeling-based info-maximality yields incomparable preferred labelings with comparable t-sets, but extension-based subset maximality is more natural for "accept as much as possible"; (2) labelings allow self-justification via dependency cycles, which is undesirable when the framework expresses support — extensions can prevent this. *(p.64)*
- **Why split ca into ca₁ and ca₂:** ca₁ is Pol14a's original; ca₂ is Pol15's refinement that doesn't penalize cycles already in the accepted set. The two yield genuinely different semantics (Examples 36, 37). *(p.90–91)*
- **Range types deprecation note:** Original definition of acyclic range (Def. 3.1.15) uses iteration but Lemma 3.1.16 shows iteration terminates after one step — the iterative form is "Deprecated" but kept for clarity. *(p.76)*
- **Minimality important:** Allowing non-minimal decisively-in interpretations introduces "fake cycles" (Ex 25); minimality is required to avoid spurious dependency cycles. *(p.74–75)*
- **Why pd-acyclic conflict-free instead of just conflict-free for stable:** Theorem 3.3.3 shows stable is exactly model + pd-acyclic cf — bypasses BW10's reduct-based definition. *(p.84)*

## Limitations

- "We do not prove all properties" — Polberg notes that the property study in §3.3.5 is not exhaustive; full proofs in [Pol15]. *(p.92)*
- ca₁/ca₂-complete extensions do not form complete meet-semilattices (Prop. 3.3.22), so the lattice-theoretic results from Dung do not transfer to all families. *(p.95)*
- For ca₁-preferred, ca₂-preferred, no perfect correspondence with labelings exists in either direction (Theorem 3.4.5). Subset-maximality vs information-precision are fundamentally different. *(p.99)*
- Realizability for complete semantics in AFs is still an open problem (DDLW15) — the necessary conditions are known but not sufficient. *(p.59)*
- For ADFs the only realizability work is single-semantics [P15]; multi-dimensional realizability is open. *(p.59)*
- The "deprecated" original acyclic range definition is preserved alongside the simplified version because Translation 83 distinguishes evaluation types — minimality semantics depend on the pd-function. *(p.75–76)*

## Implementation Notes

- **ADF data structure:** `D = (A, C)` with C: a ↦ (Boolean function over par(a) returning {in, out}). The propositional formula form (φ_a) is the natural Python representation; can use Z3 / sympy / PyEDA for the satisfiability checks behind decisiveness.
- **Decisiveness check:** For interpretation v on E, "v decisive for s" requires checking that all completions of v to E ∪ par(s) yield identical C_s value. Naive: enumerate 2^|par(s)\E| completions. SAT/QBF can compress.
- **Acceptance condition encoding:** propstore can encode C_a as either (a) a propositional formula over par(a), (b) a function 2^par(a) → bool, (c) a truth table. Form (a) is most compact and integrates with existing CEL+Z3 pipeline.
- **Three-valued interpretation:** map each argument to t/f/u; precision ordering is u≤_i t and u≤_i f. Implement as a partial dict {arg: bool} where missing keys = u.
- **Characteristic operator Γ_D:** for each argument a, evaluate φ_a over all two-valued completions [v]_2 of the partial v, take ⊓ (intersection of values).
- **Reduct construction:** Given v ∈ admissible labeling and E = v^t, reduct D^E = (E, L^E, C^E) with C_e^E = φ_e[b/f for b ∉ E]. Easy formula substitution.
- **Positive dependency function pd_E^D:** essentially a witness function — for each a ∈ E, store one minimal decisively-in interpretation. Use SAT to find minimum models.
- **pd-evaluations as triples (F, sequence, B):** matches existing AFN/EAS sequence-based abstractions in propstore. Three-tuple data type.
- **Range computation:** Iterative algorithm (Def. 3.1.13) is cheap if decisiveness is cached. Lemma 3.1.16 says acyclic-range loop terminates after one round, big optimization.
- **Twelve+ semantics families to implement:** cf, acy-cf, mod, stb, grd, acy-grd, cc-{adm,comp,pref}, aa-{adm,comp,pref}, ac-{adm,comp,pref}, ca₁-{adm,comp,pref}, ca₂-{adm,comp,pref}. Plus labeling-based: adm-lab, comp-lab, pref-lab, grd-lab, stable-lab.
- **Subset hierarchy (Lemma 3.3.15) provides shortcuts:** if computing aa-admissible, also recover ca₂-admissible cheaply (every aa-admissible is ca₂-admissible).
- **Translation pipeline:** translations 83, 85, 86, 87 (forward references in this chunk) build ADF→AF/SETAF reductions using partially acyclic evaluations. Implementation will need to materialize evaluations as graph edges.
- **Labeling vs extension correspondence (Theorems 3.4.4–3.4.6):** If propstore wants to expose both labeling and extension queries for the same ADF, it must know the correspondence is partial — labelings best correspond to ca₂; for ca₁ there can be no labeling counterpart. UI should not pretend they're equivalent.
- **Stable extension via grounded reduct vs pd-acyclic cf model (Theorem 3.3.3):** the latter avoids reduct construction; just check (a) E is a model and (b) E is pd-acyclic conflict-free. Can be cheaper.
- **Grounded extension fixpoint procedure (Prop. 3.3.7):** parallels the Dung iterative grounded; reuse propstore's existing grounded computation harness, swapping decisiveness for "is unattacked".
- **BADF (bipolar) detection:** check each link individually whether it is supporting or attacking using Def. 3.1.2; redundant links (both) and "neither" links coexist in non-bipolar ADFs. propstore's link-classification subsystem must support all four cases.
- **Realizability hooks:** Theorem 2.3.4 gives concrete signature characterizations — useful for input validation when a user supplies an extension set and asks "is there an AF realizing exactly this?"
