## Key Findings

- Figure 3.13 maps inclusions between extension-based and labeling-based ADF semantics (CF, ACY-CF, 3MOD, ADM, CMP, PRF, GRD families) — labelings sit in the center; extension types (AA/AC/CC/CA1/CA2) are weaker source nodes feeding into the labeling node, with several types feeding back. *(p.100)*
- AADF⁺ is the subclass where the seven ADF semantics families collapse into a single notion; many ADFs derived from translations turn out to be AADF⁺. *(p.101–p.103)*
- Falsum arguments (acceptance condition `⊥`) are conceptually distinct from self-attackers in Dung style: a falsum argument is "I do not exist" rather than "undecided." Translations from AFNs/EAFs/EASs to ADFs can produce undesirable falsum arguments without preventive structure. *(p.104, p.105)*
- The bypass method introduces an auxiliary `b'` that supports `a` instead of the inconsistent `b`, used to translate AFN/EAF where one argument both supports and attacks the same target. *(p.107, p.108)*
- New ADF complexity results are populated into Table 3.5 (extension-based xy-semantics rows). xy-extension semantics are usually one PH-level harder than labeling-based ones because labelings already store `f` info. *(p.121–p.122)*
- Dependence on `ca₁` family of admissibility/preferred causes results to jump up to Σ₃ᴾ / Π₃ᴾ / Π₄ᴾ levels (vs Σ₂ᴾ / Π₃ᴾ for cc/aa/ac/ca₂). *(p.117, p.119, p.120)*
- The thesis builds an "almost ninety methods" compendium of intertranslations between {AF, SETAF, AFRA, EAS, BAF, AFN, EAFC, EAF, ADF}. *(p.125, Table 4.1 p.126)*
- Translations are classified along functional / syntactical / semantical / computational properties; faithfulness has multiple non-equivalent formal definitions. *(p.133–p.139, citing DW11, Jan99, Lib14, Got95)*
- Four high-level translation strategies are introduced: basic, coalition, attack-propagation, defender. *(p.143–p.145, Table 4.2 p.145)*
- The ADF redundancy-free form parallels the SETAF/EAFC/AFN/EAS minimal normal form. *(p.147)*

## Definitions Introduced

### Definition 3.5.1 — AADF⁺ (positive dependency acyclic ADF) *(p.102)*
> Let D = (A, C) be an ADF. D is an **AADF⁺** iff for every standard evaluation (F, B) of D and the pd-function pd^D it was created with, we can construct an acyclic pd-evaluation ((a₀, ..., aₙ), B) based on pd^D s.t. F = {a₀, ..., aₙ}.

### Definition 3.7.1 — Polynomial hierarchy classes *(p.110)*
> Let Σ₀ᴾ = Π₀ᴾ = Δ₀ᴾ = P, where P is the set of decision problems solvable in polynomial time. For i ≥ 1:
> - Δᵢᴾ = P^{Σᵢ₋₁ᴾ}
> - Σᵢᴾ = NP^{Σᵢ₋₁ᴾ}
> - Πᵢᴾ = coNP^{Σᵢ₋₁ᴾ}
>
> Pᴬ is decision problems solvable in polynomial time with access to an A-oracle. Dᴾ = L₁ ∩ L₂ where L₁ ∈ NP, L₂ ∈ coNP (SAT-UNSAT is Dᴾ-complete). Dᵢᴾ for i ≥ 1: L = L₁ ∩ L₂ with L₁ ∈ Σᵢᴾ and L₂ ∈ Πᵢᴾ. D₁ᴾ = Dᴾ.

### Definition 3.7.2 — Weak positive dependency function & weak X pd-evaluation *(p.111)*
> Let D = (A, C) be an ADF and E ⊆ A. A **weak positive dependency function** on E is a function `wpd` assigning every argument a ∈ E an interpretation v ∈ dec(in, a) or 𝒩 for null iff no such interpretation can be found. A **weak X pd-evaluation** for X ∈ {standard, partially acyclic, acyclic} is an X pd-evaluation constructed using `wpd`.

### Definition 4.1.1 — Framework translation *(p.127)*
> Let T, T' be two distinct types of abstract argumentation frameworks, U^T, U^{T'} two domains of arguments and Fr^T, Fr^{T'} the collections of all frameworks of type T and T'. A **framework translation** is a function Tr : D^T → D^{T'}, where D^T ⊆ Fr^T and D^{T'} ⊆ Fr^{T'}.

### Definition 4.1.2 — Similarity relation between semantics ~ *(p.128)*
Symmetric (not transitive) relation σ_T ~ σ_{T'}. Among the rules:
- For T, T' ∈ {AF, SETAF, AFRA, AFN, EAS, EAF, EAFC} and σ_T, σ'_{T'} ∈ {conflict-free, admissible, complete, preferred, grounded, stable}, σ_T ~ σ'_{T'} iff σ_T = σ'_{T'}.
- For T ∈ {AF, SETAF, AFRA}: cf_T ~ str-coh_AFN, cf_T ~ cf-sup_EAS.
- str-ssup_EAS ~ str-coh_AFN.
- For T ∈ {AF, SETAF, AFRA}: cf_T ~ cf_ADF and cf_T ~ acy-cf_ADF; stb_T ~ stb_ADF and stb_T ~ mod_ADF; grd_T ~ grd_ADF and grd_T ~ acy-grd_ADF; for σ_T ∈ {admissible, complete, preferred} and x, y ∈ {a, c}, σ_T ~ xy-σ_ADF.
- For T ∈ {AFN, EAS}: coh_AFN ~ pdc_ADF and ssup_EAS ~ pdc_ADF; str-coh_AFN ~ acy-cf_ADF and str-ssup_EAS ~ acy-cf_ADF; grd_AFN ~ acy-grd_ADF and grd_EAS ~ acy-grd_ADF; stb_AFN ~ stb_ADF and stb_EAS ~ stb_ADF; for σ_T ∈ {admissible, complete, preferred}, σ_T ~ aa-σ_ADF.
- For T ∈ {EAF, EAFC}: cf_T ~ cf_ADF, stb_T ~ mod_ADF, grd_T ~ acy-grd_ADF, and for σ_T ∈ {admissible, complete, preferred}, σ_T ~ ca₂-σ_ADF.
*(p.128)*

### Definition 4.1.3 — Singular & collective semantics casting functions *(p.129)*
> Singular semantics casting function: SC^X_{σ^T} : σ^{T'}(Tr(X)) → σ^T(X) mapping target answers to source answers.
> Collective semantics casting function: CSC^X_{σ^T} : {σ^{T'}(Tr(X))} → {σ^T(X)} mapping the set of all target answers to a set of source answers.

### Definition 4.1.4 — Translation strength *(p.130)*
> Let SC^{Tr}_{σ^T} be the singular semantics casting functions for D^T. Tr is:
> - **⊆-weak** under (σ, SC^{Tr}_{σ^T}) if for every X ∈ D^T, σ^T(X) ⊆ SC^X_{σ^T}[σ^{T'}(Tr(X))].
> - **⊇-weak** under (σ, SC^{Tr}_{σ^T}) if σ^T(X) ⊇ SC^X_{σ^T}[σ^{T'}(Tr(X))].
> - **strong** under (σ, SC^{Tr}_{σ^T}) if σ^T(X) = SC^X_{σ^T}[σ^{T'}(Tr(X))].

### Definition 4.1.5 — Sub relation ⊑ *(p.131)*
> For two sets A and B, A ⊑ B iff A ⊆ B. For two labelings v, v' on sets A, A' where A ⊆ A', v ⊑ v' iff ∀a ∈ A, v(a) = v'(a).

### Definition 4.1.6 — Casting types: identity / removal / addition / union *(p.131–p.132)*
> Let SC^X_{σ^T} be a semantics casting function. The function is:
> - **identity** if U_{σ^T} = U_{σ^{T'}} and for any E ∈ σ^{T'}(Tr(X)), SC^X_{σ^T}(E) = E.
> - **removal** if U_{σ^T} ⊆ U_{σ^{T'}} and for any E ∈ σ^{T'}(Tr(X)), SC^X_{σ^T}(E) ⊑ E.
> - **addition** if U_{σ^{T'}} ⊆ U_{σ^T} and for any E ∈ σ^{T'}(Tr(X)), E ⊑ SC^X_{σ^T}(E).
> - **union** if U_{σ^{T'}} = 2^{U_{σ^T}} and for any E ∈ σ^{T'}(Tr(X)), SC^X_{σ^T}(E) = ⋃E.

### Definition 4.1.7 — Functional properties of translations *(p.134)*
- **full** if D^T = Fr^T.
- **partial / source-subclass** if D^T ⊆ Fr^T.
- **surjective** if D^{T'} = Fr^{T'}.
- **target-subclass** if D^{T'} ⊆ Fr^{T'}.
- **injective** if for all X, X' ∈ D^T, Tr(X) = Tr(X') → X = X'.
- **overlapping** if there exist X, X' ∈ D^T s.t. X ≠ X' and Tr(X) = Tr(X').

### Definition 4.1.8 — Syntactical properties *(p.134, p.135)*
- **argument domain preserving** if U^T = U^{T'}.
- **argument domain altering** if U^T ≠ U^{T'}.
- **weakly domain altering** if U^T ⊆ U^{T'} or U^T ⊇ U^{T'} and elements of same type.
- **argument introducing** if not every Tr(X) argument represents a source argument or some source arguments map to multiple target arguments.
- **argument removing** if some source argument has no representative in Tr(X).
- **relation introducing** / **relation removing** (analogous for relations).
- **induced relation removing** / **induced relation adding**.
- **structure preserving** if no elements removed or added.

### Definition 4.1.9 — Semantic properties of translations *(p.140)*
> Tr is:
> - **generic** if Tr and SC can be applied to three or more semantics in a strong manner.
> - **specialized** if applied to two or fewer semantics in a strong manner.
> - **semantics domain preserving** for σ^T if U_{σ^T} = U_{σ^{T'}}.
> - **weakly semantics domain altering** for σ^T if U_{σ^T} ⊆ U_{σ^{T'}} or ⊇.
> - **semantics domain altering** for σ^T if U_{σ^T} ≠ U_{σ^{T'}}.
> - **semantics bijective** under (σ^T, SC^{Tr}_{σ^T}) iff strong under (σ^T, SC^{Tr}_{σ^T}) and every casting function in SC^{Tr}_{σ^T} is bijective.
> - **faithful** under (σ^T, SC^{Tr}_{σ^T}) iff semantics bijective and the casting functions are removals or additions.
> - **exact** under (σ^T, SC^{Tr}_{σ^T}) iff semantics bijective and casting functions are identities.

### Definition 4.1.10 — ⊗ and ⊕ operators on propositional ADFs *(p.140)*
> Let D₁ = (A₁, L₁, C₁), D₂ = (A₂, L₂, C₂) be propositional ADFs with C_i = {φ_i^a}_{a ∈ A_i}. Let A = A₁ ∪ A₂, L = L₁ ∪ L₂. D₁ ⊗ D₂ = (A, L, C^⊗) and D₁ ⊕ D₂ = (A, L, C^⊕):
>
> $$C_1 \otimes_s C_2 = \begin{cases} \varphi_s^1 \wedge \varphi_s^2 & s \in A_1 \cap A_2 \\ \varphi_s^1 & s \in A_1 \setminus A_2 \\ \varphi_s^2 & \text{otherwise} \end{cases}$$
>
> $$C_1 \oplus_s C_2 = \begin{cases} \varphi_s^1 \vee \varphi_s^2 & s \in A_1 \cap A_2 \\ \varphi_s^1 & s \in A_1 \setminus A_2 \\ \varphi_s^2 & \text{otherwise} \end{cases}$$
>
> If A₁ ∩ A₂ = ∅, D₁ ⊗ D₂ = D₁ ⊕ D₂.

### Definition 4.1.11 — ⊗ and ⊕ operators on functional ADFs *(p.140)*
Joint parent set:
> $$jpar(s) = \begin{cases} par_1(s) \cup par_2(s) & s \in A_1 \cap A_2 \\ par_1(s) & s \in A_1 \setminus A_2 \\ par_2(s) & \text{otherwise} \end{cases}$$
>
> For X ⊆ jpar(s):
> $$C_1 \otimes_s C_2(X) = \begin{cases} in & C_s^i(X \cap par_i(s)) = in \text{ for all } i \in \{1,2\}, s \in A_1 \cap A_2 \\ out & C_s^i(X \cap par_i(s)) = out \text{ for any } i, s \in A_1 \cap A_2 \\ C_s^1(X) & s \in A_1 \setminus A_2 \\ C_s^2(X) & \text{otherwise} \end{cases}$$
>
> ⊕ is dual: returns out if all components return out and in if any returns in.

### Definition 4.1.12 — Computational properties of translations *(p.142)*
> Tr is:
> - **modular** if T, T' ≠ ADF and for every X, X' ∈ D^T s.t. X ∪ X' ∈ D^T, Tr(X) ∪ Tr(X') = Tr(X ∪ X').
> - **δ-modular** for δ ∈ {⊗, ⊕}: T ≠ ADF, T' = ADF: Tr(X) δ Tr(X') = Tr(X ∪ X'); or T, T' = ADF: Tr(X) δ Tr(X') = Tr(X δ X'); or T = ADF, T' ≠ ADF: Tr(X) ∪ Tr(X') = Tr(X δ X').
> - **polynomial / polytime** w.r.t. fs if running time is polynomial in fs size.
> - **polysize** w.r.t. fs if size of Tr(X) is polynomially bounded in size of X.
> - **semantical / structural / semi-structural** based on whether Tr depends on semantical notions of source or target.

### Definition 4.2.1 — Subframework / full subframework (Dung's AF) *(p.146)*
> G_F = (A', R') is a **subframework** of F = (A, R), denoted G_F ⊑ F, if A' ⊆ A and R' ⊆ R ∩ (A' × A'). G_F is a **full subframework** induced by A' ⊆ A if R' = R ∩ (A' × A').

### Definition 4.2.2 — Trimmed subframework / full trimmed subframework (AFN) *(p.147)*
> G_FN = (A', R', N') is a **trimmed subframework** of FN = (A, R, N), denoted G_FN ⊑^T FN, if A' ⊆ A, R' ⊆ R ∩ (A' × A'), and N' ⊆ {(C', a) | a ∈ A', C' ≠ ∅, ∃(C, a) ∈ N ∧ C' ⊆ C ∩ A'}. G_FN is a **full trimmed subframework** induced by A' ⊆ A if R' = R ∩ (A' × A') and N' = {(C', a) | a ∈ A', C' ≠ ∅, ∃(C, a) ∈ N ∧ C' = C ∩ A'}.

### Definition 4.2.3 — Reduct (functional ADF restatement of Def. 3.2.3) *(p.147)*
> Let D = (A, L, C) be an ADF and E ⊆ A. The **reduct** of D w.r.t. E is D^E = (E, L^E, C^E), with L^E = L ∩ (E × E) and for e ∈ E, C_e^E = C_e ∩ (2^E × {in, out}).

### Definition 4.2.4 — SETAF minimal form *(p.148)*
> SF^{min} = (A, R') is the **minimal form** of SF = (A, R), denoted SF^{min} ⊑^{min} SF, iff R' ⊆ R consists of all and only elements (T, a) in R s.t. ∄T' ⊊ T, (T', a) ∈ R.

> - φ_{d̄_i} = ¬f for d̄_i ∈ D̄
> - φ_{y_i} = ¬d_i ∧ ¬ȳ_i for y_i ∈ Y
> - φ_{ȳ_i} = ¬d̄_i ∧ ¬y_i for ȳ_i ∈ Ȳ
> - φ_{z_i} = ¬z_i for z_i ∈ Z
> - φ_f = ¬f ∨ ψ
>
> is an AADF⁺.

## Examples Worked

### Example 40 — Sample ADF tables 3.2 and 3.3 *(p.101)*
ADF ({a,b,c,d}, {C_a = ¬c, C_b = ¬d, C_c = c, C_d = d}). Lists of admissible/complete/preferred extensions and labelings. Demonstrates that every admissible labeling produces ca₁/ca₂-admissible extension and vice versa, but not for aa/cc/ac. Some complete labelings produce sets {a} or {b} that are not complete extensions of any type. The ac-preferred extension ∅ has no corresponding preferred labeling.

### Example 41 — AADF⁺ vs not-AADF⁺ *(p.102, p.103)*
D₁ = ({a,b,c,d,e}, {C_a = ⊤, C_b = ¬a ∨ c, C_c = b, C_d = ¬c ∧ ¬e, C_e = ¬d}) is NOT an AADF⁺ because v_b = {c:t} and v_c = {b:t} produce a standard evaluation ({b,c}, ∅) that cannot be ordered to satisfy pd-sequence requirements. The frameworks in Figures 3.12 and 3.14 (printed pages 86 and 88) are also not AADF⁺. Modification D₂ = ({a,b,c,d,e}, {C_a = ⊤, C_b = a ∨ ¬c, C_c = b, C_d = ¬c ∧ ¬e, C_e = ¬d}) IS an AADF⁺ — minimal decisively-in interpretations satisfy a₀ requirements with ordering a < b < c.

### Example 42 — AADF⁺ ∩ BADF relations *(p.104)*
Three frameworks illustrate that AADF⁺ and BADF subclasses overlap but neither subsumes the other. ({a,b,c}, {C_a=⊤, C_b=⊤, C_c=(a∨b)∧(¬a∨¬b)}) is AADF⁺ but not BADF (xor on a,b). ({a}, {C_a=a}) is BADF (self-supporting) but not AADF⁺ (only minimal decisively-in interpretation v_a = {a:f} cannot create acyclic pd-evaluation). Pure Dung-style ADFs ({a,b,c}, {C_a=⊤, C_b=¬a, C_c=¬b}) are both BADF and AADF⁺.

### Example 43 — AFN with self-attacking-and-supporting falsum behavior *(p.105)*
AFN FN₁ = ({a,b}, {(b,a)}, {({b}, a)}) where a is supported and attacked by b. ADF analog produces falsum argument for a (b ∧ ¬b condition). Demonstrates conflict between AFN bipolar handling and ADF acceptance condition.

### Example 44 — AFN where supported argument is also attacker *(p.106)*
FN₂ = ({a,b,c,d}, {(a,d),(b,a),(d,c)}, {({b,c}, a)}). In ADF translation, only set {c} satisfies a's acceptance condition, treating b primarily as an attacker. {d} is ADF-admissible but not AFN-admissible.

### Example 45 — EAF showing translation gap *(p.106)*
EF₁ = ({a,b}, {(a,b)}, {(a,(a,b))}) admissible extensions ∅, {a}, {a,b}. ADF ({a,b}, {C_a = ⊤, C_b = ⊤}) gives ∅, {a}, {b}, {a,b} — extra extension. Modifying C_b = a fixes it but isn't long-term. EF₂ = ({a,b,c}, {(a,b),(c,a)}, {(a,(a,b))}) admissible ∅, {c}, {c,b}; ADF ({a,b,c}, {C_a=¬a, C_b=a, C_c=⊤}) gives ∅, {c} only.

### Example 46 — Bypass method on AFN FN₁ *(p.106, p.107)*
Introducing bypass argument b' for b. New AFN ({a,b,b'}, {(b,a)}, {({b}, b'), ({b'}, a)}). Corresponding ADF D₁ = ({a,b,b'}, {C_a = ¬b ∧ b', C_b = ⊤, C_{b'} = b}) no longer turns a into a falsum argument.

### Example 47 — Bypass on FN₂ *(p.107)*
D₂ = ({a,b,c,d,b'}, {C_a = ¬b ∧ (b' ∨ c), C_b = ⊤, C_c = ¬d, C_d = ¬a, C_{b'} = b}). {d} no longer admissible — its range falsifies c only and cannot prevent acceptance of b and b', therefore not of a.

### Example 48 — Bypass for EAF *(p.107)*
Bypass ADF D₃ = ({a,b,c,a^b}, {C_a = ¬c, C_b = ¬a ∨ a^b, C_c = ⊤, C_{a^b} = a}). Admissible extensions ∅, {c}, {b,c} agree with EAF. For original EAF, bypass ADF D₄ = ({a,b,a^b}, {C_a = ⊤, C_b = ¬a ∨ a^b, C_{a^b} = a}) gives ∅, {a}, {a,a^b}, {a,a^b,b} which after removing a^b yields the desired collection.

### Example 49 — Casting function types *(p.132)*
- AF→SETAF (Translation 17): identity casting; same admissible extensions ∅, {a}, {b}, {a,c}.
- AF→AFRA (Translation 18): removal casting, AFRA extensions add attack labels which must be removed.
- AFN→AF coalition (Translation 61): union casting (target arguments are sets of source arguments).

### Example 50 — SETAF minimal form *(p.149)*
SF = ({a,b,c,d,e}, {({a,b},c), ({a,d},c), ({a,b,d},c), ({e},d), ({c,e},b)}); SF^{min} = ({a,b,c,d,e}, {({a,b},c), ({a,d},c), ({e},d), ({c,e},b)}). The set {a,b,d} attack on c is removed because it is not minimal.

## Algorithms / Procedures

### Translation 1 — SETAF Minimal Form *(p.148)*
> Let Fr^{SETAF} be the collection of all SETAFs and Min^{SETAF} the collection of those SETAFs in minimal normal form. The minimal form translation min-Tr^{SETAF}: Fr^{SETAF} → Min^{SETAF} is defined as min-Tr^{SETAF}((A, R)) = (A, R') where R' ⊆ R consists of all and only elements (T, a) in R s.t. ∄T' ⊊ T, (T', a) ∈ R.

**Properties (Analysis of Translation 1, p.148, p.149):**
- full, target-subclass, overlapping
- argument domain preserving, attack relation removing
- generic, semantics domain preserving, exact
- structural
- NOT modular (counterexample: union of minimal subframes can fall outside Min^{SETAF})

### Bypass method *(p.106, p.107)*
1. Identify argument b that directly supports and attacks the same argument a in source AFN/EAF.
2. Introduce auxiliary bypass argument b'.
3. Set b' to be supported by b (`{b}` supports b' / C_{b'} = b).
4. Replace b in support relations of a with b' (so b only attacks a, while b' supports a).
5. Translate the resulting bipolar framework to ADF.

(Detailed elsewhere in §4.8.6 and §4.9.5 — chunk forward-references this.)

## Semantics Defined / Compared

ADF semantics families recapped/used heavily in §3.7:
- **Conflict-free (cf)** — all C_a evaluate to in under E itself.
- **Acyclic conflict-free (acy-cf)** — pd-acyclic conflict-free.
- **Model (MOD)** — propositional ADF model.
- **Stable (STB)** — extension surviving the reduct.
- **Grounded (GRD)** — least complete; iterative via Γ.
- **Acyclic Grounded (acy-grd)** — least aa-complete extension.
- **Admissible (ADM) families** — aa, ac, cc, ca₁, ca₂ + LAB (labeling).
- **Complete (CMP) families** — aa, ac, cc, ca₁, ca₂ + LAB.
- **Preferred (PRF) families** — aa, ac, cc, ca₁, ca₂ + LAB.

Coincidences for AADF⁺ subclass *(Theorem 3.5.2, p.103)*:
- Conflict-free ↔ pd-acyclic conflict-free.
- Models ↔ stable.
- aa-, cc-, ac-, ca₁-, ca₂- ADM/CMP/PRF all coincide.
- Grounded ↔ acyclic-grounded.

Casting/extension hierarchy for AFs ↔ ADFs (Definition 4.1.2):
- AF/SETAF/AFRA cf ↔ ADF cf and acy-cf.
- AF/SETAF/AFRA stb ↔ ADF stb and mod.
- AF/SETAF/AFRA grd ↔ ADF grd and acy-grd.
- AF/SETAF/AFRA admissibility/completeness/preferred ↔ all ADF xy- variants (x, y ∈ {a, c}).
- AFN/EAS coh ↔ ADF pdc; AFN/EAS str-coh / str-ssup ↔ ADF acy-cf; semantics ↔ aa-σ_ADF.
- EAF/EAFC cf ↔ ADF cf; semantics ↔ ca₂-σ_ADF.

## Design Rationale

- **Why introduce AADF⁺ as a subclass?** Many translation-derived ADFs turn out to be AADF⁺, which collapses semantics families and provides a tighter extension/labeling correspondence. Useful as a "sanity zone" for showing translations preserve semantics across all families. *(p.101, p.103)*
- **Why falsum arguments are not "errors":** They cleanly represent "this statement is known false" — a structural, not semantical, constraint. Other frameworks confuse this with self-attack (undecided) or invalidity (semantical). *(p.104, p.105)*
- **Why bypass method:** preserves semantics under bipolar→ADF translations where one argument plays both supporter and attacker roles, without contaminating the source semantics — auxiliary `b'` keeps the role at a single point. *(p.106)*
- **Why focus complexity study on extension-based xy-semantics, not labeling:** "Labelings store more information (in particular, concerning the f arguments), which the extension-based semantics need to reconstruct. Also xy-semantics are more specialized than labeling based ones and require evaluation analysis, not just decisiveness as in the characteristic operator." *(p.121)*
- **Why prefer singular casting over collective:** "The formula σ(X) = {SC(E) | E ∈ σ(Tr(X))} is closer to what we want the 'relation between extensions' to stand for." Collective allows addition/removal of whole extensions which is too permissive. *(p.130)*
- **Why translations may need access to source/target frameworks:** Casting an extension may require removing or adding arguments specific to that framework. *(p.130)*
- **Why distinguish polynomiality (running-time) from polysize (output-size):** Translations from support frameworks to SETAFs can produce exponentially many group attacks even though argument count grows linearly — running-time alone does not capture this. *(p.141)*
- **Why generic/specialized cutoff at three semantics:** "Matter of taste and observation" — translations in this thesis emerge from analyzing structural differences, not single-semantics tuning, and typically work for ≥3 of {complete, preferred, grounded, stable}. *(p.137)*
- **Why prefer union-based modularity:** For AFs and many extensions, the union of frameworks is just the union of all components; ADFs require special ⊗/⊕ operators because acceptance conditions don't union trivially. *(p.139, p.140)*
- **Why local minimality (not support-path minimality) for the SETAF minimal form:** Local rule is well-defined; path-based "argument supports both directly and indirectly" minimality left for future work. *(p.147, p.148)*
- **Why allow ⊑^T trimmed subframework variant for AFNs:** the standard subframework definition would drop a support set if any single supporter is missing; trimmed form lets you intersect the support set with the new argument set, which matches user intuition ("a is necessary for c"). *(p.146)*

## Implementation Notes

- **AADF⁺ check** is the single condition that, if it holds, lets a propstore implementer collapse every ADF semantics family into one. Detection algorithm: enumerate standard pd-evaluations and for each verify acyclic refinement existence. Polynomial blow-up if minimality is dropped.
- **Verification problem catalog (§3.7.2)** maps directly to a checker API surface for an ADF reasoner: `Ver_cf`, `Ver_grd`, `Ver_acy_grd`, `Ver_xy_adm`, `Ver_xy_cmp`, `Ver_xy_prf`, plus building blocks `Ver_dec^x`, `Ver_X-pd^w`, `Ver_X^{dis}`. propstore's argumentation layer can publish the same names for traceability against Polberg's complexity table.
- **Casting functions** (Defs 4.1.5, 4.1.6) are pure functions on extension sets — implementable as adapter strategies in a `RenderPolicy`. The `extraction` casting (mentioned p.132 as deferred to §4.10) is an additional case for ADF→AF translations and should be a separate strategy.
- **Translation strength API** corresponds to a property report dataclass: `{"strength": "strong"|"⊆-weak"|"⊇-weak", "type": "exact"|"faithful"|...}`.
- **Functional/Syntactical/Semantical/Computational** properties (Defs 4.1.7–4.1.12) form a 4-axis tag space for any translation registered in propstore: each translation should publish a tag bundle for downstream policy.
- **⊗ and ⊕ operators** (Def 4.1.10/4.1.11) are the right primitive for merging two ADFs. propstore.world's IC merge over ATMS bundle nodes can re-use these operators when merging acceptance conditions for shared atoms (currently propstore uses set-of-claims merge; ⊗/⊕ extend this to non-binary support).
- **AADF⁺ subclass** is desirable for instances coming out of any source-to-ADF translation: a propstore translator should automatically check membership before exposing reduced-semantics queries.
- **Falsum vs self-attacker distinction** matters for any LLM/heuristic claim-import pipeline — distinguish "we know this is false" (acceptance condition `⊥`) from "we cannot decide / it's contested" (self-attack pattern).
- **Bypass method** is the canonical workaround when importing claim graphs that contain "X both supports and attacks Y." A propstore vocabulary-reconciliation step should auto-introduce a bypass auxiliary node before pushing into the ATMS.
- **PH-class-aware caching:** since most ADF reasoning queries are at least Σ₂ᴾ-complete, a caching layer around `query_claim()` / `build_arguments_for()` is strongly motivated. Mark cache entries with their PH level so users understand cost.
- **Minimal normal form (Translation 1)** can be applied ahead of any SETAF/AFN solver to reduce attack count from up to 2ⁿ to (ⁿ choose ⌊n/2⌋) under Sperner. Recommend running it eagerly after vocabulary reconciliation.
- **Translation classification** (Table 4.2) should ship as metadata on every translation function: `Bas` = simple structural; `Cl` = coalition (uses sets-of-source-args); `AP` = simulates support via attack chains; `Def` = supporter→defender via auxiliary attackers; `Ch` = chained translation through intermediate framework. propstore can refuse to use `?`-classified ADF→AFN/EAS translations (currently unknown).
- **Modularity matters for parallel solvers:** modular translations let propstore split a corpus, translate halves in parallel, and union; non-modular ones (like SETAF minimal form) must run on the whole corpus.
