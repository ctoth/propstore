## Key formal content captured so far

### Definition 10.5 (Binomial Abduction) *(p.183)*
Binary domains X = {x, x̄}, Y = {y, ȳ}. Given conditionals (ω_{y|x}, ω_{y|x̄})
and an opinion ω_y, derive ω_{x‖̃y} via:
1. Invert (ω_{y|x}, ω_{y|x̄}) to (ω_{x|̃y}, ω_{x|̃ȳ}) using subjective
   binomial Bayes' theorem from §10.3.
2. Apply the inverted pair with ω_y in binomial conditional deduction
   (Eq. 10.31, §9.4).

### Eqs (10.29)-(10.31): Binomial Abduction *(p.183)*
- (10.29) ω_{x‖̃y} = ω_y ⊚̃ (ω_{y|x}, ω_{y|x̄}, a_x)   — abduction operator ⊚̃
- (10.30)          = ω_y ⊚ φ̃ (ω_{y|x}, ω_{y|x̄}, a_x) — split into Bayes
  inversion φ̃ then deduction ⊚
- (10.31)          = ω_y ⊚ (ω_{x|̃y}, ω_{x|̃ȳ})

## Implementation notes captured so far

- Abduction = Bayesian inversion (§10.3) composed with deduction (§9.4); both
  steps are already-defined operators, so the abduction implementation is the
  composition.
- The "unspecificity" opinion ω_{y|x̄} is the complement of the specificity
  ω_{ȳ|x̄}. Implementations must wire the negation operator at this point.
- Apex point uncertainty u_{x‖ŷ} = 0.95 in the convergence example confirms
  that abduction can produce near-vacuous output even when each conditional is
  relatively informative — base-rate fallacy materialises here.
- Table 10.1 monotone increase of u from 0.0 (initial) toward 0.4
  (convergence), paired with belief drift 0.80 → 0.60 — useful regression
  test target for an inversion implementation.
