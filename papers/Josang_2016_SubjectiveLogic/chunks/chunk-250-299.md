## Definitions and Notation

### Definition 12.9 (Consensus & Compromise Fusion / CCF) — PNG 253 / p.235
Symbol: ⓒ (circle-cc). Three-step process produces ω_X^(A♡B) = ω_X^A ⓒ ω_X^B. Steps: (1) consensus belief; (2) compromise redistribution of conflict; (3) merge. Properties: commutative, idempotent, semi-associative; vacuous opinion is neutral element. Domain: hyperdomain ℛ(𝕏).

### Definition 13.1 (Cumulative Unfusion Operator) — PNG 256 / p.238
Symbol: ⊖ (circle-minus reversed). Inverse of cumulative fusion. Given fused ω_X^C = ω_X^{A⋄B} and contributing ω_X^B, derives ω_X^A = ω_X^{C⊝B}. Two cases on uncertainty values. Non-commutative, non-associative, non-idempotent.

### Definition 13.2 (Averaging Unfusion Operator) — PNG 257 / p.239
Symbol: ⨸ (cumulative-average circle-minus). Inverse of averaging fusion. Idempotent, non-commutative, non-associative.

### Definition 13.3 (Cumulative Fission Operator) — PNG 259 / p.241
Symbol: ⓘ (circle-i, "circle-1"). Splits ω_X^C using fission parameter φ ∈ (0,1) into ω_X^{C1} = φ ⓘ ω_X^C and ω_X^{C2} = φ̄ ⓘ ω_X^C. Special cases φ=0 or φ=1 → one resulting opinion vacuous, the other equal to argument. Base rate preserved. Recombines via ω_X^{C1} ⊕ ω_X^{C2} = ω_X^C.

### Definition 14.1 (Reliability Trust, Gambetta 1988) — PNG 262 / p.244
"Reliability trust is the subjective belief with which an entity, A, expects that another entity, B, performs a given action on which A's welfare depends."

### Definition 14.2 (Decision Trust, McKnight & Chervany 1996) — PNG 265 / p.247
"Decision trust is the commitment to depend on something or somebody in a given situation with a feeling of relative security, even though negative consequences are possible."

### Definition 14.3 (Reputation, Merriam-Webster) — PNG 266 / p.248
"The common opinion that people have about someone or something."

### Definition 14.4 (Functional Trust Derivation Criterion) — PNG 272 / p.254
Last trust edge in a derived path must be functional/belief; all previous edges must be referral.

### Definition 14.5 (Trust Scope Consistency Criterion) — PNG 272 / p.254
Trust scope of the final functional/belief edge must lie in the intersection of the scopes of all previous referral edges in the path.

### Definition 14.6 (Probability-Sensitive Trust Discounting for Two-Edge Path) — PNG 274 / p.256
Operator symbol: ⊗. ω_X^{[A;B]} = ω_B^A ⊗ ω_X^B. Components:
- b_X^{[A;B]}(x) = P_B^A · b_X^B(x)
- u_X^{[A;B]} = 1 − P_B^A · Σ_{x∈ℛ(𝕏)} b_X^B(x)
- a_X^{[A;B]}(x) = a_X^B(x)
where P_B^A is the projected probability of trust opinion ω_B^A. Special cases: P_B^A = 1 → derived opinion equals received opinion; P_B^A = 0 → derived opinion is vacuous.

### Definition 14.7 (Trust Discounting for Multi-edge Paths) — PNG 278 / p.260
For chained trust path A_1 →⋯→ A_n → X:
- b_X^{A_1}(x) = P_{A_n}^{A_1} · b_X^{A_n}(x)
- u_X^{A_1} = 1 − P_{A_n}^{A_1} · Σ_x b_X^{A_n}(x)
- a_X^{A_1}(x) = a_X^B(x)
with referral trust projected probability P_{A_n}^{A_1} = ∏_{i=1}^{n-1} P_{A_{i+1}}^{A_i} (Eq 14.13).

### Definition 15.1 (Series-Parallel Graph) — PNG 289 / p.271
SP-graph reducible to single source-sink edge by (i) replacing edges incident to a degree-2 non-endpoint vertex with a single edge, and (ii) replacing parallel edges between same endpoints with a single edge.

### Definition 15.2 (Directed Series-Parallel Graph / DSPG) — PNG 290 / p.272
SP-graph whose directed edges form loop-free paths from a single source to a single sink.

### Definition 15.3 (Outbound-Inbound Set / OIS) — PNG 291 / p.273
For ordered (V_s, V_t), OIS = (outbound set of V_s) ∩ (inbound set of V_t).

### Definition 15.4 (Parallel-Path Subnetwork / PPS) — PNG 292 / p.274
Subnetwork formed by an OIS where OD(V_s) ≥ 2 in OIS and ID(V_t) ≥ 2 in OIS. V_s = source of PPS, V_t = sink.

### Definition 15.5 (Nesting Level) — PNG 292 / p.274
NL([V_m;V_n]) = number of PPSs the edge is part of. Determines trust computation order.

## Algorithms / Procedures

CCF three-step procedure (p.234-235):
1. **Consensus** — compute b_X^cons(x) = min(b_X^A, b_X^B) over hyperdomain values.
2. **Compromise** — for each x ∈ ℘(X), compute b_X^comp combining (a) residue·other-uncertainty terms, (b) base-rate-weighted overlap of residues for y∩z=x, (c) complementary base-rate-weighted disjunctive contribution for y∪z=x, y∩z≠∅, and (d) residue product for disjoint y,z with y∪z=x. Set b_X^comp(X)=0 (whole-frame mass becomes uncertainty).
3. **Merge** — apply normalization η, then b_X^{A♡B}(x) = b_X^cons + η·b_X^comp; u_X^{A♡B} = u_X^pre + η·b_X^comp(X).

### DSPG Analysis Algorithm (Fig 15.5, p.276-277)

Inputs: directed list of trust edges with attributes (source, target, scope σ, variant ∈ {referral, functional}, opinion). Output: derived opinion ω_X^A.

Steps:
- (a) **Prepare** — represent the trust network as set of directed edges with node pairs. Verify it is a DSPG.
- (b) **Identify each PPS** with source-target (V_s, V_t). Compute nesting level NL of every edge as the count of PPSs the edge is in.
- (c) **Select PPS** with highest nesting level among edges; if no PPS remains → go to (g).
- (d) **Discount each path** in the selected PPS using two-edge or multi-edge trust discounting (Def 14.6 or 14.7) — V_t is treated as the local target. Each path collapses to a single edge.
- (e) **Fuse all paths** in the PPS using the appropriate trust-fusion operator (selected per Ch. 12 selection criteria). PPS collapses to a single edge.
- (f) **Recompute nesting level** of the new replacement edge.
- Loop back to (c).
- (g) **No PPS remains** — series-only network: apply two-edge or multi-edge trust discounting; if already single edge, do nothing.
- (h) **Done** — single edge between analyst and target carries the derived opinion.

### Complex Non-DSPG Networks (§15.4)

Two methods to find a DSPG sub-network within a non-DSPG STN:
- **Exhaustive** — enumerate all possible DSPGs and their derived target opinions; pick the canonical expression with highest confidence (lowest uncertainty). Complexity Comp = lm(2^n − 1) where n = number of possible paths, m = average paths per DSPG, l = average edges per path.
- **Heuristic** — synthesize the graph by adding new paths in decreasing order of confidence; exclude any path that would turn the graph into a non-DSPG. Complexity Comp = lm. Not guaranteed optimal (may exclude multiple low-certainty paths incompatible with a previously included high-certainty path).

## Implementation Notes

- WBF requires three case branches; dogmatic (u=0) inputs need γ-limit handling, default γ=0.5 for symmetric dogmatic case.
- CCF whole-frame compromise mass on X must always be reassigned to uncertainty: b_X^comp(X) := 0.
- Cumulative unfusion has division-by-zero risk when u_X^B − u_X^C + u_X^B u_X^C = 0; degenerate when both arguments dogmatic.
- Cumulative fission with φ ∈ {0,1} produces one vacuous and one identity opinion — guard accordingly.
- Trust-discounting operator ⊗ uses *projected probability product* (not subjective-logic multiplication ·). This is one of the input arguments of the discounting operator (p.260).
- For multi-edge paths, P_{A_n}^{A_1} = 0 → derived opinion vacuous (full distrust kills propagation). P_{A_n}^{A_1} = 1 → derived opinion equals received functional opinion (full trust preserves).
- Trust revision: revision factor RF only revises non-base-rate components; base rate preserved.
- Revision is "ad hoc" — author explicitly invites alternative designs.
- DSPG analysis: the nesting-level ordering is mandatory; computing trust on a deep edge before its enclosing PPS is collapsed produces wrong results.

## Criticisms of Prior Work

- p.235: "The counter-intuitive results produced by Dempster's rule demonstrate that it is inadequate for this particular class of situations" — Zadeh's medical example. WBF or CCF "more adequate" than BCF (Dempster).
- p.247-248: Williamson (1993) [99] argues the notion of trust is redundant ("reliability, utility, and risk are adequate and sufficient"); Jøsang rejects: "computational trust [...] adds value [...] in the architectures and mechanisms for collecting trust-relevant information [...]."
- p.251: Jøsang rejects "enemy of my enemy" distrust-transitivity heuristics as belonging to "intelligence analysis discipline rather than online trust management" and explicitly chooses the conservative reduction-of-confidence approach.
- p.270: Self-criticism — trust revision is "ad hoc."

## Quotes Worth Preserving (additional from final pages)

- p.279: "The analyst should receive a source opinion in its original form from the source, and not in the form of an indirect derived opinion." — soundness requirement for STN data collection. Hidden derivation paths produce a perceived topology different from the real topology, leading to wrong fusion.
- p.280: "trust networks involve possible deception, which system reliability networks do not, and that trust networks involve fusion, which is not an ingredient of system reliability analysis. Hence, the principles for analysing complex reliability networks can not be applied to complex STNs."
- p.281: "The optimal derived DSPG trust network produces a derived opinion with the highest confidence. The goal is to maximise confidence in the derived opinion, and not e.g. to derive the opinion with the highest projected probability of some value of the variable X."

## Implementation Notes (additional)

- DSPG construction: parser must read directed edge list with (source, target, scope σ, variant ∈ {referral, functional}, opinion) — see Table 15.2 schema.
- Trust-network sanitization: when collecting opinions from a network, enforce that each agent supplies its own (non-derived) opinion. If agent passes a derived opinion, the analyst's perceived topology will differ from the real one. The analyst should solicit from each source separately so that the topology is correctly known.
- Goal of DSPG selection in non-DSPG networks: maximize confidence (minimize uncertainty), not projected probability — important for the optimization criterion when implementing.
- Heuristic DSPG discovery is greedy by certainty; cycle/non-DSPG check is part of the inclusion test.
