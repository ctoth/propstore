Proposes "trusty URIs" that embed SHA-256 cryptographic hashes directly into URIs to make digital artifacts (datasets, code, texts, images, RDF graphs) verifiable and immutable on the Web. Defines two module types — FA for byte-level file hashing and RA for RDF graph hashing with deterministic canonicalization — and evaluates implementations in Java, Perl, and Python on nanopublication datasets with over 1 billion triples. Relevant to propstore's content-hash addressed sidecar storage and provenance tracking, demonstrating how cryptographic verification can be integrated into identifier schemes for knowledge artifacts.
