# Abstract

## Original Text (Verbatim)

This article gives a tutorial introduction to the ASPIC+ framework for structured argumentation. It is based on two ideas: the first is that conflicts between arguments are often resolved with preferences, and the second is that the process that reasons about the process of argumentation is the same as the process of argumentation itself. ASPIC+ is illustrated with examples and several ways are discussed to instantiate the framework and to use it as a modelling tool. It is not a system but a framework for specifying systems. ASPIC+ leaves you fully free to make choices as to the logical language, the strict and defeasible inference rules, the set of premises, and the use of preferences. A main objective of the study of the ASPIC+ framework is to identify conditions under which instantiations of ASPIC+ satisfy logical consistency and closure properties.

---

## Our Interpretation

The paper addresses the gap between abstract argumentation (Dung 1995), which treats arguments as atomic nodes, and the need for structured arguments built from premises and inference rules with different attack types. ASPIC+ provides a parameterized framework where users choose their logic, rules, and preferences, then the framework constructs arguments, computes attacks/defeats, and generates Dung-style abstract frameworks. The key finding is that rationality postulates (consistency, closure) are satisfied when the system is closed under transposition and axioms are consistent -- giving implementers clear conditions to check.
