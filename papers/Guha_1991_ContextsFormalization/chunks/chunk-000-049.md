### 2.1.2 Syntax and Semantics (p.20-23)
- Context Structure CS assigns each context C: a Language L(C) and set of models CM(C) *(p.21)*
- Formula meaningful in C iff every non-ist symbol is part of L(C) and used appropriately *(p.21)*
- Satisfaction: tuple <U, CS, S> satisfies F in C iff U in CM(C), F meaningful, FOL truth recursion *(p.22)*
- ist(ck, P) satisfied iff Ck is context and P satisfied in Ck by every model in CM(Ck) *(p.22)*
- presentIn(c, A): term A has a referent in context c *(p.23)*
- corefer(C1, A, C2, B): A in C1 and B in C2 denote same object *(p.23)*
- CM defines DAGs of contexts; no self-reference *(p.23)*
- Nonmonotonicity via circumscription; Cyc uses Argumentation Based Default Reasoning *(p.23)*
- Barkan formula ist(c (forall x p(x))) <=> (forall x ist(c p(x))) does NOT hold *(p.24)*
- Only one level of nesting assumed for rest of thesis *(p.24)*

### 2.2-2.3 Example Axioms and Proofs (p.28-31)

### 2.4 Lifting (p.31-34)
- Lifting = relative decontextualization
- Default Context-independence Rule (DCR): symbol meaning doesn't change between contexts by default *(p.33)*
- Default Coreference Rule (DCT): terms denote same objects across contexts by default *(p.33)*
- DCR-P for predicates, DCR-T for terms *(p.33-34)*
- Abnormality predicates (ab-dcp, ab-dct) block lifting when asserted *(p.34)*
- Composition Rule (CR) for lifting non-atomic formulas *(p.34)*

### 2.5-2.7 Lifting Examples and Model Theory (p.35-40)
- Model-theoretic account: context dependence via background theory *(p.36)*
- Relative decontextualization = reporting relative to different background theory *(p.37)*
- Containment vs biconditional: containment is stronger *(p.40)*

### Chapter 3: The Applications of Contexts (p.41-49)

**3.1 Different Uses of Contexts** *(p.41-46)*
1. Groups of related assertions:
   - (a) Microtheories (Mts) - theories of topics with suffix Mt *(p.41)*
   - (b) Problem Solving Contexts (PSC) - tailored for specific problems *(p.42)*
   - (c) Discourse Contexts *(p.42)*
2. Context of a Query - queries posed in PSC *(p.42)*
3. Combining different theories via lifting *(p.42-43)*
4. Multiple models of a situation - efficiency/accuracy tradeoffs *(p.44)*
5. Natural language representation *(p.45-46)*
   - English word "seller" highly polysemous - many related distinct meanings *(p.46)*

**3.2 Lifting Rules** *(p.47-49)*
- DCR-T: default term coreference *(p.48)*
- DCR-P: default predicate context-independence *(p.48)*
- Restrict lifting to: microtheories -> Problem Solving Contexts *(p.48)*
- Compositional lifting via CR rule *(p.48-49)*
- Two defaults: no-modification lifting + modification lifting rules *(p.49)*

**3.3 Problem solving with contexts** *(p.49)*
- Lift and Solve: lift assertions into PSC, use conventional solver *(p.49)*
- Shift and Solve: switch into existing context, solve there, lift answer *(p.49)*
- Current implementation favors Lift and Solve *(p.49)*
