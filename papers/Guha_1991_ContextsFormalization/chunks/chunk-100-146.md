## Chapter 4: Current Work and Related Work (p.117-)

### 4.1 Current Work on Contexts (p.117-123)

#### 4.1.1 Use of Context for Device Models (p.117-123)
- Forbus & Falkenheiner problem: qualitative simulation of a device requires partial model (ignore components, abstract as black boxes, ignore aspects, make operating-condition assumptions)
- **Forbus's approach (p.118)**: consider(...) statements disjoint with axioms; minimize extent of consider — similar to `ab` literals in nonmonotonic logic
- **Context-based approach (p.118)**: introduce context mc, use modeling rules (like lifting rules) to build model
- **Device Model problem (p.119)**: Context-independent models (deKleer) are goal; but universally adequate model is too detailed → use multiple models per device
- **Constraints between models (p.119-120)**: Cruder models derivable from accurate via assumptions (Weld on when extreme-value assumptions are valid)
- **Kinds of standard assumptions (p.120)**:
  - (a) Variable takes extreme value (Weld)
  - (b) "Regularizing" shape/curve to regular form
  - (c) Ignoring components
  - (d) Ignoring behaviors (e.g. transients)
- **Assumptions that may be consistent OR inconsistent with accurate model**; inconsistent ones require retraction of accurate-model axioms *(p.120)*
- **Automating device model generation (p.121)**:
  - fixVariable(CDT0 f ef): approximation assumes f(d)=ef, default inherits CDT0
    - ist(CDT0 allInstanceOf(d DT)) => ist(fixVariable(CDT0 f ef) f(d)=ef)
    - ist(CDT0 p) -> ist(fixVariable(CDT0 f ef) p)  (default)
  - ignorePart(CDT0 DT1): ignore components of type DT1
    - ist(ignorePart(...) allInstanceOf(d DT)) => ¬(Exists d1 allInstanceOf(d1 DT1) ^ parts(d d1))
    - ist(CDT0 p) -> ist(ignorePart(CDT0 DT1) p)  (default)
  - ignoreAspect(CDT0 A): ignore aspect type A of device
    - aspectType(f A); ist(CDT0 allInstanceOf(d DT) ^ aspectType(f A)) => ¬ist(ignoreAspect(CDT0 A) (f(d)=x))
    - ist(CDT0 p) -> ist(ignoreAspect(CDT0 A) p)  (default)
- **approxModel** is transitive: approxModel(C1 C2) ^ approxModel(C2 C3) => approxModel(C1 C3) *(p.122)*
- Non-default-preserved assumptions: ist(c p) ^ approxModel(c ci) => ist(ci p)
- fixVariable(c f e) ci => ist(ci allInstanceOf(d DT) => (= f(d) e))  — assumption propagated to derived
- deviceModel(DT c) — c is a model of DT (minimized)
- aspectIncluded(ci MechanicalAspect) => ist(ci (not (mass(x)=0)) and volume(x)=0)
- **Building model for simulation (p.122-123)**: recursive: pick deviceModel(DT ci), use baseMt(psc ci) to include model assertions; exclude inconsistent (ist(c False) => not baseMt(ci c)); approximation-derivation preserves exclusion
- **Model selection heuristics (p.123)**:
  - (a) Don't assume away the answer
  - (b) Eliminate by error bounds (as in example 12)
  - (c) Eliminate if model fails to decide the query
  - (d) Prefer simpler models (approxModel(c0 C1) => prefer C1)

#### 4.2.3 Framework for Context-based NLU (p.124-125)
- **Hierarchy of contexts by granularity**:
  - Utterance contexts (UC) — one per utterance, ordered temporally *(p.125)*
  - Discourse context — partial decontextualization of utterances; may also be Mt and/or PSC
- **Utterance context language features (p.125)**:
  - (a) Pronouns/Indexicals as terms: He, She, It, Now, I
  - (b) Definite/indefinite: functions The, A — "the lady owns a bag" -> owns((The Lady)(A Bag))
  - (c) Variadic Etc — "Fred likes ice cream, softees, etc." -> likes(Fred (Etc IceCream Softee))
  - (d) Preposition predicates: to, with, for — "Fred bought the rose for Jane" -> exists e allInstanceOf(e Buying) ^ object(e (The Rose)) ^ for(e Jane)
  - (e) Lexicon references when NL can't resolve: englishWord(c "bank")
  - (f) References to other utterance contexts
- **Example translations (p.125-126)**: "send a package to Karen. Where is she?"; "Fred bought a ticket. He lost it"; "A dog has an appendix. It does not use the appendix"
- **(The Lady) resolution via = (The Lady) ?x** — can be undecidable but referent still well-defined *(p.126)*
- Key restriction: a single utterance context must use each term to denote exactly one object *(p.126)*

#### 4.2.4 Utterance Contexts vs Discourse Contexts (p.127-129)
- **Lifting formula from UC to discourse context requires transformation** *(p.127)*
- **Constraints on resolving "it" (p.127-129)**:
  - (a) Linguistic-semantic: "it" != male/female human
    - (forall c_i utteranceContext(c_i) => ist(c_i (forall y (=y it) => ¬(male(y) or female(y)))))
  - (b) Linguistic context:
    - (ii) "it" must be in some previous context: (forall x ist(c_i (it=x))) => (Exists c_j allPreviousUC(c_i c_j) ^ presentIn(c_j x))
    - (iii) Minimize referent count via domain closure
    - (iv) Closer context preferred: allPreviousUC(c_i c_j) ^ ¬allPreviousUC(c_j c_i) => ab-it(c_i c_x x) < ab-it(c_j c_x y)
  - priorContext vs previousContext — priorContext can differ (e.g. "Going back to..." cue statement)
  - Structure-sensitive (Grosz&Sidner style): previousContext(c_ic_j) ^ ¬ab-pc(c_ic_j) => priorContext(c_ic_j)
  - ist(c_i P) ^ contextTopic(c_j X) ^ ¬(exists c_k contextTopic(c_k X) ^ nearer(c_ic_kc_j)) -> priorContext(c_ic_j)
  - (c) Background of speaker/listener: "transformer in my amplifier is broken" - depends on whether addressee is transformer repair person or amp store rep
  - (d) Utterance situation: shared-focus objects e.g. screwdriver in hand
  - (e) Felicity constraints: conversational postulates — "bird made of wood" over "tree made of wood" (new info preferred)
  - (f) Background common sense: "cup of coffee... I drank it" — common sense eliminates cup, leaves coffee. "Mary bought watch in shop... bought it" — likely watch over shop *(p.129)*

### Comparison to DT and DST (p.132-133)
- **vs Kamp's Discourse Theory (DT)**:
  - (a) Contexts here are domain objects, not data structures in translation
  - (b) DT utterance interpretation independent of following utterance's content; here, later utterances can affect interpretation via KB
  - (c) DT addresses only fine-grained contextuality (pronouns, anaphora)
  - (d) DT assumes THE correct decontextualized translation exists; this thesis proposes translations retain contextual elements
- **vs Grosz & Sidner DST**: DST treats contexts as data structures; focuses on discourse structure, not meaning-determination; this thesis is "meta" to DST

### 4.3 Related Work in AI (p.133-134)
- **McCarthy [jmc 59]**: concept of context as representation tool first proposed
- **ATMS [deKleer]**: context = consistent propositional-assumption set; NOT rich first-class objects; assumptions a priori enumerated; no relative decontextualization
- **FOL and descendents [Weyrauch, Fausto]**: multiple meta/base theories; meta & base don't share ontology; FOL meta-theory concerns base-theory expressions (not objects); no relative decontextualization
- **Packages [Steele]**: Common Lisp packages — different symbol↔value mappings per package; no assumptions/decontextualization
- **Partitioned Semantic Nets [Henricks, KEE, MRS, Russell]**: only similarity is multiple theories; no interaction; no assumptions; no decontextualization
- **Forbus**: closest in spirit; consider statements ≈ assumptions on STATEMENTS; doesn't handle assumption-driven vocabulary change; no explicit contexts; no relative decontextualization
