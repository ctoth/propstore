BLOOMS aligns Linked Open Data schemas by building Wikipedia/DBpedia category forests for ontology class names and comparing their asymmetric overlap. The paper defines preprocessing, tree construction, thresholded subclass/equivalence decisions, and post-processing with Alignment API plus Jena, then reports stronger LOD schema-alignment precision/recall than contemporary matchers. It is relevant to propstore as a concrete model for provenance-bearing, directional ontology alignment claims.
