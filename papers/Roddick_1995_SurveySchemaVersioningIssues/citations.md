## Key Citations for Follow-up

- **Miller, Ioannidis, Ramakrishnan (1993)** — formal "information capacity" criterion for lossless schema integration. Foundational for any propstore migration framework that wants a precise loss-detection rule.
- **McKenzie and Snodgrass (1990)** "Schema evolution and the relational algebra" — one of three temporal algebras explicitly extended for schema evolution; algebraic-expressibility precedent.
- **Jensen et al. (1994)** "A consensus glossary of temporal database concepts" — the nomenclature substrate Roddick adopts; needed for cross-paper terminological consistency in the migration batch.
- **Clamen (1992)** "Class evolution and instance adaptation" — canonical case for "versioning preserves application compatibility where strict-conversion evolution does not."
- **Sjøberg (1993)** "Quantifying schema evolution" — empirical data on real-world schema-change rates; relevant if propstore wants to choose lazy vs eager re-projection on evidence.
