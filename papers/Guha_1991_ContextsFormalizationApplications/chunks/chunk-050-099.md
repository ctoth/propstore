## Rules/Algorithms

### Lift and Solve Algorithm (p.50-52)
1. Receive query Q in Problem Solving Context PSC
2. Access Module retrieves relevant assertions from KB
3. Lifting Module determines which assertions to import from other contexts
4. For each assertion: check Relevance (is it relevant?) and Appropriateness (is theory accurate enough?)
5. Lift selected assertions into PSC using DCR rules
6. Inferencing Module derives conclusions within PSC
7. Return answer

### Temporal Lifting Protocol (p.60-65)
1. Check if assertion has explicit temporal qualification
2. If no explicit temporal: store in CSPSC-T_1 (default time projection)
3. If explicit temporal: store in CSPSC with holds(T, p) wrapper
4. When querying: check if query needs explicit time model
5. Lift between CSPSC and CSPSC-T_1 as needed

### Theory Selection (p.93-95)
1. Given multiple theories that could apply to current problem
2. For each theory: estimate error in predictions vs cost of using theory
3. If theory's predictions are of acceptable quality for the task, use it
4. If multiple theories acceptable, prefer cheapest (simplest)
5. Block lifting from theories whose precision is insufficient
