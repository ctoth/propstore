Presents two sampling-based approximation algorithms for ASPIC+ reasoning: one samples arguments independently, the other expands from a query by sampling attackers directionally. The main claim is pragmatic: if the knowledge base is large enough, you can cut runtime sharply while keeping query answers highly accurate. Useful for propstore whenever complete argument construction is too expensive but a fast, explanation-bearing subgraph is still acceptable.
