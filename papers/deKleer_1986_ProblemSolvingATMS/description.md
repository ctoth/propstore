Presents a complete control regime and constraint language for building problem solvers on top of the Assumption-based Truth Maintenance System (ATMS), including a consumer architecture where modular rules fire based on ATMS label status. Key contributions include scheduling strategies (simplest-label-first), dependency-directed backtracking via control disjunctions, and a primitive constraint language (PLUS, TIMES, AND, OR, ONEOF) translated entirely into ATMS justifications and consumers. Directly foundational for the propstore world model's assumption-tracking, environment management, and hypothetical reasoning architecture.
