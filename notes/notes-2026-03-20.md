### 2. form_utils.py (lines 1-257)
**Location:** `/c/Users/Q/code/propstore/propstore/form_utils.py`

**Key Classes:**
- `FormDefinition` dataclass (lines 17-27):
  - name: str
  - kind: KindType (from cel_checker)
  - unit_symbol: str | None
  - allowed_units: set[str]
  - is_dimensionless: bool
  - parameters: dict
  - dimensions: dict[str, int] | None
  - extra_units: list[dict[str, Any]]

**Key Functions:**

1. **load_form()** (lines 33-115):
   - Loads single form YAML, caches results
   - kind determination (lines 52-65):
     - Prefers explicit `kind` field from YAML
     - Falls back to name-based heuristic via `kind_type_from_form_name()`
     - _KIND_MAP: "quantity" → QUANTITY, "category" → CATEGORY, "boolean" → BOOLEAN, "structural" → STRUCTURAL
   - Extracts unit_symbol, allowed_units, parameters, dimensions, extra_units
   - dimensionless logic (lines 78-87):
     - Prefers explicit `dimensionless` field
     - Falls back to: base == "ratio" OR (no unit_symbol AND kind == QUANTITY AND not "structural")

2. **kind_type_from_form_name()** (lines 142-152):
   - Maps form names to KindType:
     - "category" → CATEGORY
     - "structural" → STRUCTURAL
     - "boolean" → BOOLEAN
     - others → QUANTITY (default)

3. **allowed_units_from_form_definition()** (lines 177-192):
   - Extracts unit symbols from:
     - unit_symbol field
     - common_alternatives[].unit field

4. **validate_form_files()** (lines 212-256):
   - Validates all form YAML files against JSON schema
   - Checks: dimensionless vs dimensions consistency, name field matches filename

### 4. Concept Creation via CLI (concept.py)
**Location:** `/c/Users/Q/code/propstore/propstore/cli/concept.py`

**Command: `pks concept add`** (lines 119-188):
- Options: --domain (required), --name (required), --definition (prompted), --form (prompted)
- Creates YAML file with:
  - id: conceptN (incremented via .counters/)
  - canonical_name: name
  - status: "proposed"
  - definition
  - domain
  - created_date: today's date
  - form: form_name
- Validates with validate_concepts()
- Writes file and increments counter

**Other concept commands:**
- alias: Add aliases to concept
- rename: Rename concept (updates filename, all references in relationships/conditions)
- deprecate: Mark as deprecated with replacement
- link: Add relationships (broader, narrower, related, component_of, derived_from, contested_definition)
- search: FTS or grep fallback
- list: List with filters
- show: Display YAML
