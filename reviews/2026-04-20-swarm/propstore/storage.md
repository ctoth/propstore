# Storage + Source + Provenance Subsystem Review

Review scope: `propstore/storage/`, `propstore/source/`, `propstore/provenance/`, `propstore/repository.py`, `propstore/source_calibration.py`. All line numbers are 1-indexed.

## Principle Violations

- `propstore/storage/merge_commit.py:10` — Storage (Layer 1) imports `propstore.merge.merge_classifier.build_merge_framework` and `propstore.merge.merge_report.semantic_candidate_details`. Upward dependency from storage into argumentation/merge layer. CRIT. Fix: invert — higher layer builds the merge framework, hands storage a typed payload of what to commit.
- `propstore/storage/merge_commit.py:49-55` — Merge commit filters to only include arguments whose `artifact_counts[argument.artifact_id] == 1`; every artifact where multiple rival arguments converge is silently dropped. Classic "collapse disagreement in storage." CRIT. Fix: materialize all arguments with provenance and let render filter.
- `propstore/storage/merge_commit.py:42-47` — Non-claim paths are merged last-write-wins (left-tree overwrites right-tree) rather than surfacing the conflict. Disagreement collapsed at two-parent merge for concepts/stances/forms/contexts/sources. CRIT. Fix: reject when paths conflict unless explicitly resolved.
- `propstore/storage/merge_commit.py:57-64` — Synthetic `source` block stamps `extraction_model="merge"` with today's date and no provenance status. A heuristic-derived claims file is written with fabricated source metadata. HIGH. Fix: require `ProvenanceStatus` on the merge source record; mark `defaulted`.
- `propstore/source_calibration.py:93-114` — Layer 1 calibration (`source_calibration.py`) imports from `propstore.world` (belief-set/argumentation surface) at runtime. Storage-adjacent code reaches into the world engine via `WorldModel.chain_query` to calibrate a source trust value, then writes the result back into the canonical source document. HIGH layering violation plus HIGH principle violation — derived result mutates the source of truth without a proposal artifact.
- `propstore/source_calibration.py:140-145` — If `resolved_prior is not None`, `trust["status"]` is set to `CALIBRATED`. Lines 144-145 then *downgrade* calibrated to defaulted if `derived_from` is empty; fine, but a calibrated prior (a heuristic output) overwrites the authored prior in the stored document. This is the exact "source mutated by heuristic output" pattern. HIGH. Fix: write the calibrated value as a proposal artifact referenced by the source, not into the source itself.
- `propstore/source/promote.py:180,184,206` — `resolve_source_concept_promotions` raises `ValueError("ambiguous concept mappings")` instead of letting rival concept mappings flow into storage for render-time arbitration. Gate before render. HIGH. Fix: emit both candidate mappings with provenance; resolve at render.
- `propstore/source/promote.py:504-506` — Promotion raises on unresolved concept references. Source data that cannot be auto-resolved is blocked from reaching storage entirely. Violates design checklist Q1 ("Does this prevent ANY data from reaching the sidecar?"). HIGH. Fix: record as unresolved in storage with vacuous provenance.
- `propstore/source/alignment.py:78` — `classify_relation` returns `"non_attack"` whenever entries differ by both form/uri and references. There is no code path that returns `"ignorance"` — epistemic uncertainty between two unrelated concept proposals is forcibly classified as non-attack. HIGH principle violation.
- `propstore/source/alignment.py:258` — `accepted_id = accepted[0]` silently selects the first accepted alternative in alignment decisions. Multiple accepted alternatives → first wins. Collapses disagreement. HIGH.
- `propstore/source/registry.py:47` — `handle_to_artifact.setdefault(alias_name, artifact_id)` first-wins alias resolution. Rival concepts sharing an alias are silently hidden. MED principle violation.
- `propstore/source/registry.py:127-128` — `_derived_concept_artifact_id(input_ref)` silently derives a synthetic artifact id for any unresolved handle, making "no known mapping" indistinguishable from "here is an authored mapping." Downstream consumers cannot tell derived vs. authored. MED.
- `propstore/source/claims.py:74-85` — `validate_source_claim_concepts` raises `ValueError("unknown concept reference(s)")`. Blocks source-level commits before data reaches storage. Gate before render. MED.
- `propstore/source/concepts.py:42-43` — Raises when concept entry is missing `local_name`/`proposed_name`/`definition`/`form`. Source-layer gate that prevents partial concepts from reaching storage. MED.
- `propstore/source/common.py:66-76` — Initial source document hard-codes `prior_base_rate=0.5` and `a=0.5` (OMIT without citation). The status is correctly `DEFAULTED`/`VACUOUS`, but the magic numbers are duplicated in `source_calibration.py:77-83` (`b=0, d=0, u=1, a=0.5`, prior=0.5) without a named constant with a citation. LOW. Fix: a single `DEFAULT_SOURCE_PRIOR = 0.5` with a literature pointer.
- `propstore/provenance/__init__.py:141-182` — `compose_provenance` dedupes witnesses by tuple key and dedupes `derived_from`/`operations` — this loses frequency information (how many times a defaulted witness contributed). Dedupe is principle-neutral for sets, but the function is the sole fusion path; information about "this witness asserted twice from different contexts" is silently merged. LOW.
- `propstore/provenance/__init__.py:349-371` — `stamp_file` uses regex-based YAML/frontmatter stamping. Any existing `produced_by:` block in nested mappings is also matched (both regexes have `re.MULTILINE` but only match at line start; nested blocks are indented, so they should be safe). But there is no typed provenance status; `produced_by` only records agent/skill/plugin_version/timestamp. No `ProvenanceStatus` enforced on stamped files. HIGH principle violation — the stamping shim does not carry typed provenance, contradicting "Every probability-bearing document value that enters the argumentation layer must carry typed provenance."

## Summary of critical items

1. `merge_commit.py` collapses disagreement at two-parent merge time (CRIT).
2. `merge_commit.py` imports from merge/argumentation layer (CRIT layering).
3. `source/promote.py:743-751` — Zip-slip-style path-escape in `sync_source_branch` (HIGH security).
4. `source_calibration.py` mutates the canonical source document with calibrated prior derived from world-layer query (HIGH layering + principle).
5. `source/finalize.py:100` calls `derive_source_trust` which reaches into `propstore.world` (HIGH layering).
6. `source/alignment.py` — ignorance branch unreachable; epistemic uncertainty collapsed to non-attack (HIGH principle).
7. Multiple gates that block data from reaching storage (concept refs, concept fields, unresolved promotion concepts) (HIGH/MED principle).
8. `merge_commit.py:42-47` last-write-wins for non-claim paths (CRIT).
