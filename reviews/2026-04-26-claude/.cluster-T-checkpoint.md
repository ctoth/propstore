## Observations so far (canonicalizer.py)

1. `_is_constant_value` (line 486-488) uses `type(node.value) == type(value)` — guards bool vs int (since `True == 1` in Python). Good.

2. `_SAFE_OPS` constant folding (line 367) catches `ZeroDivisionError, OverflowError, ValueError` but NOT `TypeError`. `1 + "a"` would crash. Probably unreachable from valid Python but worth noting.

3. **Identity elimination is mathematically WRONG for non-numeric types.** `visit_BinOp` (line 360+):
   - `x + 0 -> x` is wrong if x is a list/string (TypeError at runtime, not identity).
   - `x * 1 -> x` is wrong if x is a list (`[1,2]*1` works but `[1,2]*0` doesn't fold here; the issue is that `x * 1` for a list IS x but for a numpy array it's a copy).
   - The comment at line 408 explicitly notes Add commutative sort is excluded "because + is overloaded for lists/strings (not commutative)" — yet identity elimination on Add does NOT have the same guard. INCONSISTENCY.
   - For numeric IEEE-754: `x + 0.0 -> x` is wrong for x = -0.0 (gives 0.0). And `x * 1` is wrong for NaN propagation considerations only if 1 were 1.0 and x is NaN — actually `NaN * 1 == NaN` so that's fine. But `0 * x` for x=NaN/inf changes semantics: `0 * inf = NaN`, but the rule `0 * x -> ?` isn't here (good).

4. **Constant folding doesn't preserve types.** `1 / 2 -> 0.5` (float) replaces an `int/int`. If downstream code does `isinstance(result, int)` check, semantics changes. Mostly OK for "equivalence" purposes but weird.

5. **Pow identity `x ** 1 -> x`**: wrong for x=NaN? Actually `NaN ** 1 = NaN`, OK. But `(-0.0) ** 1 = -0.0`, OK. Hmm fine.

6. **Repeated multiplication to power** (`_collect_mult_power_chain`, line 203):
   - `x * x -> x ** 2`. This is WRONG if x has side effects (e.g., x is a function call). But here x is an AST node — for a `Call` node, `f() * f()` would collapse to `f() ** 2`, which calls f once instead of twice. SEMANTICS-CHANGING BUG.
   - Also `x * x` for floating point: `x * x == x ** 2` mathematically but IEEE may differ in rounding for some pathological values.

7. **Commutative sort by `ast.dump()` string comparison** (line 410-413): sort key is lexicographic string. `Constant(value=10) > Constant(value=2)` lexicographically is False (since "1" < "2"). Order depends on string repr of the dump. Stable but arbitrary; OK as long as it's consistent.

8. **Mult commutative sort but NOT recursive normalization of associativity.** `(a*b)*c` vs `a*(b*c)` will not canonicalize to the same form unless both happen to sort the same way at each level. Pierce/KB would require flat normalization. POTENTIAL FALSE NEGATIVE.

9. **AugAssign normalization** (line 293): `x += y -> x = x + y`. For `x += [1]` where x is a list, this changes from `extend` to `concatenate-and-rebind` — semantics change for shared references. But for algorithm equivalence at the abstract level, OK.

10. **Range normalization** (line 309): `range(0, N, 1) -> range(N)`. Correct.

11. **Boolean normalization** (line 324) only fires when LHS is bool-typed Name. Won't fire for `expr == True` where expr is a complex expression. Conservative — good.

12. **WhileToFor normalization** (line 491+):
    - Requires `init_val` is constant int/float, but range only accepts ints. If init is float, builds `range(2.5, N)` — TypeError at runtime. Should require int.
    - Also doesn't check that the increment is the LAST statement and that there are no `break`/`continue` in the body that would skip the increment. SEMANTICS-CHANGING BUG: a `while` with `break` mid-body that's converted to `for` runs the loop body in different iteration counts.
    - Doesn't check that `var` isn't modified mid-body other than at the end. `while i < n: i += 5; ... i += 1` would be wrongly converted.

13. **TempVariableInliner**: counts stores/loads via `ast.walk` of a synthetic Module wrapping the stmt list. This walks INTO nested function defs — if there's a nested function that uses `var`, the count is inflated/deflated wrong. PROBABLY MINOR.

14. **TempVariableInliner safety check `_is_safe_inline_expr`**: allows `math.X` calls. But e.g. `math.sqrt(-1)` raises ValueError — moving the call to the use site changes when the exception fires. If the use site is inside a conditional branch that's never taken, the exception won't fire post-inlining. SEMANTICS-CHANGING but acceptable for canonicalization perhaps.

## Observations so far (sympy_bridge.py)

20. **`ast_to_sympy` rejects boolean constants** (line 39-40) — but accepts `Name` for any identifier as a Symbol. Names like `True` pre-3.x... no, in 3.x `True` is a Constant. OK.

21. **`sympy.simplify(a-b) == 0`** (line 132) — sympy.simplify can be SLOW and INCOMPLETE. Two expressions may be algebraically equal but simplify-difference doesn't reduce to 0. So this is incomplete (false negatives) — acceptable.
   - But it can also be UNSOUND for expressions involving branch cuts: `sqrt(x*y)` vs `sqrt(x)*sqrt(y)` — sympy may incorrectly simplify these to 0 difference for symbolic x, y under simplify, which is wrong for negative reals. For abstract algorithm comparison probably OK.

22. **`math.log` is converted to `sympy.log`** (line 96-97): Python's `math.log(x)` is natural log — `sympy.log` defaults to natural. OK.

23. **`math.log(x, base)`** would have len(node.args)==2 — but the bridge requires `len(node.args) != 1` to error (line 84). So `math.log(x, 2)` raises AstToSympyError → returns False from `_sympy_expr_equal`. Correct (conservative).

24. **`_reduce_stmts`** does naive forward inlining. If RHS has side effects (e.g., function call), substituting it can duplicate or eliminate calls. Same issue as TempVariableInliner. The `_references_var` check only prevents self-ref, not side effects.

25. **`sympy_expressions_equivalent` requires exact `ast.dump(args)` match** (line 314) — exact param signature including default values, annotations. Strict.

26. **`_stmt_equivalent` for For loops requires `ast.dump(sa.iter) != ast.dump(sb.iter)` to MATCH** (line 258). So `for i in range(n)` vs `for i in range(0, n)` would differ — but those should have been normalized by `visit_Call`. OK if canonicalizer ran. But here `_stmt_equivalent` is called on `tree_a` and `tree_b` which were canonicalized. OK.

27. **`_stmt_equivalent` doesn't use SymPy on If.test or While.test** (lines 265, 272). Tests must match exactly via `ast.dump`. Inconsistent — assignments do SymPy, predicates don't. Could cause false negatives.
